Program Verification 03 - Hoare Logic

# 前置知识

在阅读本文档前，请确保你已了解操作语义的相关概念，详见 Program Verification 知识板块中的 02 - Operational Semantics 文档。

  • Program Verification 02 - Operational Semantics

# 基本概念

霍尔逻辑 (Hoare logic, or Floyd–Hoare logic) 是一种形式化系统，用于严格推理计算机程序的正确性。

霍尔三元组 (hoare triple) 是霍尔逻辑的核心，其基本形式为 {p}S{q}\{p\}S\{q\}{p}S{q} ，其中 ppp 和 qqq 分别为 前置条件 (pre-condition) 和 后置条件 (post-condition) ，SSS 是一个程序片段（即一系列的程序指令）。

  • 前置条件和后置条件是用一阶逻辑公式描述的 assertion 。

非形式化地讲，霍尔三元组 {p}S{q}\{p\}S\{q\}{p}S{q} 合法 (valid) 的含义是：假定前置条件 ppp 成立，那么在执行完程序 SSS 之后，后置条件 qqq 将会成立。

可以从程序状态的角度描述霍尔三元组：从满足前置条件 ppp 的程序状态出发，执行程序 SSS 之后，到达的程序状态应当满足后置条件 qqq 。

  • 如果从不满足 ppp 的状态出发，则执行 SSS 之后的结果是没有保证的，霍尔三元组并不生效，这和不合法是不同的概念。

如下是一些合法的霍尔三元组的示例：

{x=1}x:=x+1{x=2}{x>3}y:=x{y>0}{x≤10}while (x≠10) do (x:=x+1){x=10}{x=1}skip{true}\begin{gathered} \{x = 1\} & x := x + 1 & \{x = 2\} \\ \{x > 3\} & y := x & \{y > 0\} \\ \{x \le 10\} & \text{while } (x \ne 10) \text{ do } (x := x + 1) & \{x = 10\} \\ \{x = 1\} & skip & \{true\} \end{gathered}{x=1}{x>3}{x≤10}{x=1}​x:=x+1y:=xwhile (x​=10) do (x:=x+1)skip​{x=2}{y>0}{x=10}{true}​

在霍尔三元组中，仅用于前置和后置条件中、且具有恒定值的变量被称为 逻辑变量 (logical variables) 或 幽灵变量 (ghost variables) 。例如在下面的霍尔三元组中，x0x_0x0​ 和 y0y_0y0​ 就是逻辑变量：

{x=x0∧y=y0}r:=x;x:=y;y:=r{x=y0∧y=x0} \{x = x_0 \land y = y_0\} \enspace r := x;\ x := y;\ y := r \enspace \{x = y_0 \land y = x_0\} {x=x0​∧y=y0​}r:=x; x:=y; y:=r{x=y0​∧y=x0​}

我们沿用在命题逻辑和谓词逻辑中使用的术语，用 σ,τ\sigma,\ \tauσ, τ 表示状态，用 ⊨\vDash⊨ 表示满足，例如 σ⊨{p}S{q}\sigma \vDash \{p\}S\{q\}σ⊨{p}S{q} 。特别地，用 ⊨{p}S{q}\vDash \{p\}S\{q\}⊨{p}S{q} 表示霍尔三元组合法。

# 正确性

事实上，使用标准的霍尔逻辑只能证明 部分正确性 (partial correctness) ，因为它并不要求程序 SSS 能终止，例如下面的霍尔三元组也被视为是合法的：

{x=1}while true do skip{x=2} \{x = 1\} \enspace \text{while } true \text{ do } skip \enspace \{x = 2\} {x=1}while true do skip{x=2}

非形式化地讲，完全正确性 (total correctness) 是终止性与部分正确性的结合。我们用不同的记号 ⊨tot{p}S{q}\vDash_{tot} \{p\}S\{q\}⊨tot​{p}S{q} 来表示用于描述完全正确性的霍尔三元组，有时也用 [p]S[q][p]S[q][p]S[q] 表示。现在我们重新定义霍尔三元组的合法性：

一个描述部分正确性的霍尔三元组 {p}S{q}\{p\}S\{q\}{p}S{q} 是合法的，当且仅当：

  • 从满足前置条件 ppp 的程序状态出发，执行程序 SSS ；

  • 如果程序 SSS 能终止，则执行指令之后到达的程序状态应当满足后置条件 qqq 。

一个描述完全正确性的霍尔三元组 [p]S[q][p]S[q][p]S[q] 是合法的，当且仅当：

  • 从满足前置条件 ppp 的程序状态出发，执行程序 SSS ；

  • 程序 SSS 能终止；

  • 执行指令之后到达的程序状态满足后置条件 qqq 。

形式化地讲:

(⊨{p}S{q})⇔(∀σ,σ⊨{p}S{q})(⊨tot{p}S{q})⇔(∀σ,σ⊨tot{p}S{q})(σ⊨{p}S{q})⇔(σ⊨p→(M(c,σ)−⊥d)⊨q)(σ⊨tot{p}S{q})⇔(σ⊨p→M(c,σ)⊨q)(σ⊨{p}S{q})⇔(σ⊨tot{p}S{q}∨M(c,σ)=⊥d)\begin{aligned} & (\vDash \{p\}S\{q\}) & \lrArr & \enspace (\forall \sigma,\ \sigma \vDash \{p\}S\{q\}) \\ & (\vDash_{tot} \{p\}S\{q\}) & \lrArr & \enspace (\forall \sigma,\ \sigma \vDash_{tot} \{p\}S\{q\}) \\ \\ & (\sigma \vDash \{p\}S\{q\}) & \lrArr & \enspace (\sigma \vDash p \to (M(c,\ \sigma) - \bot_d) \vDash q) \\ & (\sigma \vDash_{tot} \{p\}S\{q\}) & \lrArr & \enspace (\sigma \vDash p \to M(c,\ \sigma) \vDash q) \\ \\ & (\sigma \vDash \{p\}S\{q\}) & \lrArr & \enspace (\sigma \vDash_{tot} \{p\}S\{q\} \lor M(c,\ \sigma) = \bot_d) \end{aligned}​(⊨{p}S{q})(⊨tot​{p}S{q})(σ⊨{p}S{q})(σ⊨tot​{p}S{q})(σ⊨{p}S{q})​⇔⇔⇔⇔⇔​(∀σ, σ⊨{p}S{q})(∀σ, σ⊨tot​{p}S{q})(σ⊨p→(M(c, σ)−⊥d​)⊨q)(σ⊨p→M(c, σ)⊨q)(σ⊨tot​{p}S{q}∨M(c, σ)=⊥d​)​

如下是一些不合法的霍尔三元组的示例：

[true]while (x≠10) do (x:=x+1)[x=10][x=1]while true do skip[x=2][x=1]while true do skip[x=1]\begin{gathered} [true] & \text{while } (x \ne 10) \text{ do } (x := x + 1) & [x = 10] \\ [x = 1] & \text{while } true \text{ do } skip & [x = 2] \\ [x = 1] & \text{while } true \text{ do } skip & [x = 1] \end{gathered}[true][x=1][x=1]​while (x​=10) do (x:=x+1)while true do skipwhile true do skip​[x=10][x=2][x=1]​

另外一些简单的特殊示例如下：

  • ⊨{true}S{q}\vDash \{true\}\ S\ \{q\}⊨{true} S {q} ：如果 ccc 能终止，则 ccc 执行后 qqq 成立。

  • ⊨tot{true}c{q}\vDash_{tot} \{true\}\ c\ \{q\}⊨tot​{true} c {q} ：ccc 总是能终止且到达满足 qqq 的状态。

  • ⊨{p}c{true}\vDash \{p\}\ c\ \{true\}⊨{p} c {true} ：该式对任意的 ppp 和 ccc 恒成立。

  • ⊨tot{p}c{true}\vDash_{tot} \{p\}\ c\ \{true\}⊨tot​{p} c {true} ：如果初始时 ppp 成立，则 ccc 能终止。

# Tips

使用霍尔逻辑为程序设计 specification 并不容易。例如，假设我们希望形式化地描述“执行程序 SSS 能将 yyy 赋值为 max⁡(x,y)\max(x,\ y)max(x, y)”，不熟练者可能会写出如下的霍尔三元组：

{true}S{y=max⁡(x,y)} \{true\} \enspace S \enspace \{y = \max(x,\ y)\} {true}S{y=max(x, y)}

然而这样的后置条件会将很多事实上并不满足要求的程序证明为正确的，例如：

  • if (x≥y) then x:=y else skip\text{if } (x \ge y) \text{ then } x := y \text{ else } skipif (x≥y) then x:=y else skip

  • y:=xy := xy:=x

  • x:=0;y:=0x := 0;\ y := 0x:=0; y:=0

事实上，后置条件 q≡(y=max⁡(x,y))q \equiv (y = \max(x,\ y))q≡(y=max(x, y)) 描述的是“在执行程序 SSS 之后 yyy 的值等于 max⁡(x,y)\max(x,\ y)max(x, y)”。如下为一种能够满足要求的形式化描述：

{x=x0∧y=y0}c{y=max⁡(x0,y0)} \{x = x_0 \land y = y_0\} \enspace c \enspace \{y = \max(x_0,\ y_0)\} {x=x0​∧y=y0​}c{y=max(x0​, y0​)}

由此可见，使用霍尔逻辑进行程序验证是 non-trivial 的任务，形式化系统本身并不能保证程序验证的正确性。

# 终止性规则

我们知道，完全正确性是终止性与部分正确性的结合，而除去运行时错误，只有循环语句可能会导致程序不能正常终止。因此只有循环的推理规则是不同的，这也是将完全正确性和部分正确性区分开来的核心规则。

事实上，完全正确性和部分正确性的关联也可以表示为推理规则：

{p}S{q},[p]S[true][p]S[q][p]S[q]{p}S{q}[p]S[q][p]S[true] \frac{\{p\}S\{q\},\ [p]S[true]}{[p]S[q]} \qquad \frac{[p]S[q]}{\{p\}S\{q\}} \qquad \frac{[p]S[q]}{[p]S[true]} [p]S[q]{p}S{q}, [p]S[true]​{p}S{q}[p]S[q]​[p]S[true][p]S[q]​

此外，令 NW(S)NW(S)NW(S) 表示 SSS 中不含 while 语句，则还有下列规则：

{p}S{q},NW(S)[p]S[q] \frac{\{p\}S\{q\},\ NW(S)}{[p]S[q]} [p]S[q]{p}S{q}, NW(S)​

# 循环不变式

循环不变式 (loop invariant) 是满足如下条件的逻辑表达式：

  • 在开始执行循环之前该式为真；

  • 如果在某次迭代之前该式为真，那么在下次迭代之前该式仍然为真；

  • 在循环终止时该式为真，并且该式应该告诉我们一些有用的、能帮助我们理解程序语义的信息。

对于某个循环语句，可能存在多种不同的循环不变式。

循环不变式是用于程序测试和验证的概念。

# 部分正确性的推理规则

while 语句的描述部分正确性的推理规则如下所示，其中 iii 是循环不变式：

(WHP){i∧b}S{i}{i} while b do c{i∧¬b} (WHP)\ \frac{\{i \land b\}\ S\ \{i\}}{\{i\}\ \text{while } b \text{ do } c\ \{i \land \neg b\}} (WHP) {i} while b do c {i∧¬b}{i∧b} S {i}​

为什么是 {p∧b}S{p}\{p \land b\}\ S\ \{p\}{p∧b} S {p} ？可能执行多次 ccc 才破坏 bbb ？

while 语句的描述完全正确性的推理规则如下所示，其中 iii 是循环不变式，x0x_0x0​ 是逻辑变量（即不会出现在 i,b,e,Si, b, e, Si,b,e,S 中），eee 是循环计数器：

(WHT)[i∧b∧(e=x0)]S[i∧(e<x0)],(i∧b)⇒(e≥0)[i] while b do c[i∧¬b] (WHT)\ \frac{[i \land b \land (e = x_0)]\ S\ [i \land (e < x_0)],\ (i \land b) \rArr (e \ge 0)}{[i]\ \text{while } b \text{ do } c\ [i \land \neg b]} (WHT) [i] while b do c [i∧¬b][i∧b∧(e=x0​)] S [i∧(e<x0​)], (i∧b)⇒(e≥0)​

具体地讲：

  • 霍尔三元组 [i∧b∧(e=x0)]S[i∧(e<x0)][i \land b \land (e = x_0)]\ S\ [i \land (e < x_0)][i∧b∧(e=x0​)] S [i∧(e<x0​)] 描述了循环计数器 eee 随迭代而递进的事实。

  • 谓词逻辑 (i∧b)⇒(e≥0)(i \land b) \rArr (e \ge 0)(i∧b)⇒(e≥0) 描述了循环在计数器超限后终止的事实。
