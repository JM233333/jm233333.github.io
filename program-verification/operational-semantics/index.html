Program Verification 02 - Operational Semantics

# 前置知识

在阅读本文档前，请确保你已掌握布尔逻辑的基础知识，详见 Program Verification 知识板块中的 01 - Basic Logic 文档。

  • Program Verification 01 - Basic Logic

在阅读本文档前，请确保你已掌握 BNF 范式。详情请自学编译原理，暂无相关文档，敬请谅解。

# 前言

操作语义 (operational semantics) 是编程语言执行的数学模型，或者说是程序在抽象机器上的执行的释义。操作语义是编程语言的形式化规范，可用于精确理解编程语言中表达式的含义，或用于对编程语言进行严格的形式化验证。

  • 操作语义很像是语言的解释器，不同的是，前者是基于数学定义的，后者是基于编程语言本身定义的。

结构化的操作语义 (structural operational semantics, SOS) 提供了一个能为编程语言和形式化语言提供操作语义的框架，其直观的吸引力和灵活性使其得到广泛应用，包括并发进程的语义研究、软件静态分析、编译器正确性证明等。

  • SOS 可以分为 small-step 和 big-step 两种，前者描述执行中的每个步骤，后者描述执行的总体结果，详见下文。

本文将基于 IMP 介绍 SOS 的相关概念。

# IMP

# 基本概念

IMP 是一种简单的、non-procedural 的 命令式语言 (imperative language) ，它只有一个条件命令和一个 while 命令，主要用于描述基本算术表达式。

IMP 的基本语法定义如下：

  • 整数域 int∈Int=defZint \in Int\overset{def}{=} \bold{Z}int∈Int=defZ

  • 布尔域 bool∈Bool=def{true,false}bool \in Bool \overset{def}{=} \{true,\ false\}bool∈Bool=def{true, false}

  • 值域 v∈Value=defInt∪Boolv \in Value \overset{def}{=} Int \cup Boolv∈Value=defInt∪Bool

  • 变量标识符 id∈Var=def{x,y,...}id \in Var \overset{def}{=} \{x,\ y,\ ...\}id∈Var=def{x, y, ...}

  • 算术表达式 a∈AExpa \in AExpa∈AExp

  • 布尔表达式 b∈BExpb \in BExpb∈BExp

  • 命令 c∈Comc \in Comc∈Com

# IMP 形式化规则

IMP 对表达式的形式化规则如下：

# Expression
e ::= a | b

# Arithmetic Expression
a ::= int
    | id
    | a + a | a - a | a * a | ...

# Boolean Expression
b ::= bool
    | b > b | b < b | b == b | ...
    | b and b | b or b | not b | ...

IMP 对命令的形式化规则如下：

# Command
c ::= skip
    | id := a
    | c; c
    | if b then c else c
    | while b do c

# IMP 状态

在 IMP 中，状态是从变量到值域的映射，即 σ:Var→Value\sigma : Var \to Valueσ:Var→Value 。

我们将 非终结配置 (non-terminal configuration) 形式化为一个二元组 ⟨e,σ⟩\langle e,\ \sigma \rangle⟨e, σ⟩ ，表示（算术或布尔）表达式 eee 正在状态 σ\sigmaσ 下等待演算。类似地，⟨c,σ⟩\langle c,\ \sigma \rangle⟨c, σ⟩ 表示命令 ccc 正在状态 σ\sigmaσ 下等待执行。

# Small-Step SOS

# 基本概念

small-step SOS 又称为 transitional semantics ，它描述了执行中的每个步骤。

  • 可以将 small-step SOS 视为一个能够 step-by-step 地执行给定代码的机器。

  • 小步语义可以清楚地为复杂特性建模，比如并发、divergence 和运行时错误。

small-step SOS 的形式化定义包括三个部分：

  • configuration ：对 IMP non-terminal configuration 的简记。

  • transition ：一组等待推导的 configuration 。

  • rule ：描述了基于 transition 推导 configuration 的规则，表示为如下形式，其中 前提 (premise) 是零至多个 transition 或 equation ，结论 (conclusion) 是一个 transition ：

premiseconclusion\frac{premise}{conclusion}conclusionpremise​

非形式化地讲，rule 的含义是在 premise 成立的情况下可以将 conclusion 式左的 configuration 推导到式右的 configuration 。

注意，形式化定义不是唯一的，原则上只需形式语义规范、语法合适、逻辑正确即可。本文仅提供了一种我个人最喜欢的定义，读者仅在参考资料中就能找到好几种不同的形式化定义，但本质上都是等价的。

# 表达式演算规则

在表达式演算中，transition 表现为如下两种形式之一，其中 v∈{int,bool}v \in \{int,\ bool\}v∈{int, bool} ：

  • ⟨e,σ⟩→⟨e′,σ⟩\langle e,\ \sigma \rangle \to \langle e',\ \sigma \rangle⟨e, σ⟩→⟨e′, σ⟩

  • ⟨e,σ⟩→v\langle e,\ \sigma \rangle \to v⟨e, σ⟩→v

我们以算术表达式的加法为例，定义 ⟨a,σ⟩→⟨a′,σ⟩\langle a,\ \sigma \rangle \to \langle a',\ \sigma \rangle⟨a, σ⟩→⟨a′, σ⟩ 的表达式演算规则如下：

σ(a)=int⟨a,σ⟩→⟨int,σ⟩(lookup)⟨a1,σ⟩→⟨a1′,σ⟩⟨a1+a2,σ⟩→⟨a1′+a2,σ⟩(arg1)⟨a2,σ⟩→⟨a2′,σ⟩⟨int1+a2,σ⟩→⟨int1+a2′,σ⟩(arg2)int1+int2=int⟨int1+int2,σ⟩→⟨int,σ⟩(add)\begin{gathered} \frac{\sigma(a) = int}{\langle a,\ \sigma \rangle \to \langle int,\ \sigma \rangle} & (look\ up) \\\\ \frac{\langle a_1,\ \sigma \rangle \to \langle a_1',\ \sigma \rangle}{\langle a_1 + a_2,\ \sigma \rangle \to \langle a_1' + a_2,\ \sigma \rangle} & (arg\ 1) \\\\ \frac{\langle a_2,\ \sigma \rangle \to \langle a_2',\ \sigma \rangle}{\langle int_1 + a_2,\ \sigma \rangle \to \langle int_1 + a_2',\ \sigma \rangle} & (arg\ 2) \\\\ \frac{int_1 + int_2 = int}{\langle int_1 + int_2,\ \sigma \rangle \to \langle int,\ \sigma \rangle} & (add) \end{gathered}⟨a, σ⟩→⟨int, σ⟩σ(a)=int​⟨a1​+a2​, σ⟩→⟨a1′​+a2​, σ⟩⟨a1​, σ⟩→⟨a1′​, σ⟩​⟨int1​+a2​, σ⟩→⟨int1​+a2′​, σ⟩⟨a2​, σ⟩→⟨a2′​, σ⟩​⟨int1​+int2​, σ⟩→⟨int, σ⟩int1​+int2​=int​​(look up)(arg 1)(arg 2)(add)​

这里给出一个简单的实例来帮助理解。

设 σ={x⇝5,y⇝12}\sigma = \{x \rightsquigarrow 5,\ y \rightsquigarrow 12\}σ={x⇝5, y⇝12} ，则对于 ⟨x+y,σ⟩\langle x + y,\ \sigma \rangle⟨x+y, σ⟩ 有：

⟨x+y,σ⟩→⟨1+y,σ⟩→⟨1+2,σ⟩→⟨3,σ⟩\langle x + y,\ \sigma \rangle \to \langle 1 + y,\ \sigma \rangle \to \langle 1 + 2,\ \sigma \rangle \to \langle 3,\ \sigma \rangle⟨x+y, σ⟩→⟨1+y, σ⟩→⟨1+2, σ⟩→⟨3, σ⟩

对于算术表达式的其它运算符，以及布尔表达式的运算，均有类似规则，不予赘述。不过，一元布尔运算符稍有不同：

⟨b,σ⟩→⟨b′,σ⟩⟨!b,σ⟩→⟨!b′,σ⟩ \frac{\langle b,\ \sigma \rangle \to \langle b',\ \sigma \rangle}{\langle !b,\ \sigma \rangle \to \langle !b',\ \sigma \rangle} ⟨!b, σ⟩→⟨!b′, σ⟩⟨b, σ⟩→⟨b′, σ⟩​

事实上，我们可以将所有二元（算术或布尔）运算符的 arg1arg\ 1arg 1 和 arg2arg\ 2arg 2 规则结合为更简洁的一条规则：

⟨e,σ⟩→⟨e′,σ⟩⟨E[e],σ⟩→⟨E[e′],σ⟩ \frac{\langle e,\ \sigma \rangle \to \langle e',\ \sigma \rangle}{\langle E[e],\ \sigma \rangle \to \langle E[e'],\ \sigma \rangle} ⟨E[e], σ⟩→⟨E[e′], σ⟩⟨e, σ⟩→⟨e′, σ⟩​

其中 E[]E[]E[] 定义为：

E[] ::= [] + e | int + []
      | [] - e | int - []
      | ...

对于更复杂的表达式（例如多元的、复合的），可以递归或归约地进行演算，并构造出对应的 派生树 (derivation tree) 。

# 命令执行规则

在命令执行中，transition 表现为如下两种形式之一：

  • ⟨c,σ⟩→⟨c′,σ′⟩\langle c,\ \sigma \rangle \to \langle c',\ \sigma' \rangle⟨c, σ⟩→⟨c′, σ′⟩

  • ⟨c,σ⟩→σ′\langle c,\ \sigma \rangle \to \sigma'⟨c, σ⟩→σ′

注意 σ\sigmaσ 和 σ′\sigma'σ′ 。与表达式的演算不同，命令的执行可能会改变状态。

对于状态 σ\sigmaσ ，定义 σ[y/x]\sigma[y/x]σ[y/x] 如下：

σ[y/x](k)={y(k=x)σ(y)(k≠x)\sigma[y/x](k) = \begin{cases} y & (k = x) \\ \sigma(y) &(k \ne x) \end{cases}σ[y/x](k)={yσ(y)​(k=x)(k​=x)​

定义 skip 命令的执行规则如下：

⟨skip,σ⟩→σ \frac{}{\langle skip,\ \sigma \rangle \to \sigma} ⟨skip, σ⟩→σ​

定义赋值命令的执行规则如下：

⟨x:=int,σ⟩→σ{x⇝int} \frac{}{\langle x := int,\ \sigma \rangle \to \sigma\{x \rightsquigarrow int\}} ⟨x:=int, σ⟩→σ{x⇝int}​
⟨e,σ⟩→⟨e′,σ⟩⟨x:=e,σ⟩→⟨x:=e′,σ⟩ \frac{\langle e,\ \sigma \rangle \to \langle e',\ \sigma \rangle}{\langle x := e,\ \sigma \rangle \to \langle x := e',\ \sigma \rangle} ⟨x:=e, σ⟩→⟨x:=e′, σ⟩⟨e, σ⟩→⟨e′, σ⟩​

定义多条命令展开的执行规则如下：

⟨c0,σ⟩→⟨c0′,σ′⟩⟨c0;c1,σ⟩→⟨c0′;c1,σ⟩ \frac{\langle c_0,\ \sigma \rangle \to \langle c_0',\ \sigma' \rangle}{\langle c_0 ; c_1,\ \sigma \rangle \to \langle c_0' ; c_1,\ \sigma \rangle} ⟨c0​;c1​, σ⟩→⟨c0′​;c1​, σ⟩⟨c0​, σ⟩→⟨c0′​, σ′⟩​
⟨c,σ⟩→σ′⟨c0;c1,σ⟩→⟨c1,σ′⟩ \frac{\langle c,\ \sigma \rangle \to \sigma'}{\langle c_0 ; c_1,\ \sigma \rangle \to \langle c_1,\ \sigma' \rangle} ⟨c0​;c1​, σ⟩→⟨c1​, σ′⟩⟨c, σ⟩→σ′​

定义 if 命令的执行规则如下：

⟨b,σ⟩→true⟨if b then c0 else c1,σ⟩→⟨c0,σ⟩ \frac{\langle b,\ \sigma \rangle \to true}{\langle \text{if } b \text{ then } c_0 \text{ else } c_1,\ \sigma \rangle \to \langle c_0,\ \sigma \rangle} ⟨if b then c0​ else c1​, σ⟩→⟨c0​, σ⟩⟨b, σ⟩→true​
⟨b,σ⟩→false⟨if b then c0 else c1,σ⟩→⟨c1,σ⟩ \frac{\langle b,\ \sigma \rangle \to false}{\langle \text{if } b \text{ then } c_0 \text{ else } c_1,\ \sigma \rangle \to \langle c_1,\ \sigma \rangle} ⟨if b then c0​ else c1​, σ⟩→⟨c1​, σ⟩⟨b, σ⟩→false​
⟨b,σ⟩→⟨b′,σ⟩⟨if b then c0 else c1,σ⟩→⟨if b′ then c0 else c1,σ⟩ \frac{\langle b,\ \sigma \rangle \to \langle b',\ \sigma \rangle}{\langle \text{if } b \text{ then } c_0 \text{ else } c_1,\ \sigma \rangle \to \langle \text{if } b' \text{ then } c_0 \text{ else } c_1,\ \sigma \rangle} ⟨if b then c0​ else c1​, σ⟩→⟨if b′ then c0​ else c1​, σ⟩⟨b, σ⟩→⟨b′, σ⟩​

定义 while 命令的执行规则稍微有些特别。如果我们简单地仿照 if 命令进行形式化，将无法处理 ⟨b,σ⟩→true\langle b,\ \sigma \rangle \to true⟨b, σ⟩→true 的情况：

⟨b,σ⟩→true⟨while b do c,σ⟩→? \frac{\langle b,\ \sigma \rangle \to true}{\langle \text{while } b \text{ do } c,\ \sigma \rangle \to\ ?} ⟨while b do c, σ⟩→ ?⟨b, σ⟩→true​

事实上，我们应当在每次循环中对 while 命令执行 transition rule ，而不是在第一次 transition 时就将 bbb 替换掉。

我们可以将循环展开：

⟨b,σ⟩→true⟨while b do c,σ⟩→⟨c;while b do c,σ⟩ \frac{\langle b,\ \sigma \rangle \to true}{\langle \text{while } b \text{ do } c,\ \sigma \rangle \to \langle c; \text{while } b \text{ do } c,\ \sigma \rangle} ⟨while b do c, σ⟩→⟨c;while b do c, σ⟩⟨b, σ⟩→true​
⟨b,σ⟩→false⟨while b do c,σ⟩→σ \frac{\langle b,\ \sigma \rangle \to false}{\langle \text{while } b \text{ do } c,\ \sigma \rangle \to \sigma} ⟨while b do c, σ⟩→σ⟨b, σ⟩→false​

也可以借助 if 命令，这仅需一条规则就能完成形式化：

⟨while b do c,σ⟩→⟨if b then (c;while b do c) else skip,σ⟩ \frac{}{\langle \text{while } b \text{ do } c,\ \sigma \rangle \to \langle \text{if } b \text{ then } (c; \text{while } b \text{ do } c) \text{ else } skip,\ \sigma \rangle} ⟨while b do c, σ⟩→⟨if b then (c;while b do c) else skip, σ⟩​

# 多步执行

定义 →∗\to^*→∗ 为 →\to→ 的自反传递闭包，有如下归约表示：

⟨c,σ⟩→∗⟨c,σ⟩ \frac{}{\langle c,\ \sigma \rangle \to^* \langle c,\ \sigma \rangle} ⟨c, σ⟩→∗⟨c, σ⟩​
⟨c,σ⟩→⟨c,σ⟩,⟨c′,σ′⟩→∗⟨c′′,σ′′⟩⟨c,σ⟩→∗⟨c′′,σ′′⟩ \frac{\langle c,\ \sigma \rangle \to \langle c,\ \sigma \rangle,\enspace \langle c',\ \sigma' \rangle \to^* \langle c'',\ \sigma'' \rangle}{\langle c,\ \sigma \rangle \to^* \langle c'',\ \sigma'' \rangle} ⟨c, σ⟩→∗⟨c′′, σ′′⟩⟨c, σ⟩→⟨c, σ⟩,⟨c′, σ′⟩→∗⟨c′′, σ′′⟩​

亦可用 nnn-step 的方式表示：

⟨c,σ⟩→0⟨c,σ⟩ \frac{}{\langle c,\ \sigma \rangle \to^0 \langle c,\ \sigma \rangle} ⟨c, σ⟩→0⟨c, σ⟩​
⟨c,σ⟩→⟨c,σ⟩,⟨c′,σ′⟩→n⟨c′′,σ′′⟩⟨c,σ⟩→n+1⟨c′′,σ′′⟩ \frac{\langle c,\ \sigma \rangle \to \langle c,\ \sigma \rangle,\enspace \langle c',\ \sigma' \rangle \to^n \langle c'',\ \sigma'' \rangle}{\langle c,\ \sigma \rangle \to^{n+1} \langle c'',\ \sigma'' \rangle} ⟨c, σ⟩→n+1⟨c′′, σ′′⟩⟨c, σ⟩→⟨c, σ⟩,⟨c′, σ′⟩→n⟨c′′, σ′′⟩​

# 有关 transition 的性质

⟨c,σ⟩→⟨c′,σ′⟩∧⟨c,σ⟩→⟨c′′,σ′′⟩⇒⟨c′,σ′⟩=⟨c′′,σ′′⟩\langle c,\ \sigma \rangle \to \langle c',\ \sigma' \rangle \land \langle c,\ \sigma \rangle \to \langle c'',\ \sigma'' \rangle\ \rArr\ \langle c',\ \sigma' \rangle = \langle c'',\ \sigma'' \rangle⟨c, σ⟩→⟨c′, σ′⟩∧⟨c, σ⟩→⟨c′′, σ′′⟩ ⇒ ⟨c′, σ′⟩=⟨c′′, σ′′⟩

# Small-Step Contextual SOS

small-step contextual SOS 又称为 reduction semantics ，是另一种形式的 small-step SOS 。

small-step contextual SOS 的形式化定义包括三个部分：

  • redex ：可以通过一步原子步骤进行归约的句法表达式或命令，记为 rrr 。

  • evaluation context, or reduction context ：“挖了洞”的表达式或命令，记为 ε\varepsilonε 。

  • local reduction rule ：基于 rrr 构建的局部归约规则，指示了如何为每个 redex 执行归约。

  • global reduction rule ：基于 rrr 和 ε\varepsilonε 构建的全局归约规则，指示了下一步要演算的 redex 。

# Redex
r ::= x
    | int + int | int - int | ...
    | x := int
    | skip; c
    | if true then c else c | if false then c else c
    | while b do c
    | ...

# Evaluation Context
ε ::= []
    | ε + e | ε - e | ...
    | int + ε | int - ε | ...
    | x := ε
    | ε; c
    | if ε then c else c
    | ...

small-step contextual SOS 的核心 idea 是：

  • 将当前项（表达式或指令）分解为 redex 和 evaluation context（事实上是原项中除 redex 外的部分）；

  • 根据局部归约规则将 redex 归约为更简单的形式；

  • 将简化后的项放回到 evaluation context 中。

这可以形式化为如下的 small-step 规则，称为全局归约规则：

⟨r,σ⟩→⟨t,σ′⟩⟨ε[r],σ⟩→⟨ε[t],σ′⟩ \frac{\langle r,\ \sigma \rangle \to \langle t,\ \sigma' \rangle}{\langle \varepsilon[r],\ \sigma \rangle \to \langle \varepsilon[t],\ \sigma' \rangle} ⟨ε[r], σ⟩→⟨ε[t], σ′⟩⟨r, σ⟩→⟨t, σ′⟩​

evaluation context 有且仅有一个“洞”，“洞”的位置唯一标识了下一步要演算的 redex 。我们用 ε[r]\varepsilon[r]ε[r] 表示将 ε\varepsilonε 中的洞替换为 redex rrr 所得的表达式。

  • 例如对于 ε=(x:=1+[]),r=2+8\varepsilon = (x := 1 + []),\ r = 2 + 8ε=(x:=1+[]), r=2+8 有 ε[r]=(x:=1+(2+8))\varepsilon[r] = (x := 1 + (2 + 8))ε[r]=(x:=1+(2+8)) 。

small-step contextual SOS 以 递归分解 (recursive decomposition) 的形式进行演算，每一步演算的 solution 都是唯一的。其分解定理可描述为：

  • 若表达式 eee 还未转换为 intintint ，则存在唯一的 ε\varepsilonε 和 rrr ，使得 e=ε[r]e = \varepsilon[r]e=ε[r] 。

  • 若命令 ccc 还未转换为 skipskipskip ，则存在唯一的 ε\varepsilonε 和 rrr ，使得 c=ε[r]c = \varepsilon[r]c=ε[r] 。

下面给出几个分解的示例：

对于 ε=(e=e1+e2)\varepsilon = (e = e_1 + e_2)ε=(e=e1​+e2​) 及其分解 ε[r]\varepsilon[r]ε[r] ：

  • 若 e1e_1e1​ 还未转换为 int1int_1int1​ ，则 e1=ε1[r],ε=ε1+e2e_1 = \varepsilon_1[r],\ \varepsilon = \varepsilon_1 + e_2e1​=ε1​[r], ε=ε1​+e2​ ；

# Big-Step SOS

# 基本概念

big-step SOS 又称为 natural semantics 或 relational semantics ，它描述了执行的总体结果。

  • 可以将 big-step SOS 视为一个递归解释器，能对于给定的代码和状态给出演算结果。

  • 在某些情况下，小步语义的细致性是不必要的，大步语义将小步语义中那些指定求值顺序的“枯燥”规则折叠起来，使规则变得更少、证明变得更快。

  • 然而在大步语义中，所有没有 final configuration 的程序（例如无限循环、get stuck 等）看起来都一样，因此我们有时无法用大步语义证明与此相关的内容。

big-step SOS 的形式化定义同样由 configuration 、transition 和 rule 三个部分组成。

premiseconclusion\frac{premise}{conclusion}conclusionpremise​

不同的是，大步语义的一步 transition 直接描述了演算的结果，而不考虑演算过程的中间状态：

  • ⟨e,σ⟩⇓int\langle e,\ \sigma \rangle \dArr int⟨e, σ⟩⇓int

  • ⟨c,σ⟩⇓σ′\langle c,\ \sigma \rangle \dArr \sigma'⟨c, σ⟩⇓σ′

# 伪状态

伪状态 (pseudo-state) 指的是不能成功终止的程序状态，用 ⊥\bot⊥ 表示，并且有 ⊥∈{⊥d,⊥e}\bot \in \{\bot_d,\ \bot_e\}⊥∈{⊥d​, ⊥e​} 。

若 ⟨S,σ⟩⇓⊥d\langle S,\ \sigma \rangle \dArr \bot_d⟨S, σ⟩⇓⊥d​ ，则称从状态 σ\sigmaσ 出发执行程序片段 SSS 导致 发散 (diverge) ，其中 ⊥d\bot_d⊥d​ 是表示程序因始终运行而不能终止（例如死循环）的伪状态。

若 ⟨S,σ⟩⇓⊥e\langle S,\ \sigma \rangle \dArr \bot_e⟨S, σ⟩⇓⊥e​ ，则称从状态 σ\sigmaσ 出发执行程序片段 SSS 时出现 运行时错误 (runtime error) ，其中 ⊥e\bot_e⊥e​ 是表示程序因出现错误而无法正常终止的伪状态。
