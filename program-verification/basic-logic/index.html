Program Verification 01 - Basic Logic

# 前置知识

在阅读本文档前，请先阅读程序验证的整体介绍文档，详见 Program Verification 知识板块中的 00 - Introduction 文档。

# 命题逻辑

命题变量 (proposition variables) 是取值可以为 真 (true) 或 假 (false) 的变量，而 命题逻辑 (propositional logic) 就是有关命题变量的逻辑规则。

# 命题运算符

下列术语中出现的运算符统称为 命题运算符 (propositional operators) 。

p,q,...p,\ q,\ ...p, q, ... 表示命题或命题变量，T/FT/FT/F 或 true/falsetrue/falsetrue/false 表示真和假。

p∧qp \land qp∧q 表示 ppp 和 qqq 的 合取 (conjunction) 或 逻辑与 (logic and) 。

p∨qp \lor qp∨q 表示 ppp 和 qqq 的 析取 (disjunction) 或 逻辑或 (logic or) 。

p→qp \to qp→q 表示 蕴涵 (implication) 或 条件 (conditional) ，其中 ppp 是 前提 (antecedent) 或 假设 (hypothesis) ，qqq 是 结果 (consequent) 或 结论 (conclusion) 。

  • ppp 蕴涵 qqq 的等价表示包括：若 ppp 则 qqq ，ppp 是 qqq 的充分条件，qqq 是 ppp 的必要条件，等等。

  • p→q⇔¬p∨qp \to q \lrArr \neg p \lor qp→q⇔¬p∨q 。

p↔qp \lrarr qp↔q 表示 等价 (equivalance) 或 双向条件 (biconditional) 。

  • 注意命题逻辑中的等价与数学中的等价不同，例如不具备传递性，p↔q,q↔rp \lrarr q,\ q \lrarr rp↔q, q↔r 不能推导出 p↔rp \lrarr rp↔r 。

  • p↔q⇔(p→q)∧(q→p)p \lrarr q \lrArr (p \to q) \land (q \to p)p↔q⇔(p→q)∧(q→p) 。

# 运算符属性

命题运算符按 优先级 (precedence) 从高到低依次为 ¬,∧,∨,→,↔\neg,\ \land,\ \lor,\ \to,\ \lrarr¬, ∧, ∨, →, ↔ 。

  • 有些地方认为 →\to→ 和 ↔\lrarr↔ 的优先级相同，因此将 p↔q→rp \lrarr q \to rp↔q→r 写作 p↔(q→r)p \lrarr (q \to r)p↔(q→r) 是更合适的。

命题运算符 ∧\land∧ 和 ∨\lor∨ 具有结合律，并且规定它们是 左结合 (left associative) 的。

命题运算符 →\to→ 和 ↔\lrarr↔ 不满足结合律，并且规定它们是 右结合 (right associative) 的。

# 语义等价和句法等价

# 基本概念

语义等价 (semantic equality) 是意义或结果上的等价，用 ⇔\lrArr⇔ 表示，例如 (p∧q)∨r⇔(p∨r)∧(q∨r)(p \land q) \lor r \lrArr (p \lor r) \land (q \lor r)(p∧q)∨r⇔(p∨r)∧(q∨r) 。

  • 语义等价类似于我们在数学算术中使用的 等于 (equal) ，例如 2+2=42 + 2 = 42+2=4 在形式化验证的上下文中表示为 2+2⇔42 + 2 \lrArr 42+2⇔4 。

  • 语义等价意味着 当且仅当 (if and only if) ，也经常写作 p iff qp \text{ iff } qp iff q 。

语义等价关系 ⇔\lrArr⇔ 和双向条件等价关系 ↔\lrarr↔ 有本质上的不同：

  • 前者是语义上的符号，后者是句法上的符号。

  • 前者具有传递性，后者不具有传递性。

句法等价 (syntactic equality) 是移除冗余括号后的结构化文本的等价，用 ≡\equiv≡ 表示。语法等价与句法等价是本质不同的，例如 2+2≢42 + 2 \not\equiv 42+2​≡4 。

句法等价考虑如下三种冗余：

  • 优先级，例如 (p∧q)∨r≡p∧q∨r(p \land q) \lor r \equiv p \land q \lor r(p∧q)∨r≡p∧q∨r 。

  • 结合性，例如 a−b−c≡(a−b)−ca - b - c \equiv (a - b) - ca−b−c≡(a−b)−c ，p→q→r≡p→(q→r)p \to q \to r \equiv p \to (q \to r)p→q→r≡p→(q→r) 。

  • 结合律，例如 (p∧q)∧r≡p∧(q∧r)(p \land q) \land r \equiv p \land (q \land r)(p∧q)∧r≡p∧(q∧r) 。

句法等价不考虑交换律，例如 p∧q≢q∧rp \land q \not\equiv q \land rp∧q​≡q∧r ，这是出于判定简便的考虑。如果考虑交换律，要判定诸如两个 nnn 个数相加的表达式是否句法等价将变得复杂，不考虑交换律使我们可以直接按文本顺序判定。

parenthesization 是为便于描述而引入的形式化概念：

  • minimal parenthesization 指的是在保证句法等价的前提下使括号尽可能少，例如 (p∧q)∨r(p \land q) \lor r(p∧q)∨r 的 minimal parenthesization 为 p∧q∨rp \land q \lor rp∧q∨r 。

  • full parenthesization 指的是在保证句法等价的前提下为每个二元表达式加上括号，例如 p∨q∧rp \lor q \land rp∨q∧r 的 full parenthesization 为 (p∨q)∧r(p \lor q) \land r(p∨q)∧r 。

对于具有结合律的左结合运算符，同样需要加上括号，例如 p∧q∧rp \land q \land rp∧q∧r 的 full parenthesization 为 (p∧q)∧r(p \land q) \land r(p∧q)∧r 。

是否在整个表达式周围加上括号是可选的，即 ((p∧q)∧r)((p \land q) \land r)((p∧q)∧r) 可以简写为 (p∧q)∧r(p \land q) \land r(p∧q)∧r 。

# 深入理解

通常情况下，我们更希望知道两个程序在语义上是否等价，然而在使用谓词时，判定语义等价可能是 impractical 的、甚至是不可能的。相比之下，句法等价要容易判定得多，我们可以使用句法等价来近似语义等价。

具体地讲，如果我们恰当地定义句法等价的判定规则，就可以保证句法等价蕴涵语义等价，即 ∀p,q,(p≡q)⇒(p⇔q)\forall p,\ q,\ (p \equiv q) \rArr (p \lrArr q)∀p, q, (p≡q)⇒(p⇔q) 。同样地，有逆否命题 ∀p,q,(p⇎q)⇒(p≢q)\forall p,\ q,\ (p \nLeftrightarrow q) \rArr (p \not\equiv q)∀p, q, (p⇎q)⇒(p​≡q) 成立，即语义不等价意味着句法不等价。

这种近似是很不精确的。一方面，语义等价的程序未必是句法等价的，例如 2+2≢42 + 2 \not\equiv 42+2​≡4 ；另一方面，句法不等价的程序经常是语义等价的，例如 p∧q⇔q∧pp \land q \lrArr q \land pp∧q⇔q∧p 。

现在我们考虑一个棘手的问题：如果命题（或表达式）同时涉及了两个优先级相同且都具有结合律的运算符，例如 x∗(y/z)x * (y / z)x∗(y/z) 和 (x/y)∗z(x / y) * z(x/y)∗z ，以及 x+(y−z)x + (y - z)x+(y−z) 和 (x+y)−z(x + y) - z(x+y)−z ，它们是否在句法上等价？

  • 我们先考虑表达式 x∗(y/z)x * (y / z)x∗(y/z) 和 (x/y)∗z(x / y) * z(x/y)∗z 在语义上是否等价。显然，这取决于 x,y,zx,\ y,\ zx, y, z 的类型：如果是整数运算，则二者语义不等价；如果是高精度实数运算，则二者语义等价。

  • 在这种情况下，如果确定表达式在句法上是否等价还需要考虑操作数的类型，将会使判定变得复杂，违背了我们进行近似的初衷，因此将其视为句法不等价是更合适的。

# 状态

# 基本概念

状态 (state) 是一系列命题变量与其真值的 绑定 (binding) ，记为 σ\sigmaσ 或 τ\tauτ 。若不含任何绑定，则称为空状态，记为 ∅\empty∅ 。

  • well-formed state ：状态中出现的每个命题变量只有一个绑定，例如 σ={p=T,q=F}\sigma = \{p = T,\ q = F\}σ={p=T, q=F} 。

  • ill-formed state ：状态中有某个命题变量存在多个绑定，或存在非单变量的命题表达式，则为不正确的状态，例如 σ={p=T,q=F,p=T}\sigma = \{p = T,\ q = F,\ p = T\}σ={p=T, q=F, p=T} 或 σ={p∧q=T}\sigma = \{p \land q = T\}σ={p∧q=T} 。

用 σ(p)\sigma(p)σ(p) 表示状态 σ\sigmaσ 中对变量 xxx 绑定的取值，例如对于 σ={p=T,q=F}\sigma = \{p = T,\ q = F\}σ={p=T, q=F} 有 σ(p)=T,σ(q)=F\sigma(p) = T,\ \sigma(q) = Fσ(p)=T, σ(q)=F 。

  • 对于 ppp 为命题表达式的情况，可将 σ()\sigma()σ() 视为具有分配律的运算符进行拆解，例如 σ(p∧q)=σ(p)∧σ(q)\sigma(p \land q) = \sigma(p) \land \sigma(q)σ(p∧q)=σ(p)∧σ(q) 。

如果命题 ppp 在状态 σ\sigmaσ 下的演算结果为真，则称状态 σ\sigmaσ 满足 (satisfy) 命题 ppp ，或命题 ppp 在状态 σ\sigmaσ 下被满足，记为 σ⊨p\sigma \vDash pσ⊨p 。

下面是几个命题满足的例子：

  • {p=F}⊨¬p\{p = F\} \vDash \neg p{p=F}⊨¬p

  • {p=T,q=F}⊨p∨q\{p = T,\ q = F\} \vDash p \lor q{p=T, q=F}⊨p∨q

  • {p=T,q=F}⊭p∧q\{p = T,\ q = F\} \nvDash p \land q{p=T, q=F}⊭p∧q

  • ∅⊨T∧(F→T)\empty \vDash T \land (F \to T)∅⊨T∧(F→T)

当我们表述状态对命题的满足性时，状态中可以有未被使用的绑定，但不能有缺失的绑定，哪怕已有的绑定已经足以对命题给出演算结果。例如，我们可以说 {p=T,q=F}⊨p\{p = T,\ q = F\} \vDash p{p=T, q=F}⊨p ，但我们不能说 {p=T}⊨p∨q\{p = T\} \vDash p \lor q{p=T}⊨p∨q 。

  • 如果状态 σ\sigmaσ 至少包含命题 ppp 中出现的所有命题变量的绑定，则称状态 σ\sigmaσ 适用于 (proper for) 命题 ppp 。

除非特殊说明，当我们在讨论命题和状态时，总是在讨论 well-formed 的状态 proper for 命题的情况，不考虑 ill-formed 的状态和 improper for 命题的情况。

# 基本性质

状态对命题的满足性具有分配律，例如：

  • σ⊨¬p⇔¬(σ⊨p)\sigma \vDash \neg p \lrArr \neg (\sigma \vDash p)σ⊨¬p⇔¬(σ⊨p)

  • σ⊨p∧q⇔(σ⊨p)∧(σ⊨q)\sigma \vDash p \land q \lrArr (\sigma \vDash p) \land (\sigma \vDash q)σ⊨p∧q⇔(σ⊨p)∧(σ⊨q)

  • σ⊨p∨q⇔(σ⊨p)∨(σ⊨q)\sigma \vDash p \lor q \lrArr (\sigma \vDash p) \lor (\sigma \vDash q)σ⊨p∨q⇔(σ⊨p)∨(σ⊨q)

  • σ⊨p→q⇔(σ⊨p)→(σ⊨q)\sigma \vDash p \to q \lrArr (\sigma \vDash p) \to (\sigma \vDash q)σ⊨p→q⇔(σ⊨p)→(σ⊨q)

对于任意的状态 σ⊆τ\sigma \subseteq \tauσ⊆τ 和命题 ppp ，若 σ⊨p\sigma \vDash pσ⊨p ，则 τ⊨p\tau \vDash pτ⊨p 。

# 命题分类

设 ppp 是一个命题，则：

  • 若 ∀σ,σ⊨p\forall \sigma,\ \sigma \vDash p∀σ, σ⊨p ，则称 ppp 是 永真式 (tautology) 或 重言式 ，记为 ⊨p\vDash p⊨p 。

  • 若 ∀σ,σ⊨¬p\forall \sigma,\ \sigma \vDash \neg p∀σ, σ⊨¬p ，则称 ppp 是 矛盾式 (contradiction) 。

  • 若 ⊭p∧⊭¬p\nvDash p\ \land \nvDash \neg p⊭p ∧⊭¬p ，即 ∃σ,σ⊨p∧∃σ,σ⊨¬p\exists \sigma,\ \sigma \vDash p \land \exists \sigma,\ \sigma \vDash \neg p∃σ, σ⊨p∧∃σ, σ⊨¬p ，则称 ppp 是 contingency 。

对于任意一个命题 ppp ，其必为永真式、矛盾式或 contingency 三者之一。

可将语义等价 p⇔qp \lrArr qp⇔q 写成状态满足形式的永真式 ⊨p↔q\vDash p \lrarr q⊨p↔q 。

# 谓词逻辑

# 基本概念

命题逻辑是对布尔表达式的真假断言，而 谓词逻辑 (predicate logic) 是对多个不同域的表达式取值的断言。可以认为，谓词逻辑是由命题逻辑扩展了域和量词的概念后形成的。

域 (domain) 表示一类取值的集合，例如整数域、实数域等。称定义在域上的变量为 域变量 (domain variables) 。

谓词 (predicate) 是描述取值的某些属性的逻辑断言。为了描述有关取值的属性，谓词逻辑在值域上追加了其它基本关系（例如小于等于）和规则（例如算术加减乘除）。

在谓词逻辑中，我们仍沿用在命题逻辑中使用的术语，例如 p,q,...p,\ q,\ ...p, q, ... 表示谓词，T/FT/FT/F 或 true/falsetrue/falsetrue/false 表示真和假，σ,τ\sigma,\ \tauσ, τ 表示状态，⊨\vDash⊨ 表示满足，等等。

在谓词逻辑中，状态中还可以有域变量的取值的绑定，例如 σ={p=T,x=0,y=α}\sigma = \{p = T,\ x = 0,\ y = \alpha\}σ={p=T, x=0, y=α} 。

# 状态更新

设状态 σ\sigmaσ 中包含对变量 xxx 的绑定，用 σ{x⇝α}\sigma\{x \rightsquigarrow \alpha\}σ{x⇝α} 表示将 σ\sigmaσ 中 xxx 所绑定的取值更新为 α\alphaα ：

σ{x⇝α}(y)={α,(y≡x)σ(y),(y≢x)\sigma\{x \rightsquigarrow \alpha\}(y) = \begin{cases} \alpha, & (y \equiv x) \\ \sigma(y), & (y \not\equiv x) \end{cases}σ{x⇝α}(y)={α,σ(y),​(y≡x)(y​≡x)​

特别地，若 σ\sigmaσ 中没有对变量 xxx 的绑定，则 σ{x⇝α}=σ∪{x=α}\sigma\{x \rightsquigarrow \alpha\} = \sigma \cup \{x = \alpha\}σ{x⇝α}=σ∪{x=α} 。

下面是两个状态取值更新的例子：

  • σ={x=1,y=2},σ{x⇝3}={x=3,y=2}\sigma = \{x = 1,\ y = 2\},\ \sigma\{x \rightsquigarrow 3\} = \{x = 3,\ y = 2\}σ={x=1, y=2}, σ{x⇝3}={x=3, y=2}

  • σ={x=1},σ{y⇝2}={x=1,y=2}\sigma = \{x = 1\},\ \sigma\{y \rightsquigarrow 2\} = \{x = 1,\ y = 2\}σ={x=1}, σ{y⇝2}={x=1, y=2}

# 量词

我们用 量词 (quantifier) 来描述能使谓词为真的变量取值范围：是每个值，还是某个值？

设变量取值 x∈Sx \in Sx∈S ，ppp 是一个涉及变量 xxx 的谓词，则：

  • 全称量词 (universal quantifier) 的形式是 ∀x∈S,p\forall x \in S,\ p∀x∈S, p ，例如 ∀x∈Z,x>1→x<x2\forall x \in \bold{Z},\ x > 1 \to x < x^2∀x∈Z, x>1→x<x2 。

  • 存在量词 (existential quantifier) 的形式是 ∃x∈S,p\exists x \in S,\ p∃x∈S, p ，例如 ∃x∈Z,x≠0∧x=x2\exists x \in \bold{Z},\ x \ne 0 \land x = x^2∃x∈Z, x​=0∧x=x2 。

有界量词 (bounded quantifier) 是量词的另一种等价表示方式，其形式为 ∀p,q\forall p,\ q∀p, q 或 ∃p,q\exists p,\ q∃p, q ，例如 ∀x>1,x<x2\forall x > 1,\ x < x^2∀x>1, x<x2 。设 ppp 涉及 xxx ，若我们希望量化 xxx ，则有如下语义等价转换规则：

  • (∀p,q)⇔(∀x,p→q)(\forall p,\ q) \lrArr (\forall x,\ p \to q)(∀p, q)⇔(∀x, p→q)

  • (∃p,q)⇔(∃x,p∧q)(\exists p,\ q) \lrArr (\exists x,\ p \land q)(∃p, q)⇔(∃x, p∧q)

量化不同变量的量词在句法上是不同的，但在语义上是等价的，即：

  • (∃x∈Z,x≠0)≢(∃y∈Z,y≠0)(\exists x \in \bold{Z},\ x \ne 0) \not\equiv (\exists y \in \bold{Z},\ y \ne 0)(∃x∈Z, x​=0)​≡(∃y∈Z, y​=0)

  • (∃x∈Z,x≠0)⇔(∃y∈Z,y≠0)(\exists x \in \bold{Z},\ x \ne 0) \lrArr (\exists y \in \bold{Z},\ y \ne 0)(∃x∈Z, x​=0)⇔(∃y∈Z, y​=0)

量词 ∀\forall∀ 和 ∃\exists∃ 的优先级被视为最低，例如 ∀x∈Z,x>1→x<x2\forall x \in \bold{Z},\ x > 1 \to x < x^2∀x∈Z, x>1→x<x2 语义等价于 ∀x∈Z,(x>1→x<x2)\forall x \in \bold{Z},\ (x > 1 \to x < x^2)∀x∈Z, (x>1→x<x2) 。

量词有如下德摩根律：

  • (¬∀p,q)⇔(∃p,¬q)(\neg \forall p,\ q) \lrArr (\exists p,\ \neg q)(¬∀p, q)⇔(∃p, ¬q)

  • (¬∃p,q)⇔(∀p,¬q)(\neg \exists p,\ q) \lrArr (\forall p,\ \neg q)(¬∃p, q)⇔(∀p, ¬q)
