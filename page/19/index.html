题面

Difficulty Time Complexity Limit Extra-Memory Complexity Limit
Medium O(n3)O(n^3)O(n3) −-−

Given an array numsnumsnums of nnn integers and an integer targettargettarget , are there elements a, b, c, da,\ b,\ c,\ da, b, c, d in numsnumsnums such that a+b+c+d=targeta + b + c + d = targeta+b+c+d=target ? Find all unique quadruplets in the array which gives the sum of targettargettarget .

Note:

The solution set must not contain duplicate quadruplets.

Example:

题意

给定一个长为 nnn 的数组，给定一个整数 targettargettarget ，求出所有的四元组 (a, b, c, d)(a,\ b,\ c,\ d)(a, b, c, d) ，满足 a+b+c+d=targeta + b + c + d = targeta+b+c+d=target 。要求返回的结果中没有重复的四元组，且每个四元组内部都是有序的。

题解

本题类似于Leetcode 0015，仍可用类似的方法解决，只是细节上有所差别。

在3-sum问题中，我们首先枚举 aaa ，然后在后半部分求取合法的二元组 (b, c)(b,\ c)(b, c) 。现在我们只需枚举 aaa 和 bbb ，然后在后半部分求取合法的二元组 (c, d)(c,\ d)(c, d) 即可，总的时间复杂度为 O(n3)O(n^3)O(n3) 。至于去重和求解2-sum问题的方法，和3-sum问题的求解都是一样的。

为提高效率还可加入一些剪枝操作，主要是利用了 numsnumsnums 的有序，但这些并不会改变最坏时间复杂度：

  • 在枚举 aaa 时，如果 nums[ia]+nums[ia+1]+nums[ia+2]+nums[ia+3]>targetnums[i_a] + nums[i_a + 1] + nums[i_a + 2] + nums[i_a + 3] > targetnums[ia​]+nums[ia​+1]+nums[ia​+2]+nums[ia​+3]>target ，则此后的任意一个四元组之和都将大于 targettargettarget ，可立即终止求解。

  • 在枚举 aaa 时，如果 nums[ia]+nums[n−3]+nums[n−2]+nums[n−1]<targetnums[i_a] + nums[n - 3] + nums[n - 2] + nums[n - 1] < targetnums[ia​]+nums[n−3]+nums[n−2]+nums[n−1]<target ，说明 aaa 的值太小，可直接跳过当次循环。

  • 在枚举 bbb 时可使用类似的两种剪枝，原理相同，因而不予赘述。

题面

Difficulty Time Complexity Limit Extra-Memory Complexity Limit
Medium O(n), C=1O(n),\ C = 1O(n), C=1 −-−

Given a linked list, remove the nnn-th node from the end of list and return its head.

Example:

Note:

Given nnn will always be valid.

Follow up:

Could you do this in one pass?

题意

给定一个链表，给定一个正整数 nnn ，要求将链表从表尾起的第 nnn 个节点删除，并返回此时的链表头。输入保证是 nnn 合法的。要求只遍历链表一次就解决问题。

题解

因为是单链表，我们无法从表尾起反向遍历。一个显然的解决方案是，先遍历一遍链表，求出链表长度 lenlenlen ，那么只要将从表头起的第 len−nlen - nlen−n 个节点删除即可，但这样总共需要遍历链表两次。

至于如何只遍历一次就解决问题，我们考虑用前后指针法，维护两个指针 ppp 和 qqq ，令 qqq 先前进 nnn 步，然后令 p, qp,\ qp, q 同时不断前进，直到 qqq 到达表尾为止。因为 p, qp,\ qp, q 的距离为 nnn ，所以此时 ppp 所指即为从表尾起的第 n+1n+1n+1 个节点，即可完成删除操作。

细节问题：

  1. 注意特判 nnn 等于链表长度的情况，此时需要将 headheadhead 删除并返回head->next 。一个避免特判的方法是开辟一个临时节点接在 headheadhead 的前面，将其置为新的 headheadhead ，这样就消除了原本的 headheadhead 的特殊性。但我的代码中并没有用此处理方法，而是直接特判。
