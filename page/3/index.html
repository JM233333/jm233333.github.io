虽然这也是一个A + B Problem，但是很容易出错哦？

题解

水题。

注意到 A, BA,\ BA, B 的数据范围，首先我们需要读入unsigned long long，其次直接求 (A+B)%P(A + B) \% P(A+B)%P 的话 A+BA + BA+B 会直接爆掉。

根据模运算法则 (a+b)%p=(a%p+b%p)%p(a + b) \% p = (a \% p + b \% p) \% p(a+b)%p=(a%p+b%p)%p 求解，即可避免 A+BA + BA+B 爆掉的问题。

题目描述

JM有一个长为 nnn 的数组 aaa ，数组的下标为 1,2,...,n1, 2, ..., n1,2,...,n 。他可以进行任意多次如下这种操作：

  • 对于两个正整数 iii 和 jjj (1≤i, j≤n)(1 \le i,\ j \le n)(1≤i, j≤n) ，如果此时数组 aaa 满足 ai+aja_i + a_jai​+aj​ 是偶数，则交换 aia_iai​ 和 aja_jaj​ 的值。

现在JM想知道，他在任意次操作之后，能得到的字典序最小的数组是什么？

题解

水题，注意到只有奇偶性相同的元素才能交换，那么所有的奇数之间可以任意排序，所有的偶数之间可以任意排序，将奇数偶数分别排序即可。

不难发现 ai+aja_i + a_jai​+aj​ 是偶数意味着 aia_iai​ 和 aja_jaj​ 的奇偶性相同，也就是说，只有奇偶性相同的元素才能交换。而我们可以进行任意次交换操作，就意味着可以排序，那么所有的奇数之间可以任意排序，所有的偶数之间可以任意排序。也就是说，我们只需要将奇数偶数分别排序即可。

首先考虑如何分别排序。我们额外维护两个数组，分别记录所有的奇数和偶数。对于原始数组的每个元素 aia_iai​ ，我们标记他的奇偶性，并将其插入对应数组中。将两个数组分别排序。

然后考虑如何复原。遍历原数组的标记，如果是奇数，则将奇数数组的首元素取出放入，否则将偶数数组的首元素取出放入，问题到此已经解决。

亦可用两个set或priority_queue求解，代码实现会更加直观。

俄罗斯套娃数组的定义如下（下面简称为套娃数组）：

  1. 套娃数组是一个不定长数组，初始为空。数组的下标为 1,2,...,n1, 2, ..., n1,2,...,n ，其中 nnn 为数组长度。

  2. 套娃数组中的元素可以是正整数，也可以是一个套娃数组。

  3. 套娃数组允许在任意位置插入和删除一个元素；如果插入的元素是一个套娃数组，则插入的肯定是一个空的数组；如果删除的元素是一个套娃数组，则该数组内部的所有元素都将被一并删除，不留痕迹。

  4. 此外，我们有一个唯一的光标，光标会指向一个套娃数组，初始时指向最外层的套娃数组。

作为一个dalao，qz是非常严谨的，他会用一些输入的操作来检测你帮JM实现的俄罗斯套娃数组是否正确。有四种操作：

1 p v：表示在当前所指的数组的下标p的后面插入元素。如果 p=0p = 0p=0 ，则表示插入到数组的最前面。如果当前数组为空，则保证 p=0p = 0p=0 。如果 v=0v = 0v=0 ，则表示插入一个空的套娃数组，否则表示插入一个正整数v。

2 p：表示删除当前所指的数组在下标p处的元素。

3 p：表示将光标指向当前所指的数组在下标p处的数组，即改变光标所指的数组。如果下标p处是一个正整数，则不做任何事，光标不变。如果 p=0p = 0p=0 ，则将光标退回到上级数组。如果当前已经指向最外层，则不做任何事，光标不变。

4：输出当前所指的数组中的所有元素，具体输出格式参考下面的样例。注意，如果元素是一个套娃数组，则只输出一个[]，不递归输出里面嵌套的内容。

题解

按题意模拟即可，可以用stack+指针求解，也可以直接建一棵多叉树求解。

本题唯一的难点在于，数组中的元素既可以是数值、也可以是数组，似乎难以处理的样子。

事实上，我们只需要用一个节点结构把一个数值和一个vector封装在一起，然后设置一个布尔变量表示该节点是数值还是vector即可。注意vector是装载Node的，这样就能实现递归嵌套定义。

也可以建树求解。
