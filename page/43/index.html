前言

前缀和序列 与 差分序列 是两种基础数据结构，多数情况下会用顺序表来实现，常用于解决简单区间问题。

前缀和序列与差分序列包含了最基础的数据结构思想，它们的优点是简单易于实现，缺点是局限性大。它们通常无法解决复杂区间问题，但能够以最优效率解决一些简单区间问题。它们很少单独成题，通常是解决题目所需的辅助手段，甚至以变体出现，因此关键是理解两者的作用和思想，并灵活使用，甚至于解决各类与之思想相似的变种问题，而非生搬硬套模板。

为便于理解它们存在的意义，两者各自通过一个经典问题来引入。参考了ddd大佬的课件。

前缀和序列

容易发现， ∑i=lrai=∑i=1rai−∑i=1l−1ai\sum_{i = l}^{r}{a_i} = \sum_{i = 1}^{r}{a_i} - \sum_{i = 1}^{l-1}{a_i}∑i=lr​ai​=∑i=1r​ai​−∑i=1l−1​ai​ ，即 query(l, r)=sum[r]−sum[l−1]query(l,\ r) = sum[r] - sum[l-1]query(l, r)=sum[r]−sum[l−1] 。也就是说，如果我们预先求出前缀和数组，就可以 O(1)O(1)O(1) 地解决每个查询，那么总的时间复杂度为 O(n+q)O(n + q)O(n+q) 。

应用

前缀和序列可用于解决不带修改的区间求和问题。注意这里强调了不带修改，这指的是 aaa 中元素的值不会改变，或很少被改变。如果问题中存在频繁修改，那么前缀和通常难以胜任。

差分序列

问题引入

应用

差分序列可用于解决不带查询的区间修改问题。注意这里强调了不带查询，这指的是不会频繁询问 aaa 的区间和，或很少询问。通常仅在所有修改结束后询问结果。如果问题中存在和修改相互交叉的频繁询问，那么差分通常难以胜任。

树上差分

学习树上差分之前需要先学习树的LCA（最近公共祖先），请保证阅读前已经掌握LCA的基本知识。

事实上，差分序列可以被推广到有根树上，俗称为树上差分，同样是一项非常重要的技术。

如果我们将DFS回溯求子树和，视为树上前缀和的话，前缀和序列也可以被推广到有根树上。尽管如此，我们主要强调树上差分，毕竟“树上前缀和”是DFS的基本操作，没有什么特别值得再讲一遍的。

树上差分可以分为点差分和边差分两种，它们都是对树上的一条路径进行操作。其核心思想仍然和差分序列相同，因而对其原理不再赘述。

为便于描述，设lca(u, v)lca(u,\ v)lca(u, v)表示树上两点uuu和vvv的LCA，设parentuparent_uparentu​表示树上一点uuu的父节点。

点差分：对树上从uuu到vvv的路径上的所有节点进行+x+x+x操作。

典型问题：给定一棵nnn个节点的树，给定mmm条树上的路径(u, v)(u,\ v)(u, v)，求树上每个节点被多少条路径所覆盖。

边差分：对树上从uuu到vvv的路径上的所有边进行+x+x+x操作。虽说如此，但因为存储问题，我们通常把这里的边信息存储在其深度较大的节点上。

典型问题：给定一棵nnn个节点的树，给定mmm条树上的路径(u, v)(u,\ v)(u, v)，求树上每条边被多少条路径所覆盖。

基本原理

观察和思考易得，二项树具有如下基本性质：

  • 在 BiB_iBi​ 中恰好有 2i2^i2i 个节点。

  • 在 BiB_iBi​ 中深度为 j (0≤j≤i)j\ (0 \le j \le i)j (0≤j≤i) 的节点恰好有 CijC_i^jCij​ 个。（其实这条大概没什么用）

由上述第一条性质推导可得：

  • 对于一个含有 nnn 个节点的二项堆，在 nnn 的二进制表示中，每一个 111 都表示堆中的一棵二项树。例如 n=1310=11012n = 13_{10} = 1101_{2}n=1310​=11012​ ，则二项堆中包含 B0, B2, B3B_0,\ B_2,\ B_3B0​, B2​, B3​ 这 333 棵二项树。

单树合并操作

二项堆的所有操作都是基于单树合并操作实现的，这是二项堆的核心思想。只要能透彻地理解单树合并操作的实现，就能理解有关二项堆的一切。

单树合并操作：将单独的一棵二项树 BiB_iBi​ 合并到一个二项堆 hhh 中。为便于描述，下文中我们会用 hhh 代指 hhh 的根节点列表。

因为在二项堆中，高度为 iii 的二项树最多只有一棵，所以将 BiB_iBi​ 并入 hhh 中只可能会遇到两种情况：

  • 在 hhh 中有且仅有一棵与 BiB_iBi​ 高度相同的二项树 Bj (j=i)B_j\ (j = i)Bj​ (j=i) ；

  • 在 hhh 中不存在与 BiB_iBi​ 高度相同的二项树。

前文已经提到过，在 hhh 中的二项树是按树的深度有序的。不妨设其为升序，那么我们只需要简单地从前往后遍历 hhh ，即可确定是否存在这样的 BjB_jBj​ ：

  • 如果遍历发现一棵二项树 Bj (j>i)B_j\ (j > i)Bj​ (j>i) ，则说明不存在与 BiB_iBi​ 高度相同的二项树，将 BiB_iBi​ 直接插入到对应的位置即可；

  • 如果遍历发现一棵二项树 Bj (j=i)B_j\ (j = i)Bj​ (j=i) ，则将 BjB_jBj​ 取出与 BiB_iBi​ 合并得到 Bi+1B_{i+1}Bi+1​ ，并重新尝试将 Bi+1B_{i+1}Bi+1​ 合并到 hhh 中。

由于 hhh 是有序的，且合并过程消除了 hhh 中所有高度为 iii 的二项树，所以在重新尝试时，我们只需从先前的位置出发继续遍历 hhh 即可，无需从头再来。

修改键值操作即对堆内指定节点的键值进行修改。该操作一般需与其它数据结构配合使用，因为在堆中查找指定节点通常是 O(n)O(n)O(n) 的，所以要想实际运用修改键值操作，通常需要用其它数据结构维护堆中节点的指针或引用，以便在需要时快速获取指定的节点。

假设我们已经取得指向指定节点的指针，增大其键值后，我们只需将修改后的键值与父节点的键值进行比较，并判断是否破坏了堆性质，如果破坏则与父节点交换，然后递归处理，直到满足堆性质为止。该过程很像二叉堆的筛选操作。

优缺点分析

作为可并堆的一种，除了堆的基本操作外，二项堆还支持对多个堆的快速合并，在特定场合可以发挥重要作用。它的另一个优点是可以将查询堆顶最值的操作优化到 O(1)O(1)O(1) 。

遗憾的是，二项堆本身的地位非常尴尬。与左偏树相比，二项堆的代码实现要复杂不少；与斐波那契堆相比，二项堆的时间复杂度则逊色一筹。这使得平时实际应用二项堆的情况较少。

但是学习二项堆仍然是有意义的，因为斐波那契堆实际上借鉴了很多二项堆的思想，先学习二项堆再学习斐波那契堆可以使学习曲线变得平滑，对一些人来说，直接学习斐波那契堆可能会感到有些困难。

基本概念

二分查找 (Binary Search) 可以高效地在一块有序连续空间内查找某个元素，比朴素线性查找要快得多，是一种极为常用的基础算法。

二分查找的思想是基于分治的，但是因为它极为常用且变化多端，所以它通常被从分治中剥离开来。二分查找的正确性要求保证被查找的内容是有序的。

二分查找并不只是被用来进行简单查找的，而是有着繁多的变种问题，因此掌握二分查找的思想及其实现原理就尤为重要，死记硬背是没有用的。

算法思想

最基本的二分查找解决的问题是：给定一个长为 nnn 的有序数组 aaa ，给定一个数 xxx ，要求查找是否存在一个元素 aia_iai​ ，满足 ai=xa_i = xai​=x ，如果存在则给出其下标 iii 。假设 aaa 中不存在重复元素。

为便于描述，设 aaa 的下标为 1,2,...,n1, 2, ..., n1,2,...,n ，且 aaa 中元素按升序排序。

二分查找充分利用了数组的有序性。我们考虑任意一个下标 iii ：

如果 ai=xa_i = xai​=x ，显然我们已经查找成功；

如果 ai≠xa_i \ne xai​​=x ：

  • 对于 ai<xa_i < xai​<x 的情况，因为数组是升序排序的，所以对于任意的下标 j∈[1, i−1]j \in [1,\ i-1]j∈[1, i−1] ，有 aj<aia_j < a_iaj​<ai​ 。因此 aj<xa_j < xaj​<x ，这意味着所有的 aja_jaj​ 都不可能满足查找条件，我们可以仅在 [i+1, n][i+1,\ n][i+1, n] 中继续查找，因为答案要么不存在，要么存在且不在区间 [1, i][1,\ i][1, i] 中。

  • 对于 ai>xa_i > xai​>x 的情况，同理可得，我们可以仅在 [1, i−1][1,\ i-1][1, i−1] 中继续查找。

容易发现，如果初始时选取 i=(1+n)/2i = (1 + n) / 2i=(1+n)/2 ，那么我们的查找范围可以稳定地缩小至原先的一半。只要我们不断地将查找范围减半，我们最多查找 ⌈log⁡n⌉\lceil \log{n} \rceil⌈logn⌉ 次就能确定答案是否存在。那么显然，二分查找的时间复杂度为 O(log⁡n)O(\log{n})O(logn) 。

将上述思想一般化即可实现最基本的二分查找。
