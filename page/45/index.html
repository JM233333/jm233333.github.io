基本概念

Dijkstra算法 (Dijkstra Algorithm) 用于求非负权图上的单源最短路径。

因为Dijkstra算法颇具内涵，初学者可能会感到一定程度的迷茫，或是对算法的理解仅止于表面。因此建议初学者配合其代码实现来学习算法，并反复咀嚼、感悟，否则很容易导致自己似懂非懂，学了没多久就失忆。

首先我们要明确Dijkstra算法解决的问题，即 单源最短路径问题 (single-source shortest path) ：指定图上一个节点 sss 为源点，求从 sss 到图上 所有 节点的最短路径。

为方便起见，我们假设从 sss 出发可以访问到图上的每个节点，否则设最短路径为无穷大。

朴素的Dijkstra算法

前置定义

为便于描述，设节点编号为 1..n1..n1..n ，设 sss 为源点编号，设 dist[u]dist[u]dist[u] 表示从 sss 到 uuu 的 当前 最短路径长度，设 visited[u]visited[u]visited[u] 表示节点 uuu 是否被访问过。

值得注意的是，对于 dist[u]dist[u]dist[u] 的定义中有“当前”的意义，其具体含义是：

  • 若节点 uuu 已访问，则 dist[u]dist[u]dist[u] 表示从 sss 到 uuu 的最短路径长度；

  • 若节点 uuu 未访问，则 dist[u]dist[u]dist[u] 表示在仅考虑已访问节点的条件下，从 sss 到 uuu 的最短路径长度。也就是说，它所表示的路径上所包含的节点一定都是已访问的，当前所有的未访问节点都不在考虑范围之内。

Dijkstra算法的目标就是求出所有节点的已访问 distdistdist 值，也就是最终求得的确切的最短路径。请务必搞清楚 distdistdist 数组的定义。

算法正确性证明

下面我们详细证明Dijkstra算法的正确性。

首先，关于最短路径的性质有如下结论：对于任意一个节点 ddd ，从 sss 到 ddd 的最短路径一定会经过某个节点 uuu ，满足 uuu 是 ddd 的直接前继节点。

这是一个显然的结论，因为从 sss 到 ddd 的任意一条路径总要经过 ddd 的某一个直接前继节点，而最短路径必然也属其列。

根据上述结论，我们只要能够求出 ddd 的每个直接前继节点 uiu_iui​ 的最短路径，然后用这些节点来松弛 ddd ，即 dist[d]=min{(ui, d)∣dist[ui]+cost(ui, d)}dist[d] = min\{(u_i,\ d) \mid dist[u_i] + cost(u_i,\ d)\}dist[d]=min{(ui​, d)∣dist[ui​]+cost(ui​, d)} ，就一定可以找到从 sss 到 ddd 的最短路径。

这其中的关键在于，当 uiu_iui​ 被用来松弛 ddd 时，该节点的最短路径必须已经被求出。

如前文所述，Dijkstra算法的正确性要求图中不含有负权边。如果存在负权边，那么贪心性质会被破坏，上述证明过程将不成立。

有些人认为仅当存在负环时算法不正确，存在负权边但是不存在负环是正确的，事实上并非如此，任何一条负权边的存在都将导致算法不正确。

首先，如果图中存在负环，意味着某些节点的最短路径的不存在的，可以达到无穷小。而Dijkstra算法不具备识别负环的能力，因而必然是不正确的。所以方便起见，下面我们只考虑不存在负环但是存在负权边的情况，并证明此时Dijkstra算法仍然是不正确的。

堆优化的Dijkstra算法

原理

在朴素的Dijkstra算法中，每轮循环都需要一次 O(n)O(n)O(n) 遍历来寻找 distdistdist 值最小的节点，这是算法的瓶颈所在，毕竟 nnn 轮循环是无法省略的。所谓堆优化就是用一个小顶堆来维护这个寻找最大值的过程，将其优化到 log⁡n\log{n}logn 的数量级。

当然，堆优化未必就比朴素算法更优，在稠密图尤其是完全图上，堆优化的表现还不如朴素算法，要分清使用优化的场合。话虽如此，在竞赛题中已经很少会见到需要用朴素算法的情况了。

堆优化的实现方式有很多，如二叉堆、斐波那契堆等，时间复杂度也各不相同。因为C++中有优先队列的实现，使用起来比手写一个堆要方便得多，所以这里只介绍用STL优先队列实现的堆优化及其时间复杂度。事实上，在默认情况下，STL优先队列的内部实现是二叉堆。

在堆优化的Dijkstra算法中，我们维护一个小顶堆，队列中的元素为二元组 (u, dist[u])(u,\ dist[u])(u, dist[u]) ，其中 uuu 为节点编号，dist[u]dist[u]dist[u] 为此二元组入队时的 distdistdist 值。算法不断循环直到队列为空为止。在每次循环中：

  1. 从堆中取出队首的二元组，得到节点编号，记为 uuu 。如果发现 visited[u]==truevisited[u] == truevisited[u]==true ，则直接将其丢弃，重新取队首，直到取出的是一个未访问的节点（或是队列取空）为止。

  2. 遍历 uuu 的所有邻接节点 vvv ，并用边 (u, v)(u,\ v)(u, v) 对节点 vvv 进行松弛操作，如果松弛成功，则将 vvv 加入队列。然后令 visited[u]=truevisited[u] = truevisited[u]=true 。

注意到用一个节点进行松弛操作的过程和朴素的算法如出一辙，只是在获取最小节点的过程中有所改动。

如果你已经基本理解了Dijkstra算法的原理，那么大体上还是能想明白这个堆优化的。但是如果你细心的话就会发现，在有些情况下，某个节点可能会多次入队，而且每次都会携带一个数值不同的 distdistdist 值，为什么这不会导致算法出问题。

别忘了我们用的是一个小顶堆，而每个节点一旦被用于松弛，将会被打上访问标记。也就是说，即便每个节点在优先队列内有多个副本，携带着不同时期不同的 distdistdist 值，当它被用于松弛时，一定是携带着最小的 distdistdist 值，也就是最新的正确的副本。此后，该节点会被标记为已访问，剩余的对答案没有贡献的历史副本都会被算法排除在外，参见前文算法过程中的这一段：

如果发现 visited[u]==truevisited[u] == truevisited[u]==true ，则直接将其丢弃，重新取队首，直到取出的是一个未访问的节点（或是队列取空）为止。

因此重复入队不会导致算法出问题。

事实上，在其他的堆优化中，如果堆结构支持修改节点所携带的权值（例如斐波那契堆），那么可以保证节点不重复入队，每当 distdistdist 值被更新时，如果当前节点已经在堆中，可以高效地对该节点所携带的权值进行修改。这样的实现更贴近算法原本的思想，不过这需要我们手写一些更优的堆结构，徒增代码量。

算法的时间复杂度为 O((n+m)log⁡n)O((n + m)\log{n})O((n+m)logn) ，在连通图上为 O(mlog⁡n)O(m\log{n})O(mlogn) ，其中 nnn 为节点数，mmm 为边数。

初学者请不要试图深究这个复杂度具体是怎么算出来的，你记住这个数量级就好。

基本概念

Floyd-Warshall算法 (Floyd-Warshall Algorithm) 用于求图上的多源最短路径，通常简称为Floyd算法。

许多Floyd算法的教程只是简单地搬出三层循环，并没有讲清楚他的原理。这样的确可以应付简单题，但是无法应对任何一道变形题。最好还是要求自己尽可能地理解Floyd算法的原理。

首先我们要明确Floyd算法解决的问题，即 多源最短路径问题 (multiple-source shortest path) ：求出图上任意点对之间的最短路径，也就是 n×(n−1)n \times (n-1)n×(n−1) 组最短路径。

为方便起见，我们假设图上任意两点之间可达，否则设最短路径为无穷大。

算法原理

和Dijkstra算法不同的是，Floyd算法是基于DP的，而非基于贪心的。

基本概念

Kruskal算法 (Kruskal Algorithm) 用于求带权无向图的最小生成树，和Prim算法相比，该算法更适合应用于稀疏图。

我们回顾一下生成树相关的基本概念：

设 G=(V, E), G′=(V′, E′)G = (V,\ E),\ G' = (V',\ E')G=(V, E), G′=(V′, E′) ，如果 V′⊆VV' \subseteq VV′⊆V 且 E′⊆EE' \subseteq EE′⊆E ，则称 G′G'G′ 是 GGG 的 子图 (subgraph) ，记为 G′⊆GG' \subseteq GG′⊆G 。

设 G′G'G′ 是无向图 GGG 的一个子图，如果 V′=VV' = VV′=V ，且 G′G'G′ 是一棵树，则称 G′G'G′ 为 GGG 的一棵 生成树 (spanning tree) ，记为 TTT 。显然 TTT 不一定存在，如果图 GGG 不连通，则 GGG 上不存在生成树。

设 TTT 为无向带权图 GGG 的一棵生成树，如果在 GGG 的所有生成树中，TTT 的边权和最小，则称 TTT 为 GGG 的 最小生成树 (minimum spanning tree, MST) 。显然一个图的最小生成树不一定是唯一的，可能存在若干棵边权相等的生成树。

很多问题都可以转化为图论中的最小生成树问题，因此最小生成树的求解算法是非常重要且有意义的。

算法流程

为简化问题，下面我们假设图中不含重边和自环，因为重边要选一定会选其中边权最小的一条，而自环一定不属于最小生成树。

Kruskal的算法流程如下：

  1. 初始化最小生成树为 T0=(V,∅)T_0 = (V, \emptyset)T0​=(V,∅) ，即初始没有任何边，只有 nnn 个孤立的点。将图上的所有边按边权从小到大排序。

可以发现，Kruskal算法的过程就是将初始的 nnn 个孤立点不断合并的过程，直到最终将其合并成一整个连通块，也就得到了最小生成树。每向 TTT 中加入一条边，就会将 TTT 中的某两个点集合并。

基本概念

从集合论的角度来讲，拓扑排序就是通过某个集合上的一个偏序序列得到该集合上的一个全序序列。从图论的角度来讲， 拓扑排序 (Topological Sorting) 就是把一个DAG上的所有节点构造成一个拓扑序列。二者是逻辑等价的。

设 GGG 是一个DAG，把 GGG 上的所有节点排成一个线性序列 a={v1, v2, ..., vn}a = \{v_1,\ v_2,\ ...,\ v_n\}a={v1​, v2​, ..., vn​} ，如果对于 GGG 中的任意一条边 (u, v)(u,\ v)(u, v) ，满足 uuu 在 aaa 中的下标比 vvv 的要小，或者说 GGG 中的任意一条边 (vi,vj)(v_i, v_j)(vi​,vj​) 都满足 i<ji < ji<j ，那么称 aaa 是 GGG 的一个 拓扑序列 。

拓扑序列是针对有向图的概念，与无向图无关。

显然，一个有向图的拓扑序列既不一定存在，也不一定唯一。当且仅当图 GGG 是一个DAG时，GGG 的拓扑序列存在。

其实所谓拓扑排序，也就是以任意一个拓扑序列的顺序去遍历全图。拓扑排序的功能就是实现一种顺序特殊的遍历，它并不是什么复杂或高深的算法。

拓扑排序的实现方式有多种，最常见是基于DFS和基于BFS的实现。考虑到前者需要用DFS记录时间戳，对于初学者而言较难理解，因而本文仅以基于BFS的实现为例。

我们知道，拓扑序列是一个受限序列，图中的每一条边都是对该边的终点施加的限制，例如边 (u, v)(u,\ v)(u, v) 是一个对节点 vvv 施加的限制。而一旦该边的起点即节点 uuu 被访问了，那么这个限制就会立刻被解除。事实上，拓扑排序就是基于这一点来设计的。

首先，我们考虑那些不受任何限制的节点，也就是入度为 000 的节点，初始时只有这些节点是可以访问的。无论是如何实现的拓扑排序，我们的拓扑序列一定始于这些节点的其中之一。

其次，每当一个节点 uuu 被访问，就意味着节点 uuu 对其他节点施加的限制已经被解除，我们事实上可以删除所有以 uuu 为起点的边，即 uuu 的所有出边。当然，我们并不需要真的把这条边删掉，只需要维护 degdegdeg 数组即可。也就是说，对于所有的边 (u, vi)(u,\ v_i)(u, vi​) ，我们只需令 deg[vi]−−deg[v_i] --deg[vi​]−− 即可。

需要注意的是，任何一个节点 uuu 被访问，当且仅当此时 deg[u]==0deg[u] == 0deg[u]==0 。

容易发现，随着入度为 000 的节点不断地被访问，图中就会产生一些新的入度为 000 的节点，就这样持续访问下去，我们就能得到一个合法的拓扑序列。

判断拓扑序列是否存在

现在我们考虑如何判断拓扑序列是否存在，即当前图是否为DAG。

其实很简单，如果图上存在环，那么在上述算法的执行过程中，一定会出现：仍有节点未被访问（即环上的那些节点），但是已经不存在任何一个入度为 000 的节点。此时，我们可以判断图上存在环，即拓扑序列不存在。那么我们只需要在搜索终止后判断被访问的节点个数是否等于节点总数即可。也可以判断是否存在一个节点的入度不为 000 。

初始时，我们将所有入度为 000 的节点加入队列中。这相当于BFS从多个起点同时出发。
