DFS + 记忆化搜索

Time Limit Memory Limit Source
1500 MS 524288 K 2019小学期

题目描述

众所周知，tyx是一个大马话家，由于马话说的太多，因此tyx本人也逐渐马化。具体表现为，tyx现在走路也只会走马步，就像中国象棋里的马一样，并且他现在热爱吃草。

现在tyx正处在一块巨大的 nnn 行 mmm 列的马场上，马场上的每个点 (i, j)(i,\ j)(i, j) 都有一定量的草 ai, ja_{i,\ j}ai, j​ 。

作为马话家，tyx自然与一般的马不一样，他每到达一个点，就会将这个点的所有草全吃光，并且他不会重复到达一个点两次。

tyx为了不让自己吃的太多，因此他行进的路线是有讲究的，他的下一个点的草的数量必须要少于当前所在的点上原有的草的数量。当然，tyx还是尽可能的想要多吃一些草，因此他可以任意选择一个点作为他的起点，并规划一条合理的路线。

现在tyx想知道，他最多可以吃到多少草？

接下来 nnn 行，每行 mmm 个正整数，表示马场上每个点的草的数量 ai, ja_{i,\ j}ai, j​ 。

样例输出1

样例输入2

数据范围与提示

1≤n,m≤20001 \le n, m \le 20001≤n,m≤2000

1≤ai, j≤2⋅1061 \le a_{i,\ j} \le 2 \cdot 10^61≤ai, j​≤2⋅106

题解

暴力搜索所有可能的路径显然是一个不可接受的时间复杂度。然而答案具有最优子结构性质，设 dp[x][y]dp[x][y]dp[x][y] 表示从 (x, y)(x,\ y)(x, y) 出发的最大吃草数，那么对于 (x, y)(x,\ y)(x, y) 的所有八方向相邻点 (nxi, nyi)(nx_i,\ ny_i)(nxi​, nyi​) ，显然有 dp[x][y]=max{dp[nxi][nyi]}+ax, ydp[x][y] = max\{dp[nx_i][ny_i]\} + a_{x,\ y}dp[x][y]=max{dp[nxi​][nyi​]}+ax, y​ 。因此可以进行记忆化搜索，时间复杂度为 O(nm)O(nm)O(nm) 。

思维 + BFS

Time Limit Memory Limit Source
1500 MS 524288 K 2019小学期

题目描述

由于tyx抱过了这么多大腿，他变得越来越棒棒了，一般的迷宫根本拦不住他。为了治一治他，你决定为他单独定制一款无限迷宫。

所谓无限迷宫是指，由一个 N×MN \times MN×M 的迷宫单元经过无限平铺得到的迷宫，即将无数份迷宫单元平铺在一个二维平面上。定制好以后，你将tyx扔到了迷宫里，不妨假定tyx落在了起点处。

出于对无限迷宫的恐惧，tyx想要尽可能逃离这里。那么问题来了，tyx能不能逃到距离起点无限远的地方去呢？

输入格式

第一行两个整数 N, MN,\ MN, M ，用来描述迷宫单元的尺寸。

接下来是一个 N×MN \times MN×M 的字符矩阵，用来描述这个迷宫，每个字符一定属于以下三种：

  1. 字符'.'代表这个点是空地。

  2. 字符'S'代表这个点是起点。

  3. 字符'#'代表这个点是墙，不可以走。

输出格式

输出一行一个字符串“Yes”或“No”（不包括引号），“Yes”表示tyx可以逃到无限远的地方，“No”表示不可以。

数据范围与提示

题解

本题看似困难，事实上，只要从一个地图块上的某一点 (x, y)(x,\ y)(x, y) 出发走到其他地图块上的同一点 (x, y)(x,\ y)(x, y) 处，即可无限前进。把这一点想清楚以后问题就解决了。

注意，上述的点 (x, y)(x,\ y)(x, y) 不一定必须是边界点。

至于如何记录是从不同地图到达的，需要一些小技巧。我们在BFS时维护无限大地图上的绝对坐标 (ax, ay)(ax,\ ay)(ax, ay) ，另外维护一个数组 from[rx][ry]from[rx][ry]from[rx][ry] 表示地图块上的某一相对坐标 (rx, ry)(rx,\ ry)(rx, ry) 所对应的绝对坐标，如果出现 from[rx][ry]from[rx][ry]from[rx][ry] 已有记录、且与当前 (ax, ay)(ax,\ ay)(ax, ay) 不同的情况，则说明我们到达了不同地图块上的同一点。

虽然本题用DFS也可以AC，但是是因为XJTUOJ上的栈空间很大，在通常的OJ上递归这么多层早就爆掉了。因此建议用BFS求解。

折半枚举 + DFS

题目描述

为了使你成仙，JM特地为你设计了觉醒之路，只要通过这条路你就能飞天成仙。

觉醒之路可以看作是一个 n×mn \times mn×m 的网格图，左上角为 (1,1)(1, 1)(1,1) ，右下角为 (n,m)(n, m)(n,m) 。网格图上的每个点 (i,j)(i, j)(i,j) 都有一个权值 ai, ja_{i,\ j}ai, j​ 。在网格图上的移动是只能向下或向右的，也就是说，从点 (i,j)(i, j)(i,j) 出发走一步只能走到 (i+1,j)(i+1, j)(i+1,j) 或 (i,j+1)(i, j+1)(i,j+1) ，当然，目标点的坐标不能越界。

初始时你手中的数值为 000 ，每到达一个点 (i,j)(i, j)(i,j) ，都会对你手中的数值异或该点的权值，包括起点和终点。

觉醒之路的觉醒值为 kkk ，现在JM希望你求出存在多少条从 (1,1)(1, 1)(1,1) 到 (n,m)(n, m)(n,m) 的路径，使得走过终点后手中的数值恰好等于觉醒值。

接下来 nnn 行，每行 mmm 个整数 ai, ja_{i,\ j}ai, j​ ，表示网格图上每个点的权值。

题解

路径长度为 n+m−2n + m - 2n+m−2 ，而行走方向要么向下、要么向右，根据乘法原理可知最多不会超过 2n+m−22^{n + m - 2}2n+m−2 条路径。事实上根据组合数可知一共有 Cn+m−2n−1C_{n + m - 2}^{n - 1}Cn+m−2n−1​ 条路径。因为 n, mn,\ mn, m 很小但又不足以直接暴力，故考虑折半搜索。

具体实现考虑用DFS，因为此题显然不适合用二进制枚举，代码实现会非常的麻烦。

先一遍DFS求出从 (1,1)(1, 1)(1,1) 出发走 (n+m−2)/2(n + m - 2) / 2(n+m−2)/2 步的所有情形，并用 nmnmnm 个map记录到达的每个位置 (i,j)(i, j)(i,j) 的所有可能的异或和，再用另一遍DFS求出从 (n,m)(n, m)(n,m) 出发倒着走 (n+m−2)−(n+m−2)/2(n + m - 2) - (n + m - 2) / 2(n+m−2)−(n+m−2)/2 步的所有情形，并在map中查询到达的每个位置存在多少种异或和为 k⊕valk \oplus valk⊕val 的情况（ valvalval 为倒着走到该位置的异或和）。对答案求和即可。

时间复杂度为 O(O(O( 能过 ))) ，懒得算确切的复杂度了。
