基本概念

离散化 (Discretization) 是一种基础的算法思想，通常是对一组数据构建一个映射，使其值域大幅度缩小，从而达到优化时间或空间的目的。

算法思想

离散化的主要目的通常是压缩值域。当我们遇到一组值域很大却数量很少的数据时，有可能会利用到离散化，例如 n≤1e5, ai≤1e18n \le 1e5,\ a_i \le 1e18n≤1e5, ai​≤1e18 。

离散化将会为一个数据集构建一个映射的关系，例如数据集为 a={−128, −2, 1, 104, 104, 1256, 23333, 1e9+7}a = \{-128,\ -2,\ 1,\ 104,\ 104,\ 1256,\ 23333,\ 1e9+7\}a={−128, −2, 1, 104, 104, 1256, 23333, 1e9+7} ，我们可以将其替换为 b={1, 2, 3, 4, 4, 5, 6, 7}b = \{1,\ 2,\ 3,\ 4,\ 4,\ 5,\ 6,\ 7\}b={1, 2, 3, 4, 4, 5, 6, 7} ，映射关系 fff 可能包括 f(−128)=1,f(-128) = 1,f(−128)=1, f(104)=4f(104) = 4f(104)=4 等等。

这样做的主要好处是便于我们处理一组值域很大的数据。

离散化的思想很简单，关键在于如何高效地构建上文所述的映射 fff 。只要能构建出这个映射，离散化也就基本上完成了。

首先 map 的效率并不尽人意，时间常数大得离谱。其次用 map 离散化必须在线操作，每次确定一个元素离散化后的值都必须从 map 中现取，这就影响了时间效率，在一些算法题目中甚至会导致 TLE 。总之，用 map 离散化虽然简单，但是效率低下，未必总是可行，因此掌握手动离散化的方法是有必要的。

基本概念

状态压缩 (State Compression) 是一种基础的算法思想，俗称 压位 ，其核心思想是利用二进制状态保存信息，从而达到优化空间或时间的目的。

算法思想

我们知道，数据在计算机中是以二进制的形式存放的。为便于描述，下文中我们均以 323232 位 int 类型为例。

考虑一个显然的事实，每一个比特位 0/10/10/1 可以表示一个是/否的状态，那么 323232 位就可以表示 323232 个这样的状态，这是非常有用的。

例如，我们可以仅用一个长为 n/32n/32n/32 的 int 数组就能代替一个长为 nnn 的 bool 数组。由于 bool 类型是 888 位的，这实际上节省了不少空间消耗，在很多情况下这非常有用。在 C++ 中可以用 std::bitset 来解决问题，不过这已经不是本文所要探讨的基础问题了，本文不予赘述。

前言

图论 (Graph Theory) 是数学的一个重要分支，图论算法是计算机科学中极为重要的一部分，它提供了对很多问题都有效的一种简单而系统的建模方式。很多问题都可以转化为图论问题，然后用图论的基本算法加以解决。

顾名思义，图论就是研究图的。当然，这里的图指的可不是图画。对于完全没有接触过图论的人而言，图是一个全新的概念。在学习图论之前，必须对图的概念有一个最基本的认识。这些概念并不困难，也不需要死记硬背，初学时只需有一个大概的认知即可。随着你对图论的进一步学习，这些概念就会自然而然地被你记住了。

树的概念

如果一个无向图 GGG 上的任意两点 uuu 和 vvv 之间都有且仅有一条简单路径，那么称 GGG 是一棵树，记为 TTT 。显然，一棵有 nnn 个节点的树一定有恰好 n−1n-1n−1 条边。

通常我们研究的树都是无向树，对于有向树此处不予赘述。

从本质上讲，树其实也只不过是一个图，但它往往会被从图中剥离开来，并且极为重要。其本质原因正是上述定义中说到的，树上任意两点之间有且仅有一条简单路径。不要小看这个特征，正是因为其存在，树才具备了许多一般图所不具备的特质。许多问题在树上可以利用树的特征来解决，如果搬到一般图上，难度会大幅增加、甚至变得在多项式时间内不可解。

设 G′G'G′ 是无向图 GGG 的一个子图，如果 V′=VV' = VV′=V ，且 G′G'G′ 是一棵树，则称 G′G'G′ 为 GGG 的一棵 生成树 (spanning tree) ，记为 TTT 。显然 TTT 不一定存在，如果图 GGG 不连通，则 GGG 上不存在生成树。

设 TTT 为带权无向图 GGG 的一棵生成树，如果在 GGG 的所有生成树中， TTT 的边权和最小，则称 TTT 为 GGG 的 最小生成树 (minimum spanning tree, MST) 。显然一个图的最小生成树不一定是唯一的，可能存在若干棵边权相等的生成树。

连通图的概念

如果一个有向图 GGG 上，将所有的有向边都变成无向边后，图上任意两点 uuu 和 vvv 都是连通的，那么称图 GGG 是一个弱连通图。

如果一个有向图 GGG 上的任意两点 uuu 和 vvv 都是连通的，那么称图 GGG 是一个 强连通图 (strong connected graph) 。

设 G′G'G′ 是有向图 GGG 的一个子图，若 G′G'G′ 是一个强连通图，则称 G′G'G′ 是 GGG 的一个强连通子图。

设 G′G'G′ 是有向图 GGG 的一个强连通子图，若向 G′G'G′ 中增加任意数量的 GGG 中的其他点和边，都将使 G′G'G′ 不满足强连通的性质，则称 G′G'G′ 是 GGG 的一个极大强连通子图，又称为 强连通分量 (strongly connected component, SCC) 。通常我们只研究强连通分量，因为非极大的强连通子图通常没有实用价值。

如果一个无向图 GGG 上的任意两点 uuu 和 vvv 都是连通的，那么称图 GGG 是一个 连通图 (connected graph) 。

设 uuu 是无向图 GGG 上的一个点，如果将 uuu 从图中删去会改变图的连通性，那么称 uuu 是 GGG 上的一个 割点 (cut vertex) 。

设 eee 是无向图 GGG 上的一条边，如果将 eee 从图中删去会改变图的连通性，那么称 eee 是 GGG 上的一条割边，又称为 桥 (bridge) 。

如果一个无向图 GGG 上删去任意一条边都不会改变图的连通性，那么称图 GGG 是一个边双连通图。类似于有向图，可以定义 边双连通分量 (edge-biconnected component) ，因为类同故省略描述。

概念

在图或树上进行 搜索 (Searching) 是解决图论问题的最基本的操作，许多图论算法的实现都需要搜索算法的支持。

因为是入门教程，本文只涉及最基础的 穷竭搜索 算法，包括深度优先搜索和广度优先搜索。

在运用搜索算法时，我们的搜索目的并不总是一样的，这决定了你要如何实现搜索算法，并决定了算法的时间复杂度。当你熟练了之后这是很简单的事情，但是对于初学者来说，明确搜索目的有利于你更有效地学习和理解搜索算法。

遍历 (traverse) 是最常见的搜索目的，遍历一棵树或一个图的含义是访问树或图上的 每个 节点 恰好一次 。

为便于理解，我们在介绍两种搜索算法的基础知识时，将以在树上搜索为例。事实上，在任意图上搜索和在树上搜索的本质是相同的，这在后文中会给出详细解释。

在图论中，树通常是无根的。容易发现，树的结构决定了，无论你以哪个节点作为根节点，树的性质都是不变的，即任意两点之间有且仅有一条路径可达，该性质不会随着根节点的改变而丢失。此外，哪怕是有根树也可以在存储时当成无根树来看待，只需要记录根节点编号即可。总而言之，你不需要像在学习二叉树时那般去特殊考虑根节点的存在，搜索的起点是谁，谁就是根节点。

时间复杂度分析

遍历全图的时间复杂度为 O(n+m)O(n + m)O(n+m) ，其中 nnn 为节点数， mmm 为边数。无论是DFS还是BFS，无论是图还是树，都是如此。

因为搜索算法保证访问每个节点仅一次，即 O(n)O(n)O(n) 。而从每个节点出发遍历邻接节点，也就会遍历到每个节点 uuu 的所有出边，而每个节点的出边个数之和是 O(∑i=1nout_degree[i])=O(m)O(\sum_{i = 1}^{n}{out\_degree[i]}) = O(m)O(∑i=1n​out_degree[i])=O(m) 的（有向图为 mmm 条边，无向图为 2m2m2m 条边），因此总的时间复杂度为 O(n+m)O(n + m)O(n+m) 。

初学者容易误以为时间复杂度为 O(n)O(n)O(n) ，这是错误的，没有考虑到遍历邻边的时间。

如果使用邻接矩阵，则时间复杂度为 O(n2)O(n^2)O(n2) ，因为在邻接矩阵上遍历一个节点的邻边是 O(n)O(n)O(n) 的，那么 nnn 个节点就一共是 n2n^2n2 次，即 O(n+n2)=O(n2)O(n + n^2) = O(n^2)O(n+n2)=O(n2) 。

如果使用链式前向星，则时间复杂度为 O(n+m)O(n + m)O(n+m) ，原理相同。

DFS和BFS都是图论的基本搜索算法，都可以用于图的遍历。如果你只需要遍历图上的所有节点，没有什么特别的约束条件，那么DFS和BFS是等价的。

DFS和BFS的时间复杂度相同。

DFS和BFS都会产生搜索树，这一点后面会介绍。

区别

虽然很多时候二者可以混用，但是在更多的情况下，DFS和BFS还是被用于能发挥出各自特点的场合。

如果是解决树上问题，则通常用DFS解决。例如，求一棵树上以每个节点 iii 为根节点的子树上的节点个数。在树上，DFS几乎可以解决BFS能解决的一切问题。

如果问题与节点的层级有关，则可能可以用BFS解决。例如，求一个无权图上从起点到每个节点 iii 的最短路径。

此外，在无限图上进行特定目标的搜索时，也会根据我们的需要来决定是使用DFS还是使用BFS，并不能随意混用。

这里只是随意列出了几种最简单的情况，后面的扩展知识会说得更详细一些。但这终究只是沧海一粟，最终还是要凭借深入学习和经验积累，才能有效地分清二者的使用场合。简单来说——实践出真知、熟能生巧。

深度优先搜索的应用

遍历所有路径的暴力搜索

解决树上问题

虽然树的本质只是一种特殊的无向图，但它确实非常特殊。许多在树上可以快速解决的图论问题，在任意图上是难以解决的，甚至可能是NP的。

单源最短路径问题 (single-source shortest path) ：指定图上一个节点 sss 为源点，求从 sss 到图上 所有 节点的最短路径。
