基本概念

欧几里德算法 (Euclid Algorithm) 用于计算两个正整数 aaa 和 bbb 的最大公约数，又称为辗转相除法。

算法原理

欧几里德算法的核心原理为 gcd(a, b)=gcd(b, a%b), (b≠0)gcd(a,\ b) = gcd(b,\ a \% b),\ (b \ne 0)gcd(a, b)=gcd(b, a%b), (b​=0) ，下面给出其简要证明。

设 a=qb+ra = qb + ra=qb+r ，其中 a, b, q, ra,\ b,\ q,\ ra, b, q, r 都是整数， q=a/bq = a / bq=a/b ， r=a%br = a \% br=a%b 。

如果 r=0r = 0r=0 ，那么 b∣ab \mid ab∣a （ bbb 是 aaa 的因数），此时显然 gcd(a, b)=bgcd(a,\ b) = bgcd(a, b)=b 。

如果 r≠0r \ne 0r​=0 ，任何同时整除 aaa 和 bbb 的数必定整除 a−qba - qba−qb ，即整除 rrr ，而且任何同时整除 bbb 和 rrr 的数必定整除 qb+rqb + rqb+r ，即整除 aaa ，所以 aaa 和 bbb 的公约数集合与 bbb 和 rrr 的公约数集合是相同的。也就是说， aaa 和 bbb 的最大公约数与 bbb 和 rrr 的最大公约数是相同的，即 gcd(a, b)=gcd(b, r)gcd(a,\ b) = gcd(b,\ r)gcd(a, b)=gcd(b, r) 。

因此可以递归求解。可以证明，算法的时间复杂度为 O(log⁡b)O(\log{b})O(logb) 。

基本概念

KMP算法 (Knuth-Morris-Pratt Algorithm, KMP Algorithm) 是字符串理论的基础算法，主要用于解决字符串的单模式匹配问题，也经常利用该算法的原理来解决一些和子串公共前后缀相关的字符串问题。

KMP算法是字符串理论中极为重要的基础算法，只有深入理解并掌握其相关理论，才能解出高难度的KMP题目。

起初，KMP算法解决的是单模式匹配问题：给定一个主串 SSS 和一个模式串 TTT ，求 TTT 在 SSS 中的匹配次数（或是其他的匹配结果，如不重叠匹配次数等等）。

不过KMP算法的 failfailfail 数组具有许多特殊性质，有很多问题仅需对一个串求取 failfailfail 数组，然后利用这些性质来解决，并不存在主串和模式串的匹配。因此KMP算法的原理必须熟练掌握，尤其是 failfailfail 数组的原理。

对于初学者来说，KMP算法较难理解，因此本文会配以大量的实例图，尽量让读者用更少的学习次数把KMP算法学懂。

为便于描述，设主串 SSS 的下标为 0..∣S∣−10..|S|-10..∣S∣−1 、索引为 i∈[0, ∣S∣−1]i \in [0,\ |S|-1]i∈[0, ∣S∣−1] ，设模式串 TTT 的下标为 0..∣T∣−10..|T|-10..∣T∣−1 、索引为 j∈[0, ∣T∣−1]j \in [0,\ |T|-1]j∈[0, ∣T∣−1] 。

考虑贪心，显然上一次匹配的结果的位置越靠前越好，因此我们只需在每次匹配成功后令 j=0j = 0j=0 ，即从头开始重新匹配，即可求解不重叠匹配问题。

基本概念

字典树 (Trie) 是字符串理论的基础数据结构，又称为 前缀树 (prefix tree) ，主要用于存储和维护大量的字符串，在字母表不大的情况下具有很高的时间效率，并能大量节省空间消耗。

一棵字典树就是一个由若干个字符串组成的集合。字典树会维护这些字符串的一些信息，如每个字符串的出现次数等，以便于高效求解一些字符串问题。

字典树是一种多叉树（准确来说是 ∣∑∣|\sum|∣∑∣ -叉树），树上的每个节点最多有 ∣∑∣|\sum|∣∑∣ 条边，每条边都表示一个字符 ch∈∑ch \in \sumch∈∑ ，每条边指向一个子节点。

我们用 next[ch]next[ch]next[ch] 表示字符 chchch 对应的边所指向的子节点，具体可以用数组或map来实现，当 next[ch]==0next[ch] == 0next[ch]==0 时表示该边不存在。

字典树上的每个节点都可以表示一个字符串，其中根节点可以表示空串。从根节点到某个节点所经过的每条边，将这些边所表示的字符自顶向下依次连接起来，得到的就是该节点所表示的字符串。每个被表示的字符串都互不相同。

但是，每个节点都可以表示一个字符串，并不意味着每个节点都表示一个字符串，有些节点是中间节点，学完后你会对此有所认识，此时先不必纠结。

时间复杂度分析

字典树的查找操作的时间复杂度为 O(∣S∣)O(|S|)O(∣S∣) ，原因很显然。

字典树的插入操作的时间复杂度为 O(∣S∣∣∑∣)O(|S||\sum|)O(∣S∣∣∑∣) ，这里多了一个 ∣∑∣|\sum|∣∑∣ 是因为产生新节点后需要对 nextnextnext 指针列表进行初始化。

如果使用顺序存储并且以如下方式初始化，花费一定的代价预先初始化整个空间，则插入操作的时间复杂度将与查找操作相同。

显然字典树的查找效率比C++的map/set要更优，后者的时间复杂度为 O(∣S∣log⁡n)O(|S|\log{n})O(∣S∣logn) ，其中 nnn 为字符串总数。而这还仅仅是朴素的查找，在处理许多与前缀有关的字符串问题时，字典树的效率要远胜于map/set，是无法替代的重要的数据结构。

优缺点分析

字典树是一种简单却十分高效的数据结构，在字符串理论中有极其重要的作用，被广泛用于解决各类字符串问题，尤其是和前缀有关的字符串问题。

字典树充分利用了被存储字符串的公共前缀，因为字符集是有限的，所以当字符串较多时，总会有一定长度的公共前缀，这就可以被字典树有效地保存，最大限度地节约空间消耗，故字典树可以用于处理大量的字符串。

然而，当字符集太大而字符串却不太多时，每个节点都要保持大量的空指针，可能会造成严重的空间浪费。此时我们可以运用一些其他的数据结构来维护每个节点上的指针集，牺牲少量的时间效率来换取空间利用率，例如可以用STL中的map来维护，这样在字符串稀疏的情况下就能避免浪费大量空间来存储空指针。
