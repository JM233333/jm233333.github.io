LeetCode 0001 - Two Sum

题面

Difficulty Time Complexity Limit Extra-Memory Complexity Limit
Medium O(n2)O(n^2)O(n2) −-−

Given an array nums of nnn integers, are there elements a, b, ca,\ b,\ ca, b, c in nums such that a+b+c=0a + b + c = 0a+b+c=0 ? Find all unique triplets in the array which gives the sum of zero.

Note:

The solution set must not contain duplicate triplets.

Example:

题意

给定一个长为 nnn 的数组，求出所有的三元组 (a, b, c)(a,\ b,\ c)(a, b, c) ，满足 a+b+c=0a + b + c = 0a+b+c=0 。要求返回的结果中没有重复的三元组，且每个三元组内部都是有序的。

题解

暴力枚举的做法很显然，时间复杂度为 O(n3)O(n^3)O(n3) 。

考虑Leetcode 0001中的2-sum问题，如果我们枚举 aaa ，在后半部分求所有的二元组 (b, c)(b,\ c)(b, c) 满足 b+c=−ab + c = -ab+c=−a ，即可将问题转化为2-sum问题，其时间复杂度为 O(n)O(n)O(n) ，因此总的时间复杂度为 O(n2)O(n^2)O(n2) 。

问题在于难以进行判重。如果直接用set暴力判重，则时间复杂度会劣化到 O(n2log⁡n)O(n^2\log{n})O(n2logn) 。虽然这仍然是可以接受的，但我们可以做到更优。

注意到一个事实，我们不能在 O(n)O(n)O(n) 的时间内求解3-sum问题，这意味着花费 O(nlog⁡n)O(n\log{n})O(nlogn) 的时间对数组进行排序是不会影响时间复杂度的。问题转化为在有序数组上求解2-sum问题并去重。

对于有序数组，求解2-sum问题非常简单，很容易想到双指针扫描，时间复杂度仍为 O(n)O(n)O(n) 。虽然没有改进，但是关键是对于有序数组我们可以很方便地在求解过程中顺便完成去重。

首先是双指针扫描的细节，我们初始令 left=ia+1, right=nleft = i_a + 1,\ right = nleft=ia​+1, right=n ，其中 iai_aia​ 表示 aaa 的下标值。考虑当前两数之和 sum2=nums[left]+nums[right]sum2 = nums[left] + nums[right]sum2=nums[left]+nums[right] ，注意到 leftleftleft 右移和 rightrightright 左移分别会使得 sum2sum2sum2 单调增和单调减，因此如果 sum2<targetsum2 < targetsum2<target ，则 leftleftleft 右移，如果 sum2>targetsum2 > targetsum2>target ，则 rightrightright 左移。

因为数组是有序的，所以所有的相等元素都是相邻的。当 leftleftleft 或 rightrightright 移动后，如果当前元素和前一元素的数值相等，显然求解的结果会是重复的，我们直接跳过当前元素。在枚举 aaa 时也可以用同样的方法去重。这样一来，我们就在求解过程中顺便完成了去重，保证了时间复杂度是最优的。
