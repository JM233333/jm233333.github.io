贪心

Time Limit Memory Limit Source
2000 MS 524288 K 2019小学期

题目描述

今天JM收到了一份来自西伯利亚的特快专递，里面装了一个字符串 sss ，仅包含小写英文字母。于是JM决定跟qz和寒域爷一起玩一个游戏：

JM手里拿着字符串 sss ，qz手里拿着字符串 ttt ，寒域爷手里拿着字符串 uuu ，初始时 ttt 和 uuu 均为空。有两种操作：

  1. 将字符串 sss 的首部字符取出，插入字符串 ttt 的尾部。

  2. 将字符串 ttt 的尾部字符取出，插入字符串 uuu 的尾部。

JM，qz和寒域爷随意地进行上述两种操作，直到字符串 sss 和 ttt 均为空时才停止。现在JM想知道，他们能得到的字典序最小的字符串 uuu 是什么。

注意，对于一个字符串 sss ，其长度为 nnn ，其中字符的下标为 1,2,...,n1, 2, ..., n1,2,...,n ，则我们定义 sss 的首部字符为 s1s_1s1​ ，尾部字符为 sns_nsn​ 。

输入格式

一行一个字符串 sss 。

输出格式

一行一个字符串 uuu 表示答案。

样例

样例输入1

样例输出1

样例输入2

样例输出2

数据范围与提示

1≤∣s∣≤1051 \le |s| \le 10^51≤∣s∣≤105

题解

本题非原创，原题为CodeForces 797C。

因为要求字典序最小，所以显然应该使用贪心策略，尽可能使最小的字符出现在前面。也就是说，如果有字符'a'，应先令尽可能多的'a'出现在答案的首部，再去考虑字符'b'。

首先，将 sss 的首部不断取出，插入 ttt ，直至 sss 中的最小元素出现在首部，将这个元素插入 uuu 。

取出第一个最小元素后，再进行贪心时，比较容易出错。我们要将 sss 中的最小元素和 ttt 的尾部进行比较，取最小值插入 uuu 。不要错认为是将 sss 的首部和 ttt 的尾部进行比较，因为可以继续将 sss 的首部不断取出，并得到 sss 中的最小元素。

重复直到 sss 为空，然后将 ttt 中剩余字符逐次插入 uuu 即可。

至于具体实现，考虑用multiset保存 sss 中剩余的所有字符，利用其有序性，取其首部即为最小字符。也可用其他方法来维护，例如开一个大小为 262626 的桶。

贪心 + 思维 + 分类讨论

Time Limit Memory Limit Source
2000 MS 524288 K 2019小学期

题目描述

在某个不可描述的一维世界，寒域爷和qz爷瓜分了世界上的所有城市。现在这个世界也迎来了科技革命，为了不在科技上落后于对方，电缆的建设成为了寒域爷和qz爷统治下的两大国的第一要务。

我们可以把一维世界视为x轴，上面有 nnn 个城市，每个城市都有一个整数的 xxx 坐标，所有城市的坐标互不相同。每个城市的归属是下列三中情况之一：

  1. 城市标记为S，表示它属于寒域爷的 The Empire of Skyair 。

  2. 城市标记为F，表示它属于qz爷的 The Republic of LittleFall 。

  3. 城市标记为N，表示它是中立的 Neutral 。

电缆是沿x轴直线铺设的，每条电缆可以连接两个城市，但是并不会把它铺设路线上途经的所有城市都连接起来。例如有三座城市的坐标为 x1=1, x2=2, x3=3x_1 = 1,\ x_2 = 2,\ x_3 = 3x1​=1, x2​=2, x3​=3，那么如果铺设一条从 111 号城市到 333 号城市的电缆，并不会把城市 222 也接入。但是电缆的通信是具有传递性的，也就是说，如果希望把这三座城市连接起来，显然我们可以先连接 111 和 222 ，再连接 222 和 333 ，这样是最节约的。此外，电缆的长度就是两端城市的坐标之差。

寒域爷和qz爷的两大国并不总是和睦相处的，例如有时候他们会进行商业竞争云云，所以在铺设电缆时，寒域爷和qz爷不得不考虑对方掐断己方的电缆的可能性。但是出于核平主义的考虑，中立城市不应当被斗争所波及，因此电缆的铺设就有了限制条件：

  1. 站在寒域爷的角度，电缆的铺设方案必须保证：即使将qz爷治下的所有城市和与之相连的电缆都删除，只考虑寒域爷的城市和中立的城市，任意两座城市之间仍然可以通过电缆直接或间接通信。

  2. 站在qz爷的角度，电缆的铺设方案必须保证：即使将寒域爷治下的所有城市和与之相连的电缆都删除，只考虑qz爷的城市和中立的城市，任意两座城市之间仍然可以通过电缆直接或间接通信。

铺设的电缆的数量是任意的，连接的城市也是任意的，不受任何限制。电缆本身没有归属，也就是说，两个人不需要各自独立地去铺，我们考虑的是电缆的总长度。

现在，寒域爷和qz爷正在进行商业谈判，商讨如何铺设电缆，既能满足两人的需求，又能保证铺设电缆的长度最短。因为他们很懒，所以他们把你绑了过去，如果你给不出答案，就会把你丢进海里喂鱼。请你给出满足条件所需的最短的电缆铺设总长。

题解

本题非原创，原题为CodeForces 962E。

因为S类城市之间的可达性不能依赖于F类城市，F类城市之间的可达性同样不能依赖于S类城市，所以在S类城市和F类城市之间连线显然是没有意义的。

可以先将这种最简单的特殊情况处理掉：如果没有中立城市，那么将所有的S, F分别依次连接即可，答案就是最大S间距加上最大F间距。

对于最左侧中立城市的左侧，也是将这部分的S, F分别依次连接，最后接到该中立城市上即可。对于最右侧同理。

对于中间的部分，经过画图思考后，我们可能有两种不同的连接思路。以S为例：

  1. 我们可以先将所有的中立城市依次连接，再将每个S贪心地去和最近的S或中立城市相连。

  2. 我们直接将所有的S和中立城市依次连接。

下图中展示了两种情况，为了看着方便，我将S, F放的分别偏上和偏下，但它们应该都是在一条水平线上的。

仔细思考会发现，对于不同的城市分布情况，不一定哪一种连接方案是最优的。

可以用中立城市将整个x轴分为若干个区段。这样一来问题就简单了，对于每两个中立城市之间的S, F城市，我们取两种方案的最小值，然后将每个区段的答案和边界的答案求和即可。

对于每个区段，不可能出现对S采取方案1、却对F采取方案2的情况，因为对S采取方案一必然会使得中立城市已经依次连接，这时对F而言方案1必然更优。

对于每个区段，设 lenlenlen 表示区段间距，设 maxd{T}maxd\{T\}maxd{T} 表示区段内相邻城市间距的最小值（不考虑S和F之间，只考虑S和S之间、F和F之间、S和N之间、F和N之间）。则方案1的答案为 len+(len−maxd{S})+(len−maxd{F})len + (len - maxd\{S\}) + (len - maxd\{F\})len+(len−maxd{S})+(len−maxd{F}) ，方案2的答案为 2∗len2 * len2∗len 。

至于具体实现，写出来其实很简单，线性扫一遍所有城市即可。大多数人都把代码写得过于复杂了。

交互题 + 二分查找

题目描述

这是一道交互题，交互题是指题目的输入会根据你的输出所产生变化。

小秋想和你玩个游戏，你需要猜出小秋最喜欢的那个数字是多少。每次小秋会告诉你，你猜测的数字是大了、小了、还是正好。

你最多可以给出 100100100 次猜测，猜测方式如下：

输出一行，一个整数，表示你所猜测的数字。
