基本概念

Floyd-Warshall算法 (Floyd-Warshall Algorithm) 用于求图上的多源最短路径，通常简称为Floyd算法。

许多Floyd算法的教程只是简单地搬出三层循环，并没有讲清楚他的原理。这样的确可以应付简单题，但是无法应对任何一道变形题。最好还是要求自己尽可能地理解Floyd算法的原理。

首先我们要明确Floyd算法解决的问题，即 多源最短路径问题 (multiple-source shortest path) ：求出图上任意点对之间的最短路径，也就是 n×(n−1)n \times (n-1)n×(n−1) 组最短路径。

为方便起见，我们假设图上任意两点之间可达，否则设最短路径为无穷大。

算法原理

和Dijkstra算法不同的是，Floyd算法是基于DP的，而非基于贪心的。

为便于描述，设节点编号为 1..n1..n1..n 。

设 dp[i][j][k]dp[i][j][k]dp[i][j][k] 表示在仅经过节点 1,2,...,k1, 2, ..., k1,2,...,k 的条件下，从 iii 到 jjj 的最短路径长度。那么对于任意的节点 uuu 和 vvv ，dp[u][v][n]dp[u][v][n]dp[u][v][n] 即为所求。

考虑状态转移，显然 dp[i][j][k]dp[i][j][k]dp[i][j][k] 要么使用了节点 kkk 、要么未使用节点 kkk ，易得状态转移方程：

{dp[i][j][0]=graph[i][j],(i,j∈[1, n], i≠j, initialize)dp[i][i][0]=0,(i∈[1, n], initialize)dp[i][j][k]=min(dp[i][j][k−1], dp[i][k][k−1]+dp[k][j][k−1]),(i,j,k∈[1, n])\begin{cases} dp[i][j][0] = graph[i][j], &(i, j \in [1,\ n],\ i \ne j,\ initialize)\\ dp[i][i][0] = 0, &(i \in [1,\ n],\ initialize)\\ dp[i][j][k] = min(dp[i][j][k-1],\ dp[i][k][k-1] + dp[k][j][k-1]), &(i, j, k \in [1,\ n]) \end{cases}⎩⎪⎨⎪⎧​dp[i][j][0]=graph[i][j],dp[i][i][0]=0,dp[i][j][k]=min(dp[i][j][k−1], dp[i][k][k−1]+dp[k][j][k−1]),​(i,j∈[1, n], i​=j, initialize)(i∈[1, n], initialize)(i,j,k∈[1, n])​

算法的时间复杂度为 O(n3)O(n^3)O(n3) ，三层循环很显然。

类似于01背包的空间优化，因为在状态转移中的第三维只有 [k−1]→[k][k-1] \to [k][k−1]→[k] 的形式，因此可以进行空间优化。

此外需要注意，对 kkk 的循环应该在最外层。理由并不复杂，此处不予赘述，只要你理解了这个DP的状态转移就很容易理解这个细节。

此外有一个常数优化，把循环写成 for(k) for(j) for(i)for (k)\ for (j)\ for (i)for(k) for(j) for(i) 的形式会比写成 for(k) for(i) for(j)for (k)\ for (i)\ for (j)for(k) for(i) for(j) 的形式要快一些，这在卡常的时候很有用。这一常数优化的具体原理此处不予赘述。

记录具体路径

设 path[u][v]path[u][v]path[u][v] 表示在从 uuu 到 vvv 的最短路径上 uuu 的后继节点。初始时，令 path[u][v]==vpath[u][v] == vpath[u][v]==v ，就像初始化 dist[u][v]=graph[u][v]dist[u][v] = graph[u][v]dist[u][v]=graph[u][v] 一样，先假设直接连边即为最短路径。

只要能求出 pathpathpath 数组，那么当我们需要从 uuu 到 vvv 的最短路径时，只需要从 uuu 开始，不断令 u=path[u][v]u = path[u][v]u=path[u][v] ，直到 u==vu == vu==v 时为止，即可得到最短路径。

至于如何求出 pathpathpath 数组，我们在Floyd算法的松弛过程中，在求解状态 dp[i][j][k]dp[i][j][k]dp[i][j][k] 时，如果松弛成功，则令 path[i][j]=path[i][k]path[i][j] = path[i][k]path[i][j]=path[i][k] 。实际上这也是一个DP的状态转移。

事实上，Floyd算法记录路径的方法不止一种，但各自没有优劣之分，并且该方法易于求解字典序最小的路径，因而此处只介绍这一种，对其他方法不予赘述。

判断负环

如果一个环上所有边的边权之和为负，则称该环为一个 负环 。当图上存在负环时，某些节点之间的最短路径可能会不存在，因为可以无限次地通过负环使距离值下降。

Floyd算法不仅可以处理含有负权边的图，还可以判断负环的存在。

事实上，Floyd算法判断负环的原理很简单。如果图中存在负环，那么必然存在至少一个节点 iii ，在松弛过后有 dist[i][i]<0dist[i][i] < 0dist[i][i]<0 。对此进行判断即可。
