基本概念

Prim算法 (Prim Algorithm) 用于求带权无向图的最小生成树，和Kruskal算法相比，该算法更适合应用于稠密图。

我们回顾一下生成树相关的基本概念：

设 G=(V, E), G′=(V′, E′)G = (V,\ E),\ G' = (V',\ E')G=(V, E), G′=(V′, E′) ，如果 V′⊆VV' \subseteq VV′⊆V 且 E′⊆EE' \subseteq EE′⊆E ，则称 G′G'G′ 是 GGG 的 子图 (subgraph) ，记为 G′⊆GG' \subseteq GG′⊆G 。

设 G′G'G′ 是无向图 GGG 的一个子图，如果 V′=VV' = VV′=V ，且 G′G'G′ 是一棵树，则称 G′G'G′ 为 GGG 的一棵 生成树 (spanning tree) ，记为 TTT 。显然 TTT 不一定存在，如果图 GGG 不连通，则 GGG 上不存在生成树。

设 TTT 为无向带权图 GGG 的一棵生成树，如果在 GGG 的所有生成树中， TTT 的边权和最小，则称 TTT 为 GGG 的 最小生成树 (minimum spanning tree, MST) 。显然一个图的最小生成树不一定是唯一的，可能存在若干棵边权相等的生成树。

很多问题都可以转化为图论中的最小生成树问题，因此最小生成树的求解算法是非常重要且有意义的。

算法流程

为简化问题，下面我们假设图中不含重边和自环，因为重边要选一定会选其中边权最小的一条，而自环一定不属于最小生成树。

Prim的算法流程如下：

  1. 初始化最小生成树为T=({V1},∅)T = (\{V_1\}, \empty)T=({V1​},∅)，即初始没有任何边，只有一个点，这个点可以任意选取。

  2. 重复如下操作 n−1n-1n−1 次：暴力枚举图 GGG 中的所有点，找到一个不在 TTT 中的、且和 TTT 中所有点的连边的边权的最小值最小的点，将该点和该最小边加入 TTT 中。这可能有些拗口，通俗地讲，就是找到一个不在 TTT 中的、且和 TTT 的“距离”最小的点。

可以发现，Prim算法是不断扩大当前生成树的过程，直到生成树的大小达到 nnn 为止。每次操作都会将一个点加入生成树中。

Prim算法是基于贪心的，它考虑每次都以最小代价扩大当前生成树，并且一旦将一个点加入生成树，就不会再将其从 TTT 中排除。后文中会给出算法的正确性证明。

算法正确性证明

下面我们简要证明算法的正确性。为便于描述，接下来我们假设图是连通的，即假设最小生成树存在。

首先，我们证明算法一定可以求出一个解。

在算法的执行过程中，每次操作一定会将一个生成树之外的点加入生成树，保证不会产生回路，因此算法最终显然会得到原图的一棵生成树。

其次，我们证明算法得到的解是最优的。

设 TTT 是算法得到的一棵生成树， MSTMSTMST 是一棵最小生成树。

若 T=MSTT = MSTT=MST ，则算法显然是正确的。

若 T≠MSTT \ne MSTT​=MST ，那么在 TTT 的构建过程中，会有若干条边被加入 TTT ，而这些边却不属于 MSTMSTMST 。设 eee 是算法遇到的第一条满足 e∈T, e∉MSTe \in T,\ e \notin MSTe∈T, e∈/​MST 的边。

设 VVV 表示加入 eee 前的生成树的点集，那么 eee 的两个端点必然有一个属于 VVV 、有一个不属于 VVV ，不妨设 e(u,v), u∈V, v∉Ve(u, v),\ u \in V,\ v \notin Ve(u,v), u∈V, v∈/​V 。

因为 MSTMSTMST 是一棵生成树，它必然是连通的，所以在MSTMSTMST必然有且仅有一条路径连通 uuu 和 vvv 两点，而 u∈V, v∉Vu \in V,\ v \notin Vu∈V, v∈/​V ，所以这条路径必定会经过某一条边 fff ，且 fff 的两个端点有一个在VVV内、有一个在 VVV 外。

当 eee 将要被加入 TTT 时，显然 fff 也是一条候选边——算法在若干条候选边中找到边权最小的那一条。而算法选择了 eee 而不是 fff ，那么显然必有 cost(e)≤cost(f)cost(e) \le cost(f)cost(e)≤cost(f) 。而如果 cost(e)<cost(f)cost(e) < cost(f)cost(e)<cost(f) ，那么 cost(T)<cost(MST)cost(T) < cost(MST)cost(T)<cost(MST) ，这与 MSTMSTMST 是最小生成树矛盾，因此必有 cost(e)=cost(f)cost(e) = cost(f)cost(e)=cost(f) 。

对接下来的每一条类似 eee 的边，都可以重复上述证明过程，最终我们得到 cost(T)=cost(MST)cost(T) = cost(MST)cost(T)=cost(MST) ，也就是说 TTT 也是一棵最小生成树。

因此贪心策略的正确性得证。

代码实现 && 时间复杂度分析

下面我们考虑Prim算法的代码实现。

相对于Kruskal算法来说，Prim算法的代码实现并不是直接按照算法流程去写就可以的，那样做的时间复杂度非常低下。

暴力实现算法流程的时间复杂度为 O(n3)O(n^3)O(n3) ，其中有 n−1n-1n−1 次操作，每次操作需要 O(n2)O(n^2)O(n2) 次枚举来找到那个“距离”最小的点。至于为什么是 O(n2)O(n^2)O(n2) 次枚举，因为你需要枚举所有 GGG 中的点，并且对每个 G∖TG \setminus TG∖T 中的点你需要再次枚举所有 TTT 中的点来求得该点和 TTT 的“距离”。

将上述暴力枚举优化到 O(n)O(n)O(n) 次枚举即可得到Prim算法，也就是说，Prim算法的时间复杂度为 O(n2)O(n^2)O(n2) 。

我们维护一个 shortestshortestshortest 数组，设 shortest[u]shortest[u]shortest[u] 表示和节点 uuu 的连边的边权最小的节点，其中 u∈G∖T, shortest[u]∈Tu \in G \setminus T,\ shortest[u] \in Tu∈G∖T, shortest[u]∈T 。

初始节点可任选，不妨选择 111 号节点，那么我们初始令所有的 shortest[u]=1shortest[u] = 1shortest[u]=1。至于shortest[1]shortest[1]shortest[1] 设成什么则无所谓，反正他已经没有意义了。

在每次操作中，在枚举时，只需取 graph[u][shortest[u]]graph[u][shortest[u]]graph[u][shortest[u]] 的值最小的节点 u∈G∖Tu \in G \setminus Tu∈G∖T 即可，只需要 O(n)O(n)O(n) 时间。将 uuu 加入 TTT 后，再用 uuu 去更新所有的 shortest[v] (v∈G∖T)shortest[v]\ (v \in G \setminus T)shortest[v] (v∈G∖T) 即可，同样只需要 O(n)O(n)O(n) 时间。

Prim算法的效率只与图中的点数有关，而与边数无关，并且适合于稠密图。

优先队列优化

类似于Dijkstra算法的优先队列优化，Prim算法邻接表实现的优先队列优化的时间复杂度为 O(mlog⁡n)O(m\log{n})O(mlogn) ，其中 nnn 为点数， mmm 为边数。

但这个优化却缺乏足够的实用价值，因为存图方式为邻接表，算法的时间复杂度已经与边数挂钩，且与Kruskal算法的时间复杂度没什么差别，所以在这种情况下，我们通常会直接选用实现代码更短的Kruskal算法，没有必要实用优先队列优化的Prim算法，故此处不再详细介绍优化的原理，也不会给出代码实现。
