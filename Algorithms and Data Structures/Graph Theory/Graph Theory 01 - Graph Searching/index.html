概念

在图或树上进行 搜索 (Searching) 是解决图论问题的最基本的操作，许多图论算法的实现都需要搜索算法的支持。

因为是入门教程，本文只涉及最基础的 穷竭搜索 算法，包括深度优先搜索和广度优先搜索。

在运用搜索算法时，我们的搜索目的并不总是一样的，这决定了你要如何实现搜索算法，并决定了算法的时间复杂度。当你熟练了之后这是很简单的事情，但是对于初学者来说，明确搜索目的有利于你更有效地学习和理解搜索算法。

遍历 (traverse) 是最常见的搜索目的，遍历一棵树或一个图的含义是访问树或图上的 每个 节点 恰好一次 。

为便于理解，我们在介绍两种搜索算法的基础知识时，将以在树上搜索为例。事实上，在任意图上搜索和在树上搜索的本质是相同的，这在后文中会给出详细解释。

在图论中，树通常是无根的。容易发现，树的结构决定了，无论你以哪个节点作为根节点，树的性质都是不变的，即任意两点之间有且仅有一条路径可达，该性质不会随着根节点的改变而丢失。此外，哪怕是有根树也可以在存储时当成无根树来看待，只需要记录根节点编号即可。总而言之，你不需要像在学习二叉树时那般去特殊考虑根节点的存在，搜索的起点是谁，谁就是根节点。

为方便起见，所有的代码实现均以无权图邻接表为例。本文假定你已经掌握了图论的基本概念，如果使用邻接矩阵或链式前向星，只需将访问图结构的代码自行替换掉即可。

因为无向图可以将每条无向边保存为两条反向的有向边，且在搜索时不涉及二者特性的区别，所以我们不对此作区分，在无向图上搜索和在有向图上搜索的本质是相同的。

深度优先搜索

基本概念

深度优先搜索 (Depth-First Search, DFS) 是最基础的搜索算法之一，通常简称为DFS。

DFS很难用理论性的语言去准确描述，堪称是“只可意会，不可言传”，更多地要靠自身体悟和看图说话来学习。但是当你学会了以后，它将会变得非常简单，它是非常重要的基础算法思想。

对于初学者来说，DFS是很难掌握的，极易陷入“表面理解”的陷阱，需要反复咀嚼和大量的练习才能真正熟练地掌握这一知识，切忌产生轻视心理。

在使用DFS遍历一棵树时，会先访问其根节点，然后依次访问其 每棵子树 。而在访问每棵子树时，又会重复上述过程，问题规模逐步减小，容易发现这正是一个递归的过程，因而DFS在绝大多数情况下就是用递归实现的。

调用函数 dfs(u)dfs(u)dfs(u) 表示的是 访问以节点 uuu 为根节点的子树 。一定要彻底搞清楚调用这个函数的意义，而不是去生搬硬套DFS的代码模板。在写代码之前，要明白你实现的这个函数究竟是什么用处，能够用语言很清晰地表达出来（就像上面的加粗字体那样），这样当你需要自己用搜索算法去解决问题时，才能比较轻松地写出代码实现。

DFS的基本概念还是很好理解的，无非就是“不撞南墙不回头”，一路搜索到底，直到无路可走才会折返。但是DFS其实并没有这么简单。

其一，在很多实际应用DFS的场景，并不总是遵循这一原则，例如我们可能还未遍历完这棵子树，就出于某种原因提前终止搜索而返回。

其二，在几乎所有实际应用DFS的场景，我们并不会这么单纯地只是访问一下这个节点，而是要在访问的过程中计算一些数据，可能是与树和图的结构有关的节点属性，也可能是和问题有关的对答案的贡献值。

其三，多数新手虽然能够快速理解DFS的基本概念，却会在自己写代码去实现DFS时屡屡碰壁，因为DFS比BFS要容易写错得多，而且错误通常会比BFS更难排查——当然，是针对新手而言。

其四，DFS对节点的访问顺序并非是唯一的，这取决于你建树建图的方式，包括存储方式和建边顺序等。例如在上图中的第三步，访问节点 222 之后是先访问的节点 444 ，但也有可能先访问节点 888 。关于这个问题后文中会有更详细的解释。

递归与回溯

回顾DFS的遍历过程，你会发现整个过程中充斥着递归的气息。事实上，DFS的遍历过程就是一个 递归 (recursion) 的过程，在示例图中由红色箭头标记。因而在大多数情况下，我们都是用递归来实现DFS的。

当我们完成对当前子树的遍历时，需要返回父节点，此时发生的是 回溯 (backtrack) 的过程，在示例图中由紫色箭头标记。回溯是DFS中非常重要的部分，这在你学习一段时间后就会有所体会。

当然你也需要明白，虽然DFS的本质就是一个递归的过程，但是这并不意味着我们只能用递归的方式来实现DFS。理论上任何一个递归的方法都可以被写成非递归的形式，虽然这通常会很麻烦，但有时我们的确需要这么做。

在图上的遍历

在任意图上进行遍历时，和在树上会稍微有些不同。

首先，在任意的一个图上，两点之间未必只有一条路径可达，并且节点不再具有父节点的概念。这就意味着我们不再能像在树上搜索一样，“毫无顾忌”地进行搜索。例如，从节点 uuu 出发前往其所有子节点 viv_ivi​ 时，我们必须拒绝访问其中那些已经被访问过的节点。

在树上遍历时，我们通过判断 vi==fv_i == fvi​==f 来防止递归父节点，因为树上任意两点之间有且只有一条路径，所以 fff 必定是 uuu 的唯一一个已访问的邻接节点。但在任意图上，除了前继节点（也就是树上的父节点）以外， uuu 可能还有零至多个已访问的邻接节点。

其次，在任意的一个图上，两点之间未必有路径可达，因为图可能并不是连通的，或是有向图可能并不是强连通的，这也就意味着从某个起点出发进行DFS未必能访问到图上的所有节点。

此时，需要根据实际需要来决定搜索方式。如果只是希望搜索包含起点在内的连通块，则这样即可，如果是希望搜索全图，则需另寻任意一个未访问的节点，作为起点再次开始搜索。

具体地讲，在图上使用DFS的遍历过程如下（是萌新向的描述，因此忽略掉了很多细节）：

广度优先搜索

基本概念

广度优先搜索 (Breadth-First Search, BFS) 是最基础的搜索算法之一，通常简称为BFS。

在使用BFS遍历一棵树时，会先访问其根节点，然后以 层序 对树上的节点进行遍历。也就是说，节点的访问顺序是按节点深度升序的，设 depth[u]depth[u]depth[u] 表示节点 uuu 的深度，当节点 uuu 被访问时，所有深度为 depth[u]−1depth[u] - 1depth[u]−1 的节点必定全部已被访问。

  2. 在搜索的第 iii 轮，访问深度为 iii 的所有节点。每一个这样的节点一定是某个深度为 i−1i-1i−1 的节点的子节点。

从理论上讲，对于同一层的节点，遍历顺序是不受限制的，你可以以任意的顺序访问深度相同的若干个节点，都符合BFS的定义，也不会改变算法的运行结果。但是在实际情况下，由于我们常常用队列来实现BFS，这个遍历顺序还是会受到一定限制的，这在图中也体现了，先无需纠结，下面的代码实现中会对此给出详细解释。

代码实现

考虑到代码复杂程度的问题，我们在树上进行BFS时，一般写得和在图上进行BFS是一样的，并不像DFS那样通过记录父节点来精简代码——那反倒是使代码变得复杂了。

类似于DFS，我们使用 visitedvisitedvisited 数组作为节点的已访问标志。

我们通常借助队列来实现BFS的层序遍历，具体来说是这样的：

  1. 定义节点的队列 qqq ，初始时队列为空。

  2. 将起点 sss 入队。

  3. 不断地取出队首节点 uuu ，访问节点 uuu ，然后遍历 uuu 的所有邻接节点 viv_ivi​ ，如果 viv_ivi​ 未被访问，则设置 viv_ivi​ 为已访问，然后将 viv_ivi​ 入队。

我们考虑队列在这个过程中所起到的作用。

首先，如果我们访问了每一个 distdistdist 值为 iii 的节点，并且每次都将所有未被访问的节点入队，那么 所有 distdistdist 值为 i+1i+1i+1 的节点一定在队列中。特别地，起点 sss 的所有邻接节点就是所有 distdistdist 值为 111 的节点。

其次，你应当知道队列具有FIFO的性质，而队列中的节点一定是按照 distdistdist 值排好序的，因此在不断地取出队首节点时就是按照 distdistdist 值升序取出的。

时间复杂度分析

遍历全图的时间复杂度为 O(n+m)O(n + m)O(n+m) ，其中 nnn 为节点数， mmm 为边数。无论是DFS还是BFS，无论是图还是树，都是如此。

因为搜索算法保证访问每个节点仅一次，即 O(n)O(n)O(n) 。而从每个节点出发遍历邻接节点，也就会遍历到每个节点 uuu 的所有出边，而每个节点的出边个数之和是 O(∑i=1nout_degree[i])=O(m)O(\sum_{i = 1}^{n}{out\_degree[i]}) = O(m)O(∑i=1n​out_degree[i])=O(m) 的（有向图为 mmm 条边，无向图为 2m2m2m 条边），因此总的时间复杂度为 O(n+m)O(n + m)O(n+m) 。

初学者容易误以为时间复杂度为 O(n)O(n)O(n) ，这是错误的，没有考虑到遍历邻边的时间。

如果使用邻接矩阵，则时间复杂度为 O(n2)O(n^2)O(n2) ，因为在邻接矩阵上遍历一个节点的邻边是 O(n)O(n)O(n) 的，那么 nnn 个节点就一共是 n2n^2n2 次，即 O(n+n2)=O(n2)O(n + n^2) = O(n^2)O(n+n2)=O(n2) 。

如果使用链式前向星，则时间复杂度为 O(n+m)O(n + m)O(n+m) ，原理相同。

DFS和BFS的联系和区别

联系

DFS和BFS都是图论的基本搜索算法，都可以用于图的遍历。如果你只需要遍历图上的所有节点，没有什么特别的约束条件，那么DFS和BFS是等价的。

例如，求一个无向图上的所有连通块（注意只能是无向图，有向图的概念是强连通分量）：

在调用 dfs(u, id)dfs(u,\ id)dfs(u, id) 或 bfs(u, id)bfs(u,\ id)bfs(u, id) 时，在遍历的过程中，对于这一轮遍历中所有访问到的节点，将其所属的连通块编号标记为 ididid 即可。

注意，这必须是一个有限的图。有些问题的解空间的无限的，此时应根据情况决定使用何种搜索算法，二者未必都适用。

DFS和BFS的时间复杂度相同。

DFS和BFS都会产生搜索树，这一点后面会介绍。

区别

虽然很多时候二者可以混用，但是在更多的情况下，DFS和BFS还是被用于能发挥出各自特点的场合。

如果是解决树上问题，则通常用DFS解决。例如，求一棵树上以每个节点 iii 为根节点的子树上的节点个数。在树上，DFS几乎可以解决BFS能解决的一切问题。

如果问题与节点的层级有关，则可能可以用BFS解决。例如，求一个无权图上从起点到每个节点 iii 的最短路径。

此外，在无限图上进行特定目标的搜索时，也会根据我们的需要来决定是使用DFS还是使用BFS，并不能随意混用。

这里只是随意列出了几种最简单的情况，后面的扩展知识会说得更详细一些。但这终究只是沧海一粟，最终还是要凭借深入学习和经验积累，才能有效地分清二者的使用场合。简单来说——实践出真知、熟能生巧。

深度优先搜索的应用

遍历所有路径的暴力搜索

首先明确搜索的目的：给定起点 sss 和终点 ddd ，要求遍历所有从 sss 到 ddd 的简单路径。许多问题都和该模型等价。

通常这种问题都用DFS来解决，理论上来说也可以写BFS，但是代码会复杂到令人窒息的程度，因而我们不会去考虑这样做。

搜索所有路径需要运用回溯思想。

考虑一个问题，我们现在似乎不能用之前的 visitedvisitedvisited 数组了，因为不同的路径之间显然可能存在交叉重叠，一个节点会被访问多次，而又不能不使用访问标志，那样我们将无法处理环路。

事实上仍然是用这样的方法，因为虽然一个节点会被访问多次，但是我们所需要的一条路径上并不会重复访问一个节点。我们在进入节点 uuu 时令 visited[u]=truevisited[u] = truevisited[u]=true ，在离开节点 uuu 时令 visited[u]=falsevisited[u] = falsevisited[u]=false ，即可解决问题。

解决树上问题

虽然树的本质只是一种特殊的无向图，但它确实非常特殊。许多在树上可以快速解决的图论问题，在任意图上是难以解决的，甚至可能是NP的。

利用DFS的递归与回溯，我们可以高效地解决大量的树上问题，例如：

  • 求树上每个节点 uuu 的深度 depth[u]depth[u]depth[u] ；

  • 求树上以每个节点 uuu 为根节点的子树 TuT_uTu​ 的大小 siz[u]siz[u]siz[u] ；

  • 求树上以每个节点 uuu 为根节点的子树 TuT_uTu​ 的高度 height[u]height[u]height[u] ；

我们仅以这三个问题为例，每个问题都可以在一遍DFS中解决。由于三个问题互相独立，我们可以仅用一遍DFS将这三个问题一次性全部解决。希望读者能通过这三个问题对树上DFS加深掌握。

深度的求解很好理解，显然 depth[u]=depth[f]+1depth[u] = depth[f] + 1depth[u]=depth[f]+1 ，其中 fff 是 uuu 的父节点，递归即可求出。

后两个问题则需要用到回溯，可能对初学者而言有些难以理解，必须要先对递归和回溯有一定的理解，再看下面的详细解释即可。

我们考虑对于任意一个节点 uuu 而言，子树 TuT_uTu​ 的大小和高度是如何求出的。不难发现，子树大小满足 siz[u]=∑visiz[vi]+1siz[u] = \sum_{v_i}{siz[v_i]} + 1siz[u]=∑vi​​siz[vi​]+1 ，子树高度满足 height[u]=max{height[vi]∣(u, vi)}+1height[u] = max\{height[v_i] \mid (u,\ v_i)\} + 1height[u]=max{height[vi​]∣(u, vi​)}+1 。

也就是说，我们必须先求出以 uuu 的每个子节点 viv_ivi​ 为根节点的子树 TviT_{v_i}Tvi​​ 的大小和高度，才能求出 TuT_uTu​ 的大小和高度。

在自上而下的递归过程中，对于当前遍历到的节点，我们只可能知晓该节点的所有祖先节点的独立的信息。当我们递归终止（例如完成了对某个节点的所有子孙节点的访问，或是因为某种理由提前终止）之后，在自下而上的回溯过程中，回到某个节点时，我们才可能知晓关于该节点的所有子孙节点的信息。

所以我们应当在回溯过程中求解 siz[u]siz[u]siz[u] 和 height[u]height[u]height[u] ，在 dfs(vi)dfs(v_i)dfs(vi​) 完成后回溯时， siz[vi]siz[v_i]siz[vi​] 和 height[vi]height[v_i]height[vi​] 必然已经被求出，此时我们就可以用 viv_ivi​ 的信息去更新 uuu 的信息。当所有子节点都完成回溯时， siz[u]siz[u]siz[u] 和 height[u]height[u]height[u] 也就被求得了。

广度优先搜索的应用

无权图的单源最短路径

BFS可用于解决无权图的单源最短路径问题，这是BFS最简单也是最常见的应用。

在阅读本小节前，请确保你已经掌握Dijkstra算法的基本概念，详见 Graph Theory 知识板块中的 02 - Dijkstra Algorithm 文档。

  • Graph Theory 02 - Dijkstra Algorithm

单源最短路径问题 (single-source shortest path) ：指定图上一个节点 sss 为源点，求从 sss 到图上 所有 节点的最短路径。

为便于描述，设 dist[u]dist[u]dist[u] 表示从 sss 到 uuu 的最短路径长度。

我们知道，BFS对节点的访问是层序的，因此在无权图中，当任意一个节点 uuu 首次被访问时，他一定是经由从 sss 到 uuu 的最短路径被访问的。换句话说，当 uuu 被首次访问时，我们即可确定 dist[u]=dist[f]+1dist[u] = dist[f] + 1dist[u]=dist[f]+1 ，其中 fff 是 uuu 的前继节点。

我们可以避免存储每个节点的前继节点。我们只需在处理节点入队时求解 distdistdist 值即可，对于当前从队列中取出的节点 uuu ，对其所有未被访问的邻接节点 viv_ivi​ ，显然 uuu 是 viv_ivi​ 的前继节点，因此我们只需令 dist[vi]=dist[u]+1dist[v_i] = dist[u] + 1dist[vi​]=dist[u]+1 即可。

搜索生成树

当我们在一个图上进行DFS或BFS时，都会隐式地产生一棵 搜索生成树 。当然，我们也可以把它显式地记录下来。

以下图中给出的有向图为例，我们把DFS和BFS搜索过程中所经过的边都标记出来（在示意图中是把未经过的边淡化了）。当然，可能的搜索顺序不止一种，这只是其中一个例子。

现在我们把这些边的方向去掉，视为无向边。可以发现，我们在搜索过程中所经过的所有的边，和图上的所有节点在一起，恰好构成了一棵树，无论是DFS还是BFS皆是如此。事实上，这样的树就是所谓的搜索生成树。

这很容易理解，因为我们访问图上的每个节点恰好一次，那么对于除搜索起点以外的任意一个节点 uuu ，必然只有一条 uuu 的入边被经过，否则 uuu 将被访问不止一次，因此这些边加起来恰好是 n−1n-1n−1 条。由于每个节点都是可达的，所以它们必然会构成一棵树。

对于有向图而言，容易发现将边的方向剔除并不会改变搜索生成树的性质，因此我们可以忽略这个问题。
