基本概念

二分查找 (Binary Search) 可以高效地在一块有序连续空间内查找某个元素，比朴素线性查找要快得多，是一种极为常用的基础算法。

二分查找的思想是基于分治的，但是因为它极为常用且变化多端，所以它通常被从分治中剥离开来。二分查找的正确性要求保证被查找的内容是有序的。

二分查找并不只是被用来进行简单查找的，而是有着繁多的变种问题，因此掌握二分查找的思想及其实现原理就尤为重要，死记硬背是没有用的。

算法思想

最基本的二分查找解决的问题是：给定一个长为 nnn 的有序数组 aaa ，给定一个数 xxx ，要求查找是否存在一个元素 aia_iai​ ，满足 ai=xa_i = xai​=x ，如果存在则给出其下标 iii 。假设 aaa 中不存在重复元素。

为便于描述，设 aaa 的下标为 1,2,...,n1, 2, ..., n1,2,...,n ，且 aaa 中元素按升序排序。

二分查找充分利用了数组的有序性。我们考虑任意一个下标 iii ：

如果 ai=xa_i = xai​=x ，显然我们已经查找成功；

如果 ai≠xa_i \ne xai​​=x ：

  • 对于 ai<xa_i < xai​<x 的情况，因为数组是升序排序的，所以对于任意的下标 j∈[1, i−1]j \in [1,\ i-1]j∈[1, i−1] ，有 aj<aia_j < a_iaj​<ai​ 。因此 aj<xa_j < xaj​<x ，这意味着所有的 aja_jaj​ 都不可能满足查找条件，我们可以仅在 [i+1, n][i+1,\ n][i+1, n] 中继续查找，因为答案要么不存在，要么存在且不在区间 [1, i][1,\ i][1, i] 中。

  • 对于 ai>xa_i > xai​>x 的情况，同理可得，我们可以仅在 [1, i−1][1,\ i-1][1, i−1] 中继续查找。

容易发现，如果初始时选取 i=(1+n)/2i = (1 + n) / 2i=(1+n)/2 ，那么我们的查找范围可以稳定地缩小至原先的一半。只要我们不断地将查找范围减半，我们最多查找 ⌈log⁡n⌉\lceil \log{n} \rceil⌈logn⌉ 次就能确定答案是否存在。那么显然，二分查找的时间复杂度为 O(log⁡n)O(\log{n})O(logn) 。

将上述思想一般化即可实现最基本的二分查找。

算法实现

二分查找的思想非常简单，但是其代码实现却是算法初学者最大的噩梦之一。二分查找的代码非常容易写错，尤其是在变种问题中，更号称是十个二分九个错。

二分查找的具体实现方式有很多种，本文采用的是闭区间的实现方式。不同实现方式之间没有优劣可言，按个人喜好选择即可。

我们维护一个窗口 [left, right][left,\ right][left, right] ，初始时窗口为 [1, n][1,\ n][1, n] ，即 left=1, right=nleft = 1,\ right = nleft=1, right=n 。

我们不断循环直到查找范围缩小至 000 为止。在每轮循环中，令 mid=(left+right)/2mid = (left + right) / 2mid=(left+right)/2 ，然后：

  • 若 a[mid]=xa[mid] = xa[mid]=x ，则查找成功，返回 midmidmid ；

  • 若 a[mid]>xa[mid] > xa[mid]>x ，则将窗口置为 [left, mid−1][left,\ mid-1][left, mid−1] ；

  • 若 a[mid]<xa[mid] < xa[mid]<x ，则将窗口置为 [mid+1, right][mid+1,\ right][mid+1, right] 。

求上下界的二分查找

二分查找最经典的扩展就是求上下界的查找。

之前为了方便起见，我们假设数组中不存在重复元素。如果存在重复元素的话，可能有不止一个满足条件的下标，例如 a=1,2,2,3, x=2a = {1, 2, 2, 3},\ x = 2a=1,2,2,3, x=2 ，那么 a2a_2a2​ 和 a3a_3a3​ 都是符合的。在存在重复元素的情况下，我们可能会允许返回任意一个合法结果，也可能会希望求出一个上下界，例如求第一个满足条件的元素，那么答案将唯一确定为 a2a_2a2​ 。

求上下界的变种问题有很多，但都是大同小异的，只要你真正地理解了二分查找的实现原理，这些变体你可以自己实现，不会遇到太大的困难，否则就说明你对二分查找的领悟还不够到位。

二分答案

二分答案的策略是二分查找的一个经典应用。

有些题目并不是显式地存在一个可排序的数组，但是它的答案具有单调性，此时我们仍然可以运用二分策略。

最简单的例子，比如我们有一个比较复杂的单调函数 f(x)=tf(x) = tf(x)=t ，给定 ttt 的值，让你求它的解 xxx 。

设答案的取值范围为 x∈[left, right]x \in [left,\ right]x∈[left, right] ，那么每次对于当前值 midmidmid ，计算出 f(mid)f(mid)f(mid) 的值，和 ttt 进行比较，据此修改窗口。其余的部分就是上述的二分查找了，如出一辙。

这个策略俗称为二分答案 + check 。只要问题的解具有单调性，我们就可以二分答案，然后我们通常会写一个 check 函数（因为这个过程通常比较复杂，直接整个塞进二分的代码里会非常恶心），据此进行二分。
