基本概念

KMP算法 (Knuth-Morris-Pratt Algorithm, KMP Algorithm) 是字符串理论的基础算法，主要用于解决字符串的单模式匹配问题，也经常利用该算法的原理来解决一些和子串公共前后缀相关的字符串问题。

KMP算法是字符串理论中极为重要的基础算法，只有深入理解并掌握其相关理论，才能解出高难度的KMP题目。

起初，KMP算法解决的是单模式匹配问题：给定一个主串 SSS 和一个模式串 TTT ，求 TTT 在 SSS 中的匹配次数（或是其他的匹配结果，如不重叠匹配次数等等）。

不过KMP算法的 failfailfail 数组具有许多特殊性质，有很多问题仅需对一个串求取 failfailfail 数组，然后利用这些性质来解决，并不存在主串和模式串的匹配。因此KMP算法的原理必须熟练掌握，尤其是 failfailfail 数组的原理。

对于初学者来说，KMP算法较难理解，因此本文会配以大量的实例图，尽量让读者用更少的学习次数把KMP算法学懂。

为便于描述，设主串 SSS 的下标为 0..∣S∣−10..|S|-10..∣S∣−1 、索引为 i∈[0, ∣S∣−1]i \in [0,\ |S|-1]i∈[0, ∣S∣−1] ，设模式串 TTT 的下标为 0..∣T∣−10..|T|-10..∣T∣−1 、索引为 j∈[0, ∣T∣−1]j \in [0,\ |T|-1]j∈[0, ∣T∣−1] 。

暴力求解

我们不妨先考虑在没有KMP算法时如何解决单模式匹配问题。暴力求解的方法很显然，只需枚举匹配的起点 ttt ，判断是否有 S[t, t+∣T∣−1]==TS[t,\ t+|T|-1] == TS[t, t+∣T∣−1]==T 即可，其中 t+∣T∣−1≤∣S∣−1t + |T| - 1 \le |S| - 1t+∣T∣−1≤∣S∣−1 。显然暴力求解的时间复杂度为 O(∣S∣∣T∣)O(|S||T|)O(∣S∣∣T∣) 。

这样的效率是不能令人满意的，下面我们引入KMP算法。

算法原理

事实上，在暴力匹配的过程中，有许多比较操作是没有意义的。而KMP算法正是通过排除这些没有意义的比较操作来优化算法效率的。

为便于理解，我们也认为KMP算法是不断将 TTT 向右滑动的过程，区别在于，KMP算法不会将主串 SSS 上的索引 iii 回退。如下图所示，第一行为暴力求解算法匹配失败后的操作，第二行为KMP算法匹配失败后的操作。

关键在于，模式串应该向右滑动多少，才能保证不会发生匹配遗漏。

算法流程

下面给出KMP算法进行单模式匹配的流程。

  1. 初始时，令 i=0, j=0i = 0,\ j = 0i=0, j=0 。

  2. 若 Si≠TjS_i \ne T_jSi​​=Tj​ ，则不断令 j=fail[j−1]j = fail[j-1]j=fail[j−1] ，直到 Si=TjS_i = T_jSi​=Tj​或j=0j = 0j=0 为止。这就是模式串向右滑动的过程。

  3. 若 Si==TjS_i == T_jSi​==Tj​ ，则 i++, j++i ++,\ j ++i++, j++ ；否则，仅 i++i ++i++ 。此时，若 j==∣T∣j == |T|j==∣T∣ ，则匹配成功。

  4. 重复步骤2和步骤3，直到 i==∣S∣i == |S|i==∣S∣ 为止。

建立 failfailfail 数组的流程类似，不予赘述。

时间复杂度分析

KMP算法的时间复杂度为 O(∣S∣+∣T∣)O(|S| + |T|)O(∣S∣+∣T∣) 。因为 iii 至多递增 ∣S∣|S|∣S∣ 次， jjj 至多递增 ∣T∣|T|∣T∣ 次，而且内层while循环中每次循环 jjj 至少 −1-1−1 ，因而对 iii 和 jjj 的操作都线性的，故总的时间复杂度也是线性的。不要因为有两层循环就觉得是平方级的。

建立 failfailfail 数组的时间复杂度为 O(∣T∣)O(|T|)O(∣T∣) 。和KMP算法类似，同理可证。

不重叠匹配

如果对单模式匹配问题加以限制，要求模式串每次匹配的结果互相不重叠，求最大的总匹配次数，我们仍可用KMP算法解决。

考虑贪心，显然上一次匹配的结果的位置越靠前越好，因此我们只需在每次匹配成功后令 j=0j = 0j=0 ，即从头开始重新匹配，即可求解不重叠匹配问题。
