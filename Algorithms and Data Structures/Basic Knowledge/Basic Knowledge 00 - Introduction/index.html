前言

这里是关于我的文档内容的一些规范和解释。

我的所有文档均使用C++语言。

我的文档被划分为若干个大的知识板块，在每个知识板块内，每篇文档会有一个标号，这个标号表示了该文档大致的在当前知识板块内的相对级别，建议按标号从小到大阅读，标号相同的则无所谓。

有些文档会具有相同的标号，这是按层划分的，可能会让您感到有些困惑，您可以参阅本文底部的阅读指引图来获得确切的阅读顺序。

一些文档的开头会给出前置知识声明，如果你没有掌握这些知识，建议进行递归学习。这些前置声明和阅读指引图提供的顺序是相同的。

每篇文档的开头均包含版权声明和对本文适用人群的声明，分为“面向初学者”和“面向进阶读者”两大类。

面向初学者：

  1. 适合对当前的大的知识板块掌握尚浅的读者。注意，是相对于当前知识板块，而不是相对于所有知识。

  2. 默认读者没有接触过任何相关知识，会对所有涉及到的、读者可能不知道的知识进行解释。

  3. 通常会以大量的篇幅对简单的基础知识进行解释，用语言引导读者举一反三深入思考。

  4. 会使用精简代码并分段提取出来讲解，用大量的简单示例图来帮助学习，完整的代码实现也会附于文末。

  5. 因为是面向初学者的，哪怕是非常简单的内容，文中也会花费大量篇幅进行解释，因而通常不适合基础较好的读者，请自行避雷。

面向进阶读者：

  1. 适合对当前的大的知识板块已经有所了解，希望学习该板块中更高级的内容的读者。

  2. 默认读者的基础较为良好，不会花费大量篇幅解释一些我认为过于简单的内容。

  3. 通常会深入解释知识的内在原理，甚至给出正确性证明等，帮助读者完全掌握知识，而非浮于表面。

  4. 会提供更多课本上没有的扩展知识，并且通常只给出完整的代码实现，除非是一些重要或较难的关键代码会分段提取出来讲解。

  5. 有些是专门针对ACM选手的文章，知识难度大且实用价值没有那么高，此类情况会特别标注在文章的开头，不打算参加算法竞赛的同学可以自行避过。

特别声明，我的文章并不能帮助你通过学校的书面考试，但能帮助你真正地掌握这些知识，能让你学会如何用代码实现这些基础算法与数据结构。如果只是为了应付考试，那么我的文章并不适合你。

推荐网站

这两个网站提供了一些算法和数据结构的动态可视化预览界面，读者能够非常方便地观看算法和数据结构的整个动态的运行过程，初学者可以借助这些可视化界面进行更加全面的理解和学习。

  • https://visualgo.net/en

  • https://www.cs.usfca.edu/~galles/visualization/Algorithms.html

下面将分别给出每个知识板块对应的阅读指引图，其中：

  • 淡黄色方块表示面向初学者的文章。

  • 小麦色方块表示面向进阶读者的文章。

  • 箭头表示一组前置知识的关系。在阅读一篇文章前，推荐您先阅读其所有前置文章。

  • 空心大箭头表示你将进入到进阶区域。
