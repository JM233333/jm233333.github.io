基本概念

二叉堆 (Binary Heap) 是一种满足堆性质的完全二叉树。

二叉堆可以高效地支持如下基本功能：

  • 插入操作：向堆中插入一个元素，并维持堆性质。

  • 查询最值操作：获取堆顶元素，即获取堆中的最大值（或最小值）。注意通常只允许访问堆顶元素的键值。

  • 弹出操作：将堆顶元素取出，并维持堆性质。注意通常只允许取出堆顶。

二叉堆是最基础、最简单的堆的实现方式，并且能满足多数的使用需求，但有时它是无法胜任的，详见下文中的介绍。

基本操作

为便于理解，我们先在已经存在一个二叉堆的基础上，考虑如何进行插入和弹出操作，再考虑如何建立一个二叉堆。

插入操作

二叉堆执行插入操作的方法是，在堆的末尾处（也就是下标为 n+1n+1n+1 的、目前为空的位置）插入元素，然后对堆进行调整，将新节点不断和父节点进行比较和交换，直到重新满足堆性质为止。

具体地讲，插入操作可分为如下步骤：

  1. 在堆的末尾处插入元素，记其下标为 xxx ，初始为 x=n+1x = n + 1x=n+1 。然后令 n++n ++n++ 。或是先令 n++n ++n++ 再令 x=nx = nx=n ，这不重要。

  2. 将 axa_xax​ 与它的父节点 ax/2a_{x / 2}ax/2​ 比较，如果 ax/2<axa_{x / 2} < a_xax/2​<ax​ ，则说明当前不满足堆性质，将 axa_xax​ 与 ax/2a_{x / 2}ax/2​ 交换。下标 xxx 变为 x/2x / 2x/2 。该过程被称为 向上筛选 (sift-up) 。

  3. 重复步骤 2 ，直到 ax/2≥axa_{x / 2} \ge a_xax/2​≥ax​ 为止，说明当前满足堆性质，插入操作完成。

现在我们考虑插入操作的具体原理及其正确性。

其一，当我们把新元素插入到叶子节点处时，不仅无需先挪动堆中其它节点，而且整个堆上仅仅从新节点到根节点的这条路径上的这些节点可能会失去堆性质，而堆中其他节点的堆性质不受影响，因为只有这些节点是新节点的祖先节点。

其二，每次将 axa_xax​ 和父节点 ax/2a_{x/2}ax/2​ 交换后，在 原先的 下标 xxx 处，以 xxx 为根节点的子树一定满足堆性质，因此可能不满足堆性质的节点个数必定会 −1-1−1 。

其三，一旦当前 ax/2≥axa_{x / 2} \ge a_xax/2​≥ax​ 成立，就意味着 axa_xax​ 的更上层的祖先节点也都将满足堆性质，无需继续调整。因为在 axa_xax​ 被插入前，堆中所有节点都满足堆性质，也就是说 a(x/2)/2≥ax/2a_{(x / 2) / 2} \ge a_{x / 2}a(x/2)/2​≥ax/2​ ，由此递推可知结论成立。

由于二叉堆是一棵完全二叉树，树的高度不超过 log⁡n\log{n}logn ，因此最多向上调整 log⁡n\log{n}logn 次即可保证满足堆性质，所以插入操作的时间复杂度为 O(log⁡n)O(\log{n})O(logn) 。

弹出操作

二叉堆执行弹出操作的方法是，获取堆顶元素后，将堆的末尾元素（也就是下标为 nnn 的位置）移至堆顶位置，然后对堆进行调整，将新节点不断和子节点进行比较和交换，直到重新满足堆性质为止。

具体地讲，弹出操作可分为如下步骤：

  1. 获取堆顶元素，然后将堆的末尾元素移至堆顶位置，记其下标为 xxx ，初始为 x=1x = 1x=1 。然后令 n−−n --n−− 。

  2. 将 axa_xax​ 与它的两个子节点 a2x, a2x+1a_{2x},\ a_{2x + 1}a2x​, a2x+1​ 比较，如果 ax<max(a2x, a2x+1)a_x < max(a_{2x},\ a_{2x + 1})ax​<max(a2x​, a2x+1​) ，则说明当前不满足堆性质，将 axa_xax​ 与子节点 a2x, a2x+1a_{2x},\ a_{2x + 1}a2x​, a2x+1​ 中的较大者交换。下标 xxx 变为 2x2x2x 或 2x+12x+12x+1 。该过程被称为 向下筛选 (sift-down) 。

  3. 重复步骤2，直到 ax≥max(a2x, a2x+1)a_x \ge max(a_{2x},\ a_{2x + 1})ax​≥max(a2x​, a2x+1​) 为止，说明当前满足堆性质，弹出操作完成。

建堆

现在我们考虑如何由一个长为 nnn 的乱序数组 aaa 构建一个堆。

显然，最简单粗暴的办法就是执行 nnn 次插入，时间复杂度为 O(nlog⁡n)O(n\log{n})O(nlogn) 。但是我们有办法可以在 O(n)O(n)O(n) 的时间内进行建堆。该算法是由 Floyd 提出的，通常称为 筛选法建堆 。

算法流程

筛选法建堆的过程很简单，就是倒序遍历整个堆，对每个节点 iii 执行一次向下筛选的操作。这个遍历节点的过程是自下而上的，要严格按照层序进行。

考虑到对叶子节点执行向下筛选是没有意义的，实际上只需从编号为 n/2n/2n/2 的节点开始倒序遍历即可，因为 nnn 是最后一个节点，它的父节点即为编号最大的非叶子节点。

因为是倒序遍历，事实上也是按层遍历，当节点 iii 被遍历到的时候，其左右子树，即以 2i, 2i+12i,\ 2i + 12i, 2i+1 为根节点的子树，一定已经被遍历到。

我们假设这些小一级的子树已经满足堆性质，那么在以 iii 为根节点的子树中，从结果上来看，其结构等价于之前我们的删除操作，那么根据前文所述的原理可知，我们只要对 iii 进行调整即可将该子树调整至满足堆性质。正确性得证。

时间复杂度分析

从直觉上，你可能会认为筛选法建堆的时间复杂度为 O(nlog⁡n)O(n\log{n})O(nlogn) ，因为要遍历 nnn 个节点，每个节点都要进行一次形如删除操作的向下筛选，而删除操作的时间复杂度为 O(log⁡n)O(\log{n})O(logn) 。事实上这是不对的。

需要认清楚的是，无论是向上还是向下，筛选操作的时间复杂度为 O(h)O(h)O(h) ，其中 hhh 为树的高度。在插入和删除操作中，执行的筛选操作是 sift_up(n) 和 sift_down(1) ，都是对整个堆进行的，即 h=⌊log⁡n⌋h = \lfloor \log{n} \rfloorh=⌊logn⌋ 。而在建堆过程中，每次是对以 iii 为根节点的子树进行的，执行的筛选操作是 sift_down(i) 。

下面我们详细分析为什么筛选法建堆的时间复杂度为 O(n)O(n)O(n) 。我们直接计算最大情况，因此将 ⌊log⁡n⌋\lfloor \log{n} \rfloor⌊logn⌋ 记为 log⁡n\log{n}logn ，这不影响结果。

回顾一下二叉树的基本性质，对于二叉树而言，树的第 kkk 层上最多有 2k−12^{k-1}2k−1 个节点，其中树的根节点为第 111 层。那么：

  • 深度为 kkk 的节点个数为 cntk≤2k−1cnt_k \le 2^{k-1}cntk​≤2k−1 。

  • 以深度为 kkk 的节点为根节点的子树的高度为 h≤log⁡n−k+1h \le \log{n} - k + 1h≤logn−k+1 ，也就是说，子树高度为 hhh 的节点的深度为 k≤log⁡n−h+1k \le \log{n} - h + 1k≤logn−h+1 ，因此这样的节点个数最多为 cntlog⁡n−h+1≤2log⁡n−h=n/2hcnt_{\log{n} - h + 1} \le 2^{\log{n} - h} = n / 2^hcntlogn−h+1​≤2logn−h=n/2h 。

所以，这些节点执行筛选操作的总时间为：

优缺点分析

二叉堆的优点就是堆的基本特征，提供快速取出数据集中的最值的功能，以及快速更新并同时维持堆性质不变的功能。此外，二叉堆具有简单易懂的特点，并且其功能足以满足多数情况下的效率要求。

二叉堆的缺点是无法快速地将若干个堆合并。合并两个二叉堆的时间复杂度为 O(n+m)O(n + m)O(n+m) ，其中 n, mn,\ mn, m 为两个堆的大小。即便使用启发式合并也是 O(min⁡(n, m)log⁡(max⁡(n, m)))O(\min(n,\ m)\log{(\max(n,\ m))})O(min(n, m)log(max(n, m))) 的，当 n, mn,\ mn, m 的数量级相近时并无太大意义。当我们需要分多次合并若干个堆的时候，二叉堆显然无法胜任。在这种情况下，我们需要使用任意一种可并堆，详见后续的教程。
