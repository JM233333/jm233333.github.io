基本概念

二项树 (Binomial Tree) 是一种追加了特殊限制的有根有序树。设 Bi (i≥0)B_i\ (i \ge 0)Bi​ (i≥0) 表示高度为 iii 的二项树，则二项树的递归定义如下：

  • B0B_0B0​ 只有 111 个节点，即根节点；

  • BiB_iBi​ 的根节点的度数为 iii ，其子树从左到右依次为 Bi−1, Bi−2, ..., B0B_{i-1},\ B_{i-2},\ ...,\ B_0Bi−1​, Bi−2​, ..., B0​ 。

  • BiB_iBi​ 恰好有 i+1i + 1i+1 层。

为便于描述，我们设二项树的根节点的深度为 000 。

二项堆 (Binomial Heap) 是一种由二项树组成的森林，其中的每棵树都满足堆性质，并且森林中高度为 iii 的二项树最多只有一棵。二项堆的堆顶为所有二项树的根节点中的最值节点。

二叉堆是一种 可并堆 (Mergeable Heap) ，这意味着除了基本的插入操作、查询最值操作和弹出操作外，二项堆还可以高效地支持如下功能：

  • 合并操作：将两个二项堆合并为一个，并保持堆性质。

原理

基本原理

观察和思考易得，二项树具有如下基本性质：

  • 在 BiB_iBi​ 中恰好有 2i2^i2i 个节点。

  • 在 BiB_iBi​ 中深度为 j (0≤j≤i)j\ (0 \le j \le i)j (0≤j≤i) 的节点恰好有 CijC_i^jCij​ 个。（其实这条大概没什么用）

由上述第一条性质推导可得：

  • 对于一个含有 nnn 个节点的二项堆，在 nnn 的二进制表示中，每一个 111 都表示堆中的一棵二项树。例如 n=1310=11012n = 13_{10} = 1101_{2}n=1310​=11012​ ，则二项堆中包含 B0, B2, B3B_0,\ B_2,\ B_3B0​, B2​, B3​ 这 333 棵二项树。

  • 对于一个含有 nnn 个节点的二项堆，其中的二项树不超过 log⁡n\log{n}logn 棵。

我们知道任意一个正整数 nnn 可以表示为形如 ∑ti2i (ti=0/1)\sum{t_i2^i}\ (t_i = 0/1)∑ti​2i (ti​=0/1) 的二进制形式，这也是二项堆被定义为不重复二项树森林的原因，这解决了二项树无法用于表示任意节点个数的问题。此外，我们还可以由此推导证明二项堆各项操作的时间复杂度，具体详见下文。

  • 连接操作：将两个二项树 BiB_iBi​ 合并为 Bi+1B_{i+1}Bi+1​ 。

单树合并操作

二项堆的所有操作都是基于单树合并操作实现的，这是二项堆的核心思想。只要能透彻地理解单树合并操作的实现，就能理解有关二项堆的一切。

单树合并操作：将单独的一棵二项树 BiB_iBi​ 合并到一个二项堆 hhh 中。为便于描述，下文中我们会用 hhh 代指 hhh 的根节点列表。

因为在二项堆中，高度为 iii 的二项树最多只有一棵，所以将 BiB_iBi​ 并入 hhh 中只可能会遇到两种情况：

  • 在 hhh 中有且仅有一棵与 BiB_iBi​ 高度相同的二项树 Bj (j=i)B_j\ (j = i)Bj​ (j=i) ；

  • 在 hhh 中不存在与 BiB_iBi​ 高度相同的二项树。

前文已经提到过，在 hhh 中的二项树是按树的深度有序的。不妨设其为升序，那么我们只需要简单地从前往后遍历 hhh ，即可确定是否存在这样的 BjB_jBj​ ：

  • 如果遍历发现一棵二项树 Bj (j>i)B_j\ (j > i)Bj​ (j>i) ，则说明不存在与 BiB_iBi​ 高度相同的二项树，将 BiB_iBi​ 直接插入到对应的位置即可；

  • 如果遍历发现一棵二项树 Bj (j=i)B_j\ (j = i)Bj​ (j=i) ，则将 BjB_jBj​ 取出与 BiB_iBi​ 合并得到 Bi+1B_{i+1}Bi+1​ ，并重新尝试将 Bi+1B_{i+1}Bi+1​ 合并到 hhh 中。

由于 hhh 是有序的，且合并过程消除了 hhh 中所有高度为 iii 的二项树，所以在重新尝试时，我们只需从先前的位置出发继续遍历 hhh 即可，无需从头再来。

由 hhh 中的二项树不超过 log⁡n\log{n}logn 棵可知：

  • 因为遍历是单向的，所以遍历 hhh 的时间复杂度为 O(log⁡n)O(\log{n})O(logn) ；

  • 因为每次连接都会减少一棵二项树，所以 O(1)O(1)O(1) 的连接操作至多执行 log⁡n\log{n}logn 次，总的时间复杂度为 O(log⁡n)O(\log{n})O(logn) 。

查询最值操作

因为二项堆中的二项树不超过 log⁡n\log{n}logn 棵，所以我们只需要暴力地遍历 hhh 中的所有根节点，并取最值即可，时间复杂度为 O(log⁡n)O(\log{n})O(logn) ，也是可以接受的。

但是我们可以将这个遍历的过程转移到插入、弹出、合并等操作中。我们用指针 pmp_{m}pm​ 来维护二项堆的堆顶，每次对二项堆进行任意一种修改操作之后，立即遍历 hhh 并更新 pmp_{m}pm​ 。这样不仅不会导致这些操作的时间复杂度变差，而且将查询最值操作的时间复杂度优化到了 O(1)O(1)O(1) ，只是其中一些操作的时间常数会变差而已。对于需要大量查询堆顶的场合，该优化的效果非常显著。

修改键值操作

对于大顶堆，该操作为增大键值操作；对于小顶堆，该操作为减小键值操作。事实上，无论是大顶堆还是小顶堆，都可同时以 O(log⁡n)O(\log{n})O(logn) 的复杂度实现增大键值和减小键值操作，但这在其它一些堆中并不通用，且原理又基本相同，因而就不赘述了，读者可自己推广。本文以大顶堆为例，故本节将介绍增大键值操作的原理。

修改键值操作即对堆内指定节点的键值进行修改。该操作一般需与其它数据结构配合使用，因为在堆中查找指定节点通常是 O(n)O(n)O(n) 的，所以要想实际运用修改键值操作，通常需要用其它数据结构维护堆中节点的指针或引用，以便在需要时快速获取指定的节点。

假设我们已经取得指向指定节点的指针，增大其键值后，我们只需将修改后的键值与父节点的键值进行比较，并判断是否破坏了堆性质，如果破坏则与父节点交换，然后递归处理，直到满足堆性质为止。该过程很像二叉堆的筛选操作。

考虑二项堆中二项树高度最高的情况，即只有一棵二项树，因为在 BiB_iBi​ 中恰好有 2i2^i2i 个节点，所以这一棵二项树必为 Blog⁡nB_{\log{n}}Blogn​ ，且 nnn 恰为 222 的整数幂次。又因为 BiB_iBi​ 的高度为 iii ，所以二项堆中二项树的最大高度是 O(log⁡n)O(\log{n})O(logn) 的，所以修改键值操作的时间复杂度为 O(log⁡n)O(\log{n})O(logn) 。

优缺点分析

作为可并堆的一种，除了堆的基本操作外，二项堆还支持对多个堆的快速合并，在特定场合可以发挥重要作用。它的另一个优点是可以将查询堆顶最值的操作优化到 O(1)O(1)O(1) 。

遗憾的是，二项堆本身的地位非常尴尬。与左偏树相比，二项堆的代码实现要复杂不少；与斐波那契堆相比，二项堆的时间复杂度则逊色一筹。这使得平时实际应用二项堆的情况较少。

但是学习二项堆仍然是有意义的，因为斐波那契堆实际上借鉴了很多二项堆的思想，先学习二项堆再学习斐波那契堆可以使学习曲线变得平滑，对部分人的思维方式来说，直接学习斐波那契堆可能会感到有些困难。
