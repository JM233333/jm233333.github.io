基本概念

AVL树 (AVL Tree) 是最先被发明的平衡树，它是由它的发明者——两位苏联科学家 Georgy Adelson-Velsky 和 Evgenii Landis 的名字来命名的。

为便于描述，设 nnn 表示树上节点总数，设 li, ril_i,\ r_ili​, ri​ 表示节点 iii 的左右子节点，设 TiT_iTi​ 表示以 iii 为根节点的子树，设 Height(T)Height(T)Height(T) 表示树 TTT 的高度。特别地，若 TTT 是空树，则 Height(T)=0Height(T) = 0Height(T)=0 。

AVL树中的每个节点 iii 都有一个 平衡因子 (balance factor) ，定义平衡因子为 fi=Height(Tri)−Height(Tli)f_i = Height(T_{r_i}) - Height(T_{l_i})fi​=Height(Tri​​)−Height(Tli​​) ，即节点 iii 的左右子树的高度之差。 fif_ifi​ 为负表示左子树较高，为正表示右子树较高。

AVL树要求保证，对于树上的任意一个节点 iii ，其平衡因子满足 fi∈{−1, 0, 1}f_i \in \{-1,\ 0,\ 1\}fi​∈{−1, 0, 1} ，由此可以保证树高最大为 1.44log⁡n1.44\log{n}1.44logn 左右。绝大多数的教程都没有说清楚这个数据是怎么来的，本文在后文中给出了详细的推导过程，可供有兴趣的同学参考。

AVL旋转

失衡与AVL旋转

除了为了维持平衡性而加入的旋转操作以外，AVL树的基本操作的实现与朴素的二叉查找树完全一致，因此不再赘述其基本原理，只考虑失衡是如何发生的、以及恢复是如何进行的。

对于所有非修改操作，包括查询和遍历操作，显然不会改变AVL树中任何节点的平衡因子，因此无需进行任何额外操作。

对于插入和删除操作，二者均可能会破坏若干个节点的平衡性，所以AVL树在此类操作之后必须进行检查，如果平衡性被破坏，则通过若干次旋转来改变树的结构，在保持二叉查找树的性质的情况下恢复平衡性。

你可能会觉得，子树不平衡有非常多的可能性，事实上，因为在每次插入和删除后都会立即将树恢复平衡，所以实际上需要考虑的情况很少，且必有 ∣fi∣≤2|f_i| \le 2∣fi​∣≤2 。

设 XXX 为失衡节点，即 ∣fX∣=2|f_X| = 2∣fX​∣=2 ，设 ZZZ 是 XXX 的左右子树中高度值较大的那棵子树的根节点。下图中给出了其中一些例子。记住，节点 XXX 和 ZZZ 是我们关注的重点。

需要明白的是，以 ZZZ 为根节点的子树一定是平衡的，只有 XXX 处存在不平衡，否则 ∣fZ∣=2|f_Z| = 2∣fZ​∣=2 将会使得 ∣fX∣=3>2|f_X| = 3 > 2∣fX​∣=3>2 ，这种情况显然是不应该发生的，因为每次插入或删除操作至多会令节点的平衡因子发生大小为 111 的变化，而在 ∣fX∣=2|f_X| = 2∣fX​∣=2 时我们就会将其恢复至 ∣fX∣≤1|f_X| \le 1∣fX​∣≤1 ，这保证了 ∣fX∣|f_X|∣fX​∣ 只会由 111 变成 222 ，而不会变成其他任何值。

此外，平衡因子的计算涉及左右子树的高度，如果我们每次都当场计算这些高度值，会导致时间复杂度劣化成 O(n2log⁡n)O(n^2\log{n})O(n2logn) ，而这些计算中显然是存在冗余的。

为了避免无用的重复计算，我们在每个节点的结构上，存储以该节点为根节点的子树的高度值，然后在插入或删除后更新高度值即可。具体更新方式详见下文中的代码实现。

可能出现的不平衡的子树有四类，包括右右失衡、左左失衡、右左失衡、左右失衡。这里先只列举出来，无需担心，后面会给出详细解释。

对这四类失衡情况，我们执行四类不同的 AVL旋转 (AVL rotations) 操作，包括左旋、右旋、右左旋、左右旋，其中前两者统称为单旋、后两者统称为双旋。这里先只列举出来，无需担心，后面会给出详细解释。

执行AVL旋转的过程有时也被称为 再平衡 (rebalance, or maintain) 过程。

AVL旋转对初学者来说较为复杂难懂，祝你好运。

记住，旋转只是一种被发明出来解决问题的手段，你无需去纠结这个天才的想法是怎么来的，你只需要知道他确实能恢复节点的平衡性，记住这个神一般的操作即可。

单旋

单旋 (single rotation) 包括左旋和右旋，分别用于处理右右失衡和左左失衡。单旋操作会逆转 XXX 和 ZZZ 之间的父子关系。

  • 右右失衡： ZZZ 是 XXX 的右子树， fX=+2f_X = +2fX​=+2 且 fZ≥0f_Z \ge 0fZ​≥0 。当发生右右失衡时，需要执行 左旋 (Left Rotation, or L-Rotation) 操作。

  • 左左失衡： ZZZ 是 XXX 的左子树， fX=−2f_X = -2fX​=−2 且 fZ≤0f_Z \le 0fZ​≤0 。当发生左左失衡时，需要执行 右旋 (Right Rotation, or R-rotation) 操作。

双旋

双旋 (double rotation) 包括右左旋和左右旋，分别用于处理右左失衡和左右失衡。双旋可以被认为是用来处理单旋无法恢复平衡的情况的。

  • 右左失衡： ZZZ 是 XXX 的右子树， fX=+2f_X = +2fX​=+2 且 fZ=−1f_Z = -1fZ​=−1 。当发生右左失衡时，需要执行 右左旋 (Right-Left Rotation, or RL-Rotation) 操作。

  • 左右失衡： ZZZ 是 XXX 的左子树， fX=−2f_X = -2fX​=−2 且 fZ=+1f_Z = +1fZ​=+1 。当发生左右失衡时，需要执行 左右旋 (Left-Right Rotation, or LR-Rotation) 操作。

右左旋和左右旋对应的失衡情况是类似的，只是失衡的方向不同。所以我们接下来将会详细介绍右左旋，而不再介绍左右旋。

如图所示，在右左失衡的情况下，对 XXX 执行左旋操作是没有用处的，这只会让 XXX 从右左失衡转化为左右失衡而已，因此才引入了双旋的方法。

双旋的方法非常巧妙。仔细思考，如果我们在对 XXX 执行左旋操作之前，先对 ZZZ 执行右旋操作，就可以将右左失衡转化为右右失衡！此时再对 XXX 执行左旋操作，就一定可以使之恢复平衡。

所以不要把双旋想得太复杂，如果你已经较为透彻地理解了单旋，那么双旋将会变得很容易理解，它只不过是对两个不同的节点接连执行单旋而已。

基本操作与AVL旋转

单次旋转能使以 XXX 为根节点的子树恢复平衡，但是不一定能使整棵树恢复平衡，原因是在某些失衡情况下，施加旋转操作后，新子树的高度值相对于 插入或删除之前 发生了变化，那么你无法保证更上级的祖先节点的平衡性没有被破坏。

因此，我们需要在操作完成后，从发生变化的节点出发，沿着祖先链不断回溯，对链上的每个节点进行检查和恢复，一直到根节点为止。注意一定是自下而上的进行调整，而不是自上而下的。这个过程可以合并到插入和删除操作的代码实现中。

推导AVL树的最大高度

前文已经提到，维护AVL树的平衡性质，可以保证树的高度值最大为 1.44log⁡n1.44\log{n}1.44logn 左右。这里将给出详细推导。

考虑最大情况

设高度为 hhh 的AVL树的最少节点个数为 n(h)n(h)n(h) 。

我们考虑如何能使节点个数最少而不失衡。

考虑根节点的平衡因子显然满足 ∣f∣=1|f| = 1∣f∣=1 ，否则，一定可以在左子树（或右子树）处删除一些节点，从而使 ∣f∣|f|∣f∣ 从 000 变为 111 。

那么对于高度为 hhh 的AVL树，当 ∣f∣=1|f| = 1∣f∣=1 时，其左右子树的高度分别为 h−1h-1h−1 和 h−2h-2h−2 。

那么我们递归地考虑，每棵子树都应满足同样的条件，我们再让每棵子树的节点个数都尽可能少，即可得到下列递推式：

n(h)=n(h−1)+n(h−2)+1n(h) = n(h-1) + n(h-2) + 1 n(h)=n(h−1)+n(h−2)+1

即树的大小等于左右子树的大小之和 +1+ 1+1 。

此外，显然有 n(0)=0, n(1)=1, n(2)=2n(0) = 0,\ n(1) = 1,\ n(2) = 2n(0)=0, n(1)=1, n(2)=2 。由此我们就得到了一个类似于斐波那契数列的递推公式。

下面我们考虑 1.44log⁡n1.44\log{n}1.44logn 的具体数字是如何得到的。

n=(1−25)(1−52)h−1 + (1+25)(1+52)h−1−1n = (1 - \frac{2}{\sqrt{5}}) {(\frac{1 - \sqrt{5}}{2})}^{h-1}\ +\ (1 + \frac{2}{\sqrt{5}}) {(\frac{1 + \sqrt{5}}{2})}^{h-1} - 1 n=(1−5​2​)(21−5​​)h−1 + (1+5​2​)(21+5​​)h−1−1

我们知道 (1−5)/2≈−0.618(1 - \sqrt{5}) / 2 \approx -0.618(1−5​)/2≈−0.618 ，而 0.61810≈0.008{0.618}^{10} \approx 0.0080.61810≈0.008 ，因此当 hhh 不是太小时，我们就可以将该项对结果的影响忽略不计。而若 hhh 太小，那么时间复杂度稍大一些我们也不在意。

总结

在AVL树中可能出现的失衡情况有四类，对这四类失衡情况，我们执行四类不同的AVL旋转操作：

  • 右右失衡： ZZZ 是 XXX 的右子树， fX=+2f_X = +2fX​=+2 且 fZ≥0f_Z \ge 0fZ​≥0 。当发生右右失衡时，需要执行左旋操作。

  • 左左失衡： ZZZ 是 XXX 的左子树， fX=−2f_X = -2fX​=−2 且 fZ≤0f_Z \le 0fZ​≤0 。当发生左左失衡时，需要执行右旋操作。

  • 右左失衡： ZZZ 是 XXX 的右子树， fX=+2f_X = +2fX​=+2 且 fZ=−1f_Z = -1fZ​=−1 。当发生右左失衡时，需要执行右左旋操作。

  • 左右失衡： ZZZ 是 XXX 的左子树， fX=−2f_X = -2fX​=−2 且 fZ=+1f_Z = +1fZ​=+1 。当发生左右失衡时，需要执行左右旋操作。
