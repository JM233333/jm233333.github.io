题目描述

JM今天懒得编题面了，于是他直接把题意赤裸裸地甩了出来：

给定一个 nnn 个点、 mmm 条边的无向连通图，节点编号为 1,2,...,n1, 2, ..., n1,2,...,n ，边的编号为 1,2,...,m1, 2, ..., m1,2,...,m ，每条边的长度均为 111 。

设 dist(u, v)dist(u,\ v)dist(u, v) 表示图上从节点 uuu 到节点 vvv 的最短路径长度。要求在这 mmm 条边中选择恰好 n−1n-1n−1 条边，将其余边全部删除，使得 sum=∑i=1ndist(1, i)sum = \sum_{i = 1}^{n}{dist(1,\ i)}sum=∑i=1n​dist(1, i) 的值最小。

现在，你需要求出使得 sumsumsum 最小的选边方案有多少种，并输出所有的合法方案。输出格式为 b1b2...bmb_1b_2...b_mb1​b2​...bm​ ，其中 bi=1/0b_i = 1/0bi​=1/0 表示编号为 iii 的边选或不选。输出时按照01串 bbb 的字典序从大到小输出。具体可参考输出样例。

当然，在某些情况下，合法的方案数可能会非常大。因此另外给定一个正整数 kkk ，若合法的方案数超过 kkk 种，则只输出字典序前 kkk 大的 kkk 种合法方案。

输入格式

第一行三个正整数 n, m, kn,\ m,\ kn, m, k ，其中 n, mn,\ mn, m 分别表示图的点数和边数， kkk 表示你需要输出的最大方案数。

接下来 mmm 行，第 iii 行表示编号为 iii 的边，每行两个正整数 uuu 和 vvv ，表示这条双向边所连接的两个节点的编号。

输入保证图是连通的。

输出格式

第一行一个正整数 xxx ，表示使得 sumsumsum 最小的选边方案有多少种。如果方案数超过 kkk ，则输出 kkk 而不是实际的方案数，也就是说你需要输出 min(x, k)min(x,\ k)min(x, k) 。

接下来 min(x, k)min(x,\ k)min(x, k) 行，每行一个01串 b1b2...bmb_1b_2...b_mb1​b2​...bm​ ，表示一种合法的方案。输出时按照01串 bbb 的字典序从大到小输出。若合法的方案数超过 kkk 种，则只输出字典序前 kkk 大的 kkk 种合法方案。

题解

本题非原创，原题为CodeForces 1005F，在原题基础上增加了难度。

先考虑如何求出 sum=∑i=1ndist(1, i)sum = \sum_{i = 1}^{n}{dist(1,\ i)}sum=∑i=1n​dist(1, i) 的最小值。这很简单，只需要从起点跑一次BFS，即可计算出所有的 dist(1, i)dist(1,\ i)dist(1, i) ，实际上就是BFS生成树上节点 iii 的深度。

考虑多种最小方案的存在。当 sumsumsum 最小时，每个 dist(1, i)dist(1,\ i)dist(1, i) 都是最小的，也就是说生成树上每个节点的深度是确定的。我们对节点按深度分层，那么很显然，之所以会有多种方案，是因为从浅层节点到深层节点的路径可能不唯一。也就是说，对于一个节点 uuu 而言，可能存在多个节点 fff 满足：存在一条边 edge(f,u)edge(f, u)edge(f,u) ，且 depth[f]=depth[u]−1depth[f] = depth[u] - 1depth[f]=depth[u]−1 ，这样就有不同的方案以最短距离到达节点 uuu 。

考虑如何求得最小方案数。我们称上述的边 edge(f,u)edge(f, u)edge(f,u) 为有效边，因为只有这样的边会对答案有贡献。我们可以处理出每个点的邻边中的所有有效边，并用 nnn 个 vectorvectorvector 或其他容器将其记录下来，记节点 uuu 的有效边集为 edges[u]edges[u]edges[u] ，时间和空间复杂度均为 O(n+m)O(n + m)O(n+m)。由乘法原理易得，方案数为 ∏u=2nedges[u].size()\prod_{u = 2}^{n}{edges[u].size()}∏u=2n​edges[u].size() 。

原题中只需要输出任意 kkk 种合法方案，一旦知道了多种方案的产生原理，任意寻找合法解就是一件很简单的事情。因为对于一个最小方案，对于每个边集 edges[u]edges[u]edges[u] ，一定有且仅有一条边被记入方案中。所以我们只需要对每个 edges[u]edges[u]edges[u] 枚举是选取哪条边被记入方案即可，直接DFS回溯，直到输出了 min(k, ∏u=2nedges[u].size())min(k,\ \prod_{u = 2}^{n}{edges[u].size()})min(k, ∏u=2n​edges[u].size()) 次方案后停止搜索即可。

但是现在要求输出字典序前 kkk 大的 kkk 种合法方案，难度陡然增加。我们需要尽可能让编号靠前的边被选中，这听起来很容易，实际上并没有那么简单，稍微想想就能发现胡乱贪心是错误的。

考虑暴力地从字典序最大的序列开始倒序枚举，枚举每条边选或不选，然后维护一下每个节点是否被连接到，避免成环。具体地讲，设 visited[u]visited[u]visited[u] 表示节点 uuu 是否被连接到， out[id]out[id]out[id] 表示编号为 ididid 的边连向的节点，那么选中边 iii 时只需令 visited[out[i]]=truevisited[out[i]] = truevisited[out[i]]=true 即可，在选边之前检测节点 out[i]out[i]out[i] 是否已经被选中即可。显然这样一定能得到正确的答案，但却毫无疑问会导致TLE，因为在极端情况下时间消耗会指数爆炸。

我们需要进行剪枝。设 lastid[u]lastid[u]lastid[u] 表示连接到节点 uuu 的编号最靠后的边，那么如果当前已经枚举到了边 lastid[u]lastid[u]lastid[u] ，而节点 uuu 仍然未被连接到，则这条边必须选中。
