题目描述

神奇的JM最近沉迷玩俄罗斯套娃不能自拔，于是他灵机一动，想搞一个俄罗斯套娃数组，但他的电脑昨天被寒域爷浇了一罐肥宅快乐水，现在还在抢修中，所以他万不得已只能找你帮忙了。

俄罗斯套娃数组的定义如下（下面简称为套娃数组）：

  1. 套娃数组是一个不定长数组，初始为空。数组的下标为 1,2,...,n1, 2, ..., n1,2,...,n ，其中 nnn 为数组长度。

  2. 套娃数组中的元素可以是正整数，也可以是一个套娃数组。

  3. 套娃数组允许在任意位置插入和删除一个元素；如果插入的元素是一个套娃数组，则插入的肯定是一个空的数组；如果删除的元素是一个套娃数组，则该数组内部的所有元素都将被一并删除，不留痕迹。

  4. 此外，我们有一个唯一的光标，光标会指向一个套娃数组，初始时指向最外层的套娃数组。

作为一个dalao，qz是非常严谨的，他会用一些输入的操作来检测你帮JM实现的俄罗斯套娃数组是否正确。有四种操作：

1 p v：表示在当前所指的数组的下标p的后面插入元素。如果 p=0p = 0p=0 ，则表示插入到数组的最前面。如果当前数组为空，则保证 p=0p = 0p=0 。如果 v=0v = 0v=0 ，则表示插入一个空的套娃数组，否则表示插入一个正整数v。

2 p：表示删除当前所指的数组在下标p处的元素。

3 p：表示将光标指向当前所指的数组在下标p处的数组，即改变光标所指的数组。如果下标p处是一个正整数，则不做任何事，光标不变。如果 p=0p = 0p=0 ，则将光标退回到上级数组。如果当前已经指向最外层，则不做任何事，光标不变。

4：输出当前所指的数组中的所有元素，具体输出格式参考下面的样例。注意，如果元素是一个套娃数组，则只输出一个[]，不递归输出里面嵌套的内容。

保证输入的所有操作都是合法的，即保证：

  1. 对于操作1有 0≤p≤n0 \le p \le n0≤p≤n 且 v≥0v \ge 0v≥0 。

  2. 对于操作2有 1≤p≤n1 \le p \le n1≤p≤n。

  3. 对于操作3有 0≤p≤n0 \le p \le n0≤p≤n。

  4. 不会出现 1,2,3,41, 2, 3, 41,2,3,4 以外的操作。

对于每次操作4，输出一行，表示当前所指的数组。

题解

按题意模拟即可，可以用stack+指针求解，也可以直接建一棵多叉树求解。

本题唯一的难点在于，数组中的元素既可以是数值、也可以是数组，似乎难以处理的样子。

事实上，我们只需要用一个节点结构把一个数值和一个vector封装在一起，然后设置一个布尔变量表示该节点是数值还是vector即可。注意vector是装载Node的，这样就能实现递归嵌套定义。

具体的实现方法不止一种。

可以用stack+指针求解。

因为需要模拟嵌套数组的进出，不能进去以后出不来了，所以在进行操作3时必须维护所有的父级节点。我们用一个栈来维护指向每个祖先节点的指针，如果向内进就将节点地址入栈，如果向外出就出栈。

也可以建树求解。

题目本质上就是实现一棵多叉树，建树是最直接的思路。但是本题实际上并不需要建树，而且建树也比较麻烦。
