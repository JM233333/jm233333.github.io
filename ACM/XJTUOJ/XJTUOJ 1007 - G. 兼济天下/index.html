题目描述

你和你的迷弟们吃饭的时候，一般是你请客。

但万一你csf囊中羞涩（怎么可能？），大家就会执行AA制(平分饭费)，并且其中一个迷弟去结账，其它人把钱转给他。这个去结账的人也是需要给钱的，但是此时你csf是不需要给钱的。

csf的迷弟们在给人转钱的时候总是向上取整，那么结账者就会收到比饭费总数更多的钱，我们把结账者额外获得的钱数称为福利值（单位：元）。

现在有 nnn 个人一起吃饭（包括csf），一共花了 mmm 元钱，请问这顿饭的福利值是多少？

如果福利大于 000 ，输出一个小数，精确到小数点后六位，表示福利值。

否则，输出“CSFNB!”。（不包括引号）

输出格式

输出共 TTT 行，每行要么是一个小数，保留六位，表示结账者的福利值。要么是“CSFNB!”（不包括引号）。

题解

这是本场比赛的签到题，但是仍然有不少人栽在了读题上。

现在有 nnn 个人一起吃饭（包括csf），一共花了 mmm 元钱。

其中一个迷弟去结账，其它人把钱转给他。这个去结账的人也是需要给钱的，但是此时你csf是不需要给钱的。

csf的迷弟们在给人转钱的时候总是向上取整，那么结账者就会收到比饭费总数更多的钱，我们把结账者额外获得的钱数称为福利值。

根据题意，显然有：

  • 当 m%(n−1)==0m \% (n - 1) == 0m%(n−1)==0 时，输出“CSFNB!”；

  • 否则，答案为 res=((n−2)∗⌈part⌉+part)−mres = ((n - 2) * \lceil part \rceil + part) - mres=((n−2)∗⌈part⌉+part)−m ，其中 part=(double)m/(n−1)part = (double)m / (n - 1)part=(double)m/(n−1) 表示每个人本来应当付的钱。

具体解释，结账者会收到另外 n−2n - 2n−2 个人的向上取整的钱，加上结账者自己出的钱，即 (n−2)∗⌈part⌉+part(n - 2) * \lceil part \rceil + part(n−2)∗⌈part⌉+part ，减去付出的钱 mmm ，即为他的福利值。

注意 float 类型的精度只有 10−610^{-6}10−6 ，因此我们应当使用 double 类型求解，否则存在WA的可能性。这一点我们已经在提示中写明了。
