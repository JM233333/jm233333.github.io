题目描述

两位创始人商议成立量化科技有限公司，取名 “全天候”，但你们还需要招募合适的团队成员，才能顺利开展工作，你将招募到匹配的人员，称之为匹配星星游戏，游戏规则如下：

天上有 nnn 颗星星，每颗星星有二维坐标 (xi, yi)(x_i,\ y_i)(xi​, yi​) ，还有一个属性值 ziz_izi​ ，若两颗星星 A, BA,\ BA, B 满足 xA<xBx_A < x_BxA​<xB​ 且 yA<yBy_A < y_ByA​<yB​ 且 zA<zBz_A < z_BzA​<zB​ ，则这两颗星星可以配成一对，每颗星星最多只能在一对之中，求最多能配成多少对星星。

题解

这道题目可以使用贪心算法求解。先按照 xxx 坐标从小到大排序，然后依次遍历每个点，进行如下处理：

  • 如果 z=0z = 0z=0 ，将该点加入候选点；

  • 如果 z=1z = 1z=1 ，查询比该点 xxx 坐标小且 z=0z = 0z=0 的所有点中 yyy 坐标最大的的点，进行配对，如果配对成功则将那个点都从候选点中排除。

贪心正确性证明：

假设最优方案中排序后 z=1z = 1z=1 的第 111 个没有按贪心策略匹配的点 iii ，在设 iii 在 z=0z = 0z=0 中它能匹配的 yyy 最大的点为 jjj ，如果 jjj 被点 kkk 匹配了，有两种情况：

  • 原先 iii 没有和任何匹配，则直接去掉 kkk 的匹配，将 iii 和 jjj 匹配；

  • 原先 iii 和 mmm 匹配，则将 kkk 改成和 mmm 匹配，将 iii 和 jjj 匹配，由于 yj>ym​y_j > y_m​yj​>ym​​ ，这样的匹配一定是成立的。并且修改后的匹配数不会变少，因此按照该贪心策略可以得到最优解。

至于具体实现，考虑到 yyy 坐标的有序性质，我们用 multiset 维护所有的候选点，按 yyy 坐标排序，然后每次查询时在 multiset 上 lower_boundlower\_boundlower_bound 即可。

需要注意的是，对于 xxx 坐标相等的点，我们要按照 zzz 坐标从大到小排序，以避免误将 xA=xBx_A = x_BxA​=xB​ 且 yA<yBy_A < y_ByA​<yB​ 且 zA<zBz_A < z_BzA​<zB​ 的情况计入答案。
