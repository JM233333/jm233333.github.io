LeetCode 0023 - Merge k Sorted Lists

Posted on 2019-11-12 In LeetCode Changyan:

Hints

  • 链表 + 优先队列

题面

Difficulty Time Complexity Limit Extra-Memory Complexity Limit
Hard O(nlog⁡k)O(n\log{k})O(nlogk) −-−

Merge kkk sorted linked lists and return it as one sorted list. Analyze and describe its complexity.

Example:

题意

给定 kkk 个有序链表，要求将其中所有节点合并为一个新的有序链表。

题解

我没明白为什么这道题被设为Hard，我觉得顶多是Medium……

暴力的做法很显然，类似LeetCode 0021，我们维护 kkk 个指针，每次取其中的最小者递进，时间复杂度为 O(nk)O(nk)O(nk) ，其中 nnn 为节点总数。

上述 “kkk-指针扫描” 的方法低效的原因在于，对于每个节点我们都需要比较 kkk 次，而在LeetCode 0021中我们只需要比较 222 次。

我们考虑维护一个优先队列，将所有指针放入其中，以其所指节点的键值大小排序，每次取出堆顶指针递进，这样我们就将 O(k)O(k)O(k) 的比较优化至 O(log⁡k)O(\log{k})O(logk) ，那么总的时间复杂度为 O(nlog⁡k)O(n\log{k})O(nlogk) 。

细节问题：

  1. 注意特判空链表。
