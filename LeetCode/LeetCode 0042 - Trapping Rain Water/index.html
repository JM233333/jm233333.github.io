LeetCode 0042 - Trapping Rain Water

题面

Difficulty Time Complexity Limit Extra-Memory Complexity Limit
Hard O(n)O(n)O(n) −-−

Given nnn non-negative integers representing an elevation map where the width of each bar is 111 , compute how much water it is able to trap after raining.

The above elevation map is represented by array [0,1,0,2,1,0,1,3,2,1,2,1][0,1,0,2,1,0,1,3,2,1,2,1][0,1,0,2,1,0,1,3,2,1,2,1] . In this case, 666 units of rain water (blue section) are being trapped. Thanks Marcos for contributing this image!

题意

给定一个长为 nnn 的非负整数数组 heightheightheight ，从左往右有 nnn 根柱子，第 iii 根柱子的高度为 height[i]height[i]height[i] ，宽度为 111 ，求这些柱子最多能盛多少体积的水。

题解

我们考虑从左往右扫描，不难想到，如果遇到柱子 iii 满足 height[i]≥max⁡{height[j]∣j<i}height[i] \ge \max\{height[j] \mid j < i\}height[i]≥max{height[j]∣j<i} ，则区间 [p, i][p,\ i][p, i] 的盛水量已经可以计算，其中 ppp 为上次计算的末端。但是这样的方法无法处理柱子的最高峰的右侧。

方法1：栈。

既然无法一次性计算区间 [p, i][p,\ i][p, i] 的盛水量，那么我们就利用栈的性质将其分段计算。

我们用栈来维护柱子的标号 iii ，设 toptoptop 表示栈顶的柱子的标号。我们从左往右依次遍历每根柱子，如果发现 height[i]≥height[top]height[i] \ge height[top]height[i]≥height[top] ，就说明我们可以将栈顶取出，并进行一些计算。

具体地讲，我们将栈顶取出并记为 indexindexindex ，同时获取下一个栈顶 toptoptop ，那么区间 [top+1, i−1][top + 1,\ i - 1][top+1, i−1] 中高度为 [height[index]+1, min(height[i], height[top])][height[index] + 1,\ min(height[i],\ height[top])][height[index]+1, min(height[i], height[top])] 的部分将被计算。如果出栈后栈为空，说明不存在盛水，我们不进行上述计算。

方法2：动态规划。

这个方法的思路巧妙而又简单。既然我们难以处理最高峰的右侧，不妨正反求解两次。

设 L[i]L[i]L[i] 表示 iii 左侧的柱子高度的最大值， R[i]R[i]R[i] 表示 iii 右侧的柱子高度的最大值。显然柱子 iii 处的盛水高度取决于其左右两侧最高的柱子，也就是取决于 L[i]L[i]L[i] 和 R[i]R[i]R[i] ，那么柱子 iii 处的盛水体积为 min(L[i], R[i])−height[i]min(L[i],\ R[i]) - height[i]min(L[i], R[i])−height[i] 。

方法3：双指针扫描。

考虑在动态规划法中计算过的 LLL 和 RRR ，当 L[i]<R[i]L[i] < R[i]L[i]<R[i] 时，柱子 iii 处的盛水体积就取决于 L[i]L[i]L[i] ，反之则取决于 R[i]R[i]R[i] 。

那么我们就可以维护两个指针 leftleftleft 和 rightrightright ，令 left=0left = 0left=0 并从左往右移动， right=n−1right = n - 1right=n−1 并从右往左移动。当 height[left]<height[right]height[left] < height[right]height[left]<height[right] 时，我们可以放心地去计算左侧的答案而不会产生任何的错漏，反之亦然。这个保证是该方法正确的前提。

总的时间复杂度为 O(n)O(n)O(n) 。
