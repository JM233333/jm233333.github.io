Trie树

题目描述

你是csf，是一名刚刚考上xjtu的大一新生，无意间在图书馆看到了一本Quant算法秘籍，其中有神秘人留下的一道多重集合谜题，如果你将该谜题解出，你就有机会构建出足以支撑你创设一家Hedge Fund的基础模型。谜题如下：

有一个多重集合，初始这个集合是空的，你需要执行 nnn 次操作，有两种操作：

111 ：表示将目前集合中所有元素 +1+1+1 。

2 x2\ x2 x ：表示在多重集合终插入一个数 xxx 。

每做完一个操作，你需要输出当前集合中元素的异或和。

相关知识：

异或运算：语法为 x⊕yx \oplus yx⊕y ，满足 0⊕1=10 \oplus 1 = 10⊕1=1 , 1⊕0=11 \oplus 0 = 11⊕0=1 , 0⊕0=00 \oplus 0 = 00⊕0=0 , ​1⊕1=01 \oplus 1 = 01⊕1=0 。

异或和：多个数连续异或的结果，例如 a,b,c,da, b, c, da,b,c,d 的异或和为 a⊕b⊕c⊕da \oplus b \oplus c \oplus da⊕b⊕c⊕d 。

当两个数字进行异或时，首先将两个数字分别写成二进制，进行按位 0/1​0 / 1​0/1​ 异或，然后将所得二进制数再转化成十进制。

如： 3⊕6=0112⊕1102=1012=53 \oplus 6 = 011_2 \oplus 110_2 = 101_2 = 53⊕6=0112​⊕1102​=1012​=5 。

2 xi2\ x_i2 xi​ ：表示在多重集合终插入一个数 xix_ixi​ 。

输出格式

输出 nnn 行，每行一个整数表示当前集合内元素的异或和。

题解

吐槽：本题的数据范围为 1≤n≤2⋅1051 \le n \le 2 \cdot 10^51≤n≤2⋅105 ，因此 O(n2)O(n^2)O(n2) 的暴力做法显然是会TLE的，许多没学过时间复杂度分析的同学都栽在了这里，所以说还是要看榜开题啊，明明是全场0AC的题，居然还有人一直在做……

本题的前置知识为Trie树（即字典树），高效处理二进制数的异或类问题是Trie树的一个经典应用。

我们将所有插入数 xix_ixi​ 转化为二进制数，然后按从低位到高位的顺序将其数位插入Trie树。

对于 +1+1+1 操作，我们考虑对于当前所处的树上某层， 000 子树上的所有数的该位变成 111 ，且其下一层不会再变化， 111 子树上的所有数的该位变成0，且其下一层均 +1+1+1 。因此，对于 +1+1+1 操作，我们只需要对 111 子树递归处理，然后交换 000 子树和 111 子树即可。因为最多递归 log⁡Ai\log{A_i}logAi​ 层，所以单次操作的时间复杂度为 O(log⁡Ai)O(\log{A_i})O(logAi​) 。

对于求异或和操作，有多种本质相同的做法，其中一种做法是维护一个长为 323232 的数组，设 sum1[t]sum1[t]sum1[t] 表示第 ttt 位为 111 的数 xix_ixi​ 的个数，该信息显然可以在递归时顺便维护，那么只需要 O(log⁡Ai)O(\log{A_i})O(logAi​) 的时间即可用这 323232 个数求出答案。

总的时间复杂度为 O(nlog⁡Ai)O(n\log{A_i})O(nlogAi​) 。

高呼三声csfnb后csf和cyy开始打牌。

和一般的牌不同，他们打的牌上面有两个点数，分别设为 a, ba,\ ba, b 。如果csf获得这张牌，他会获得 aaa 点分数；如果 cyy获得这张牌，他会获得 bbb 点分数。

游戏开始后csf和cyy轮流从牌堆里拿牌，csf先手，直到牌堆的牌被拿完时游戏结束。csf的策略很简单，他每次选牌堆里 aaa 最大的那张牌据为己有，如果同时有多张牌有同样的 aaa ，那么他会取 bbb 最大的那一张。现在cyy想知道他可以获得的最大的分数。

题解

这是一道很容易出错的题，非常容易想出假算法掉入大坑，包括容易误以为本题不可贪心。

csf先手，csf使用优先 max{a}max\{a\}max{a} 其次 max{b}max\{b\}max{b} 的策略，cyy使用最优策略。

考虑不断更新答案使答案更优。

错误思路之一：先让cyy拿取理论最优解，即取走 n/2n/2n/2 张 bbb 最大的牌，然后让csf逐个拿牌，如果发现某张牌是cyy无法取走的，则将其舍弃。

正确思路：让csf逐个拿牌，cyy逐个拿牌，在拿牌时不断更新cyy的策略，使其更优。

首先，将牌按 aaa 从大到小、当 aaa 相同时按 bbb 从大到小排序，也就是按照csf的拿取顺序排序，然后让csf和cyy依次拿牌，并记录cyy拿的每一张牌。

当csf拿第 iii 张牌时，如果发现 bib_ibi​ 的值比cyy已经拿走的牌中的 min{b}min\{b\}min{b} 更大，则说明cyy可以舍弃这张牌，并选择拿走第 iii 张牌。因为被舍弃的这张牌一定是在更早的时间被拿走的，因此它一定可以被置换。每次置换都会使答案更优，因此贪心的正确性显然。

可以用 multiset 或 priority_queue 来维护cyy已经拿走的牌。时间复杂度为 O(nlog⁡n)O(n\log{n})O(nlogn) 。
